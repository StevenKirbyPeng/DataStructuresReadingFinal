二、1000 字以上條列式詳解

  • 觀念定位：這一頁不能只背文字，而要把它放回 Chapter 5 Trees Part I 的整體脈絡中理解。樹狀結構的核心是「階層關係」，也就是資料不是一條線，而是從一個 root 往下展開多個 subtree。只要題目出現父子、祖先、子孫、階層、遞迴、左右子樹、走訪順序，就通常與本章有關。
  • 與前面章節的關聯：Array、Stack、Queue、Linked List 都偏向線性資料結構；Tree 則是非線性資料結構。線性結構通常只能往前或往後找資料，但 Tree 可以用分支表達分類、語法、檔案目錄、組織圖與運算式。考試常要求你說明為什麼 Tree 比 Linked List 更適合表示階層資料。
  • 圖片說明：本頁投影片圖片的重點是把抽象定義視覺化。若圖片中出現節點與箭頭，請先找最上方 root，再觀察每個節點的 children。若圖片中出現陣列或方格，代表老師正在說明 representation；若圖片中出現 L、V、R，代表正在說明 traversal；若圖片中出現虛線或 thread，代表 threaded binary tree。
  • 學習方法：讀這頁時建議分成三層：第一層背定義，第二層會畫圖，第三層會寫程式。只背定義通常只能拿基本分；會畫圖可以處理表示法題；會寫 C++ 則可以處理演算法題與 trace code 題。
  • 常見錯誤：很多同學會把 level 與 height 混在一起，也會把 complete binary tree 和 full binary tree 混在一起。還有一個常見錯誤是以為 binary tree 是 general tree 的特殊情況；投影片中特別提醒 binary tree 與一般 tree 的定義差異，因為 binary tree 允許空樹，而且左右孩子有順序。
  • 解題策略：遇到樹的題目，第一步先判斷題目是在問「名詞」、「表示法」、「走訪」、「程式」、「複雜度」還是「應用」。名詞題通常靠圖判斷；表示法題要把結構轉換成括號、陣列或指標；走訪題要固定使用 LVR、VLR、LRV 或 level-order；程式題則要找 base case 與 recursive case。
  • 時間複雜度觀念：Tree 的多數走訪演算法都是 O(n)，因為每個節點都被拜訪一次。若是求高度，最壞情況也可能拜訪所有節點，所以是 O(n)。若是完整二元樹或平衡樹，某些查找可以降到 O(log n)，但本章 Part I 主要重點仍是表示法與走訪。
  • 空間複雜度觀念：遞迴走訪會用到 call stack，空間通常是 O(h)，h 是高度。若樹嚴重傾斜，h 可能等於 n；若樹接近平衡，h 約為 log n。Threaded Binary Tree 的重要性就在於能利用原本浪費的 NULL link，讓 inorder traversal 不需要額外 stack。
  • 考試提醒：老師常用投影片上的同一棵 A、B、C、D 範例樹改數字或改字母出題。你要能從圖中快速寫出 root、leaf、degree、height、preorder、inorder、postorder、level-order，以及可能的 array index 對應。

三、演算法步驟分析

  1. 先確認資料結構種類：general tree、binary tree、array representation、linked representation、expression tree 或 threaded binary tree。
  2. 若是定義題，列出 root、subtree、node、degree、level、height 等名詞。
  3. 若是走訪題，先在每個節點標記 L、V、R 的相對順序，再依規則輸出。
  4. 若是程式題，先寫 base case：節點為 NULL 時 return。
  5. 再寫 recursive case：依照題型處理 left subtree、right subtree 或 child/sibling。
  6. 最後補上時間複雜度與空間複雜度，並用一句話說明原因。

四、考試重點與老師常考題型

  • 給一張樹圖，要求寫出 root、leaf、degree、level、height。
  • 給 general tree，要求轉成 left-child right-sibling tree 或 degree-two tree。
  • 給 binary tree，要求寫出 preorder、inorder、postorder、level-order。
  • 給 expression tree，要求轉成 infix、prefix、postfix。
  • 給 C++ 程式，要求 trace 遞迴呼叫順序與輸出結果。
  • 給 threaded binary tree，要求找 inorder successor 或說明 thread 的方向。

  • 觀念定位：這一頁不能只背文字，而要把它放回 Chapter 5 Trees Part I 的整體脈絡中理解。樹狀結構的核心是「階層關係」，也就是資料不是一條線，而是從一個 root 往下展開多個 subtree。只要題目出現父子、祖先、子孫、階層、遞迴、左右子樹、走訪順序，就通常與本章有關。
  • 與前面章節的關聯：Array、Stack、Queue、Linked List 都偏向線性資料結構；Tree 則是非線性資料結構。線性結構通常只能往前或往後找資料，但 Tree 可以用分支表達分類、語法、檔案目錄、組織圖與運算式。考試常要求你說明為什麼 Tree 比 Linked List 更適合表示階層資料。

  • 學習方法：讀這頁時建議分成三層：第一層背定義，第二層會畫圖，第三層會寫程式。只背定義通常只能拿基本分；會畫圖可以處理表示法題；會寫 C++ 則可以處理演算法題與 trace code 題。
