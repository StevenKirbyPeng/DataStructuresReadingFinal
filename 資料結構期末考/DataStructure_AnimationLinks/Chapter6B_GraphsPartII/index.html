二、每頁 1000 字以上條列式詳解

  • 觀念定位：Chapter 6 Graphs Part II 延續 Part I 的 graph 定義、representation、DFS/BFS 與 MST，進一步討論路徑問題與 DAG 應用。這一份投影片的核心包含 shortest paths、transitive closure、AOV network、topological sort、AOE network、critical path，以及附錄 static timing analysis。這些主題共同重點是：在 graph 中找出可達性、最短成本、合法執行順序或最長延遲路徑。
  • Shortest Path 重點：Shortest path 題目要先分辨 edge weight 是否為 nonnegative。若所有權重非負，Dijkstra 可以用 greedy，每次挑目前 dist 最小且尚未確定的 vertex。若允許 negative edge，但不能有 negative cycle，則用 Bellman-Ford。若要所有點到所有點的最短路徑，使用 Floyd-Warshall 的 all-pairs dynamic programming。
  • Dijkstra 的核心：Dijkstra 維護 dist[u]，代表目前從 source 到 u 的最短距離估計。每次 ExtractMin 選出 dist 最小的 vertex，將它視為已確定，接著 relax 其 outgoing edges。如果 dist[u]+w(u,v) 小於 dist[v]，就更新 dist[v] 與 parent[v]。Dijkstra 與 Prim 很像，但 Dijkstra 更新的是 source 到該點的總路徑長；Prim 更新的是把該點接進 MST 的最小邊。
  • Bellman-Ford 的核心：Bellman-Ford 的想法是如果沒有 negative cycle，最短路徑最多只需要 n-1 條邊。因此可以重複進行 edge relaxation，逐步允許使用更多條邊。投影片中的 dist_k[u] 表示使用最多 k 條邊可得到的最短距離。考試常問它為什麼能處理負邊，答案是它不依賴 greedy 的永久確定，而是反覆改善。
  • All-Pairs 與 Transitive Closure：Floyd-Warshall 定義 a^k[i][j]：從 i 到 j，只允許經過 index ≤ k 的中繼點時的最短距離。轉移式是 min(a[i][j], a[i][k]+a[k][j])。Transitive closure 則把距離問題改成可達性問題，更新式改成 a[i][j] = a[i][j] OR (a[i][k] AND a[k][j])。前者求最短距離，後者回答是否有 path。
  • AOV Network：AOV network 是 activity-on-vertex，vertex 代表工作或課程，edge 代表 precedence relation。例如課程先修關係中，C1 和 C2 必須在 C3 前完成。AOV 必須是 DAG，因為若有 cycle，就代表某些工作互相依賴，無法排出合法執行順序。這也導向 topological sort。
  • Topological Sort：Topological order 是 DAG 的一種線性排序，若有 edge <i,j>，則 i 必須出現在 j 前面。它不一定唯一。常見演算法是使用 indegree：先找 indegree 為 0 的 vertex，輸出它，刪除其 outgoing edges，更新其他 vertex 的 indegree。若最後無法輸出所有 vertex，表示 graph 有 cycle。
  • AOE 與 Critical Path：AOE network 是 activity-on-edge，edge 代表活動與 duration，vertex 代表事件完成點。Critical path 是最長路徑，代表整個 project 最快完成時間。Earliest event time 用 topological order 往前算最大值，latest event time 用 reverse topological order 往回算最小值。若 activity 的 slack = latest - earliest = 0，就是 critical activity。
  • Static Timing Analysis：投影片附錄把 combinational logic network 視為 DAG，vertex 代表 gate/input/output，edge 代表 signal wire。Arrival time 是信號最早到達時間，required time 是不違反總延遲限制下最晚需要到達的時間，slack = required - arrival。Slack 為 0 或負值代表 critical 或 timing violation，這與 AOE critical path 的觀念高度相似。
  • 考試常見錯誤：Dijkstra 不能用在有負邊的圖；Bellman-Ford 可以處理負邊但不能接受 negative cycle；Floyd-Warshall 是 O(n^3)，不是 O(E log V)；topological sort 只適用於 DAG；AOV 的 vertex 是 activity，但 AOE 的 edge 是 activity；critical path 是最長路徑，不是最短路徑。

三、演算法步驟分析

  1. 先判斷題目類型：shortest path、reachability、topological order、critical path 或 STA。
  2. 若是 shortest path，檢查是否有負邊；無負邊用 Dijkstra，有負邊用 Bellman-Ford。
  3. 若是 all-pairs，使用 Floyd-Warshall 三層迴圈，以 k 當中繼點。
  4. 若是 transitive closure，把加法與 min 改成 AND 與 OR。
  5. 若是 topological sort，先計算 indegree，再反覆輸出 0-indegree vertices。
  6. 若是 critical path，先算 earliest time，再反向算 latest time，最後找 slack=0。
  7. 作答最後一定補上時間複雜度、適用限制與是否需要 DAG。

四、考試重點與老師常考題型分析

  • 給 weighted digraph，要求用 Dijkstra 算 dist[] 每一輪變化。
  • 給含負邊圖，要求用 Bellman-Ford 算 dist_k[] 表格。
  • 給 adjacency matrix，要求用 Floyd-Warshall 更新 all-pairs shortest paths。
  • 給 unweighted digraph，要求算 transitive closure A+ 或 A*。
  • 給課程先修圖，要求找 topological order 或判斷是否有 cycle。
  • 給 AOE network，要求算 earliest time、latest time、slack 與 critical path。
  • 給 logic DAG，要求找 arrival time、required time、slack 與 critical path delay。

  • Shortest Path 重點：Shortest path 題目要先分辨 edge weight 是否為 nonnegative。若所有權重非負，Dijkstra 可以用 greedy，每次挑目前 dist 最小且尚未確定的 vertex。若允許 negative edge，但不能有 negative cycle，則用 Bellman-Ford。若要所有點到所有點的最短路徑，使用 Floyd-Warshall 的 all-pairs dynamic programming。
  • Dijkstra 的核心：Dijkstra 維護 dist[u]，代表目前從 source 到 u 的最短距離估計。每次 ExtractMin 選出 dist 最小的 vertex，將它視為已確定，接著 relax 其 outgoing edges。如果 dist[u]+w(u,v) 小於 dist[v]，就更新 dist[v] 與 parent[v]。Dijkstra 與 Prim 很像，但 Dijkstra 更新的是 source 到該點的總路徑長；Prim 更新的是把該點接進 MST 的最小邊。
  • Bellman-Ford 的核心：Bellman-Ford 的想法是如果沒有 negative cycle，最短路徑最多只需要 n-1 條邊。因此可以重複進行 edge relaxation，逐步允許使用更多條邊。投影片中的 dist_k[u] 表示使用最多 k 條邊可得到的最短距離。考試常問它為什麼能處理負邊，答案是它不依賴 greedy 的永久確定，而是反覆改善。
  • All-Pairs 與 Transitive Closure：Floyd-Warshall 定義 a^k[i][j]：從 i 到 j，只允許經過 index ≤ k 的中繼點時的最短距離。轉移式是 min(a[i][j], a[i][k]+a[k][j])。Transitive closure 則把距離問題改成可達性問題，更新式改成 a[i][j] = a[i][j] OR (a[i][k] AND a[k][j])。前者求最短距離，後者回答是否有 path。
  • AOV Network：AOV network 是 activity-on-vertex，vertex 代表工作或課程，edge 代表 precedence relation。例如課程先修關係中，C1 和 C2 必須在 C3 前完成。AOV 必須是 DAG，因為若有 cycle，就代表某些工作互相依賴，無法排出合法執行順序。這也導向 topological sort。
  • Topological Sort：Topological order 是 DAG 的一種線性排序，若有 edge <i,j>，則 i 必須出現在 j 前面。它不一定唯一。常見演算法是使用 indegree：先找 indegree 為 0 的 vertex，輸出它，刪除其 outgoing edges，更新其他 vertex 的 indegree。若最後無法輸出所有 vertex，表示 graph 有 cycle。
  • AOE 與 Critical Path：AOE network 是 activity-on-edge，edge 代表活動與 duration，vertex 代表事件完成點。Critical path 是最長路徑，代表整個 project 最快完成時間。Earliest event time 用 topological order 往前算最大值，latest event time 用 reverse topological order 往回算最小值。若 activity 的 slack = latest - earliest = 0，就是 critical activity。
  • Static Timing Analysis：投影片附錄把 combinational logic network 視為 DAG，vertex 代表 gate/input/output，edge 代表 signal wire。Arrival time 是信號最早到達時間，required time 是不違反總延遲限制下最晚需要到達的時間，slack = required - arrival。Slack 為 0 或負值代表 critical 或 timing violation，這與 AOE critical path 的觀念高度相似。

  3. 若是 all-pairs，使用 Floyd-Warshall 三層迴圈，以 k 當中繼點。
  4. 若是 transitive closure，把加法與 min 改成 AND 與 OR。
  5. 若是 topological sort，先計算 indegree，再反覆輸出 0-indegree vertices。
  6. 若是 critical path，先算 earliest time，再反向算 latest time，最後找 slack=0。
  7. 作答最後一定補上時間複雜度、適用限制與是否需要 DAG。

  • Shortest Path 重點：Shortest path 題目要先分辨 edge weight 是否為 nonnegative。若所有權重非負，Dijkstra 可以用 greedy，每次挑目前 dist 最小且尚未確定的 vertex。若允許 negative edge，但不能有 negative cycle，則用 Bellman-Ford。若要所有點到所有點的最短路徑，使用 Floyd-Warshall 的 all-pairs dynamic programming。
  • Dijkstra 的核心：Dijkstra 維護 dist[u]，代表目前從 source 到 u 的最短距離估計。每次 ExtractMin 選出 dist 最小的 vertex，將它視為已確定，接著 relax 其 outgoing edges。如果 dist[u]+w(u,v) 小於 dist[v]，就更新 dist[v] 與 parent[v]。Dijkstra 與 Prim 很像，但 Dijkstra 更新的是 source 到該點的總路徑長；Prim 更新的是把該點接進 MST 的最小邊。
  • Bellman-Ford 的核心：Bellman-Ford 的想法是如果沒有 negative cycle，最短路徑最多只需要 n-1 條邊。因此可以重複進行 edge relaxation，逐步允許使用更多條邊。投影片中的 dist_k[u] 表示使用最多 k 條邊可得到的最短距離。考試常問它為什麼能處理負邊，答案是它不依賴 greedy 的永久確定，而是反覆改善。
  • All-Pairs 與 Transitive Closure：Floyd-Warshall 定義 a^k[i][j]：從 i 到 j，只允許經過 index ≤ k 的中繼點時的最短距離。轉移式是 min(a[i][j], a[i][k]+a[k][j])。Transitive closure 則把距離問題改成可達性問題，更新式改成 a[i][j] = a[i][j] OR (a[i][k] AND a[k][j])。前者求最短距離，後者回答是否有 path。
  • AOV Network：AOV network 是 activity-on-vertex，vertex 代表工作或課程，edge 代表 precedence relation。例如課程先修關係中，C1 和 C2 必須在 C3 前完成。AOV 必須是 DAG，因為若有 cycle，就代表某些工作互相依賴，無法排出合法執行順序。這也導向 topological sort。
  • Topological Sort：Topological order 是 DAG 的一種線性排序，若有 edge <i,j>，則 i 必須出現在 j 前面。它不一定唯一。常見演算法是使用 indegree：先找 indegree 為 0 的 vertex，輸出它，刪除其 outgoing edges，更新其他 vertex 的 indegree。若最後無法輸出所有 vertex，表示 graph 有 cycle。
  • AOE 與 Critical Path：AOE network 是 activity-on-edge，edge 代表活動與 duration，vertex 代表事件完成點。Critical path 是最長路徑，代表整個 project 最快完成時間。Earliest event time 用 topological order 往前算最大值，latest event time 用 reverse topological order 往回算最小值。若 activity 的 slack = latest - earliest = 0，就是 critical activity。

  1. 先判斷題目類型：shortest path、reachability、topological order、critical path 或 STA。
  2. 若是 shortest path，檢查是否有負邊；無負邊用 Dijkstra，有負邊用 Bellman-Ford。
  3. 若是 all-pairs，使用 Floyd-Warshall 三層迴圈，以 k 當中繼點。

  5. 若是 topological sort，先計算 indegree，再反覆輸出 0-indegree vertices。
  6. 若是 critical path，先算 earliest time，再反向算 latest time，最後找 slack=0。
  7. 作答最後一定補上時間複雜度、適用限制與是否需要 DAG。

  • Dijkstra 的核心：Dijkstra 維護 dist[u]，代表目前從 source 到 u 的最短距離估計。每次 ExtractMin 選出 dist 最小的 vertex，將它視為已確定，接著 relax 其 outgoing edges。如果 dist[u]+w(u,v) 小於 dist[v]，就更新 dist[v] 與 parent[v]。Dijkstra 與 Prim 很像，但 Dijkstra 更新的是 source 到該點的總路徑長；Prim 更新的是把該點接進 MST 的最小邊。
  • Bellman-Ford 的核心：Bellman-Ford 的想法是如果沒有 negative cycle，最短路徑最多只需要 n-1 條邊。因此可以重複進行 edge relaxation，逐步允許使用更多條邊。投影片中的 dist_k[u] 表示使用最多 k 條邊可得到的最短距離。考試常問它為什麼能處理負邊，答案是它不依賴 greedy 的永久確定，而是反覆改善。
  • All-Pairs 與 Transitive Closure：Floyd-Warshall 定義 a^k[i][j]：從 i 到 j，只允許經過 index ≤ k 的中繼點時的最短距離。轉移式是 min(a[i][j], a[i][k]+a[k][j])。Transitive closure 則把距離問題改成可達性問題，更新式改成 a[i][j] = a[i][j] OR (a[i][k] AND a[k][j])。前者求最短距離，後者回答是否有 path。
  • AOV Network：AOV network 是 activity-on-vertex，vertex 代表工作或課程，edge 代表 precedence relation。例如課程先修關係中，C1 和 C2 必須在 C3 前完成。AOV 必須是 DAG，因為若有 cycle，就代表某些工作互相依賴，無法排出合法執行順序。這也導向 topological sort。
  • Topological Sort：Topological order 是 DAG 的一種線性排序，若有 edge <i,j>，則 i 必須出現在 j 前面。它不一定唯一。常見演算法是使用 indegree：先找 indegree 為 0 的 vertex，輸出它，刪除其 outgoing edges，更新其他 vertex 的 indegree。若最後無法輸出所有 vertex，表示 graph 有 cycle。
  • AOE 與 Critical Path：AOE network 是 activity-on-edge，edge 代表活動與 duration，vertex 代表事件完成點。Critical path 是最長路徑，代表整個 project 最快完成時間。Earliest event time 用 topological order 往前算最大值，latest event time 用 reverse topological order 往回算最小值。若 activity 的 slack = latest - earliest = 0，就是 critical activity。
  • Static Timing Analysis：投影片附錄把 combinational logic network 視為 DAG，vertex 代表 gate/input/output，edge 代表 signal wire。Arrival time 是信號最早到達時間，required time 是不違反總延遲限制下最晚需要到達的時間，slack = required - arrival。Slack 為 0 或負值代表 critical 或 timing violation，這與 AOE critical path 的觀念高度相似。
  • 考試常見錯誤：Dijkstra 不能用在有負邊的圖；Bellman-Ford 可以處理負邊但不能接受 negative cycle；Floyd-Warshall 是 O(n^3)，不是 O(E log V)；topological sort 只適用於 DAG；AOV 的 vertex 是 activity，但 AOE 的 edge 是 activity；critical path 是最長路徑，不是最短路徑。

  1. 先判斷題目類型：shortest path、reachability、topological order、critical path 或 STA。
  2. 若是 shortest path，檢查是否有負邊；無負邊用 Dijkstra，有負邊用 Bellman-Ford。
  3. 若是 all-pairs，使用 Floyd-Warshall 三層迴圈，以 k 當中繼點。
  4. 若是 transitive closure，把加法與 min 改成 AND 與 OR。
  5. 若是 topological sort，先計算 indegree，再反覆輸出 0-indegree vertices。
  6. 若是 critical path，先算 earliest time，再反向算 latest time，最後找 slack=0。
  7. 作答最後一定補上時間複雜度、適用限制與是否需要 DAG。

  • 給 AOE network，要求算 earliest time、latest time、slack 與 critical path。

  • Bellman-Ford 的核心：Bellman-Ford 的想法是如果沒有 negative cycle，最短路徑最多只需要 n-1 條邊。因此可以重複進行 edge relaxation，逐步允許使用更多條邊。投影片中的 dist_k[u] 表示使用最多 k 條邊可得到的最短距離。考試常問它為什麼能處理負邊，答案是它不依賴 greedy 的永久確定，而是反覆改善。
  • All-Pairs 與 Transitive Closure：Floyd-Warshall 定義 a^k[i][j]：從 i 到 j，只允許經過 index ≤ k 的中繼點時的最短距離。轉移式是 min(a[i][j], a[i][k]+a[k][j])。Transitive closure 則把距離問題改成可達性問題，更新式改成 a[i][j] = a[i][j] OR (a[i][k] AND a[k][j])。前者求最短距離，後者回答是否有 path。
  • AOV Network：AOV network 是 activity-on-vertex，vertex 代表工作或課程，edge 代表 precedence relation。例如課程先修關係中，C1 和 C2 必須在 C3 前完成。AOV 必須是 DAG，因為若有 cycle，就代表某些工作互相依賴，無法排出合法執行順序。這也導向 topological sort。
  • Topological Sort：Topological order 是 DAG 的一種線性排序，若有 edge <i,j>，則 i 必須出現在 j 前面。它不一定唯一。常見演算法是使用 indegree：先找 indegree 為 0 的 vertex，輸出它，刪除其 outgoing edges，更新其他 vertex 的 indegree。若最後無法輸出所有 vertex，表示 graph 有 cycle。
  • AOE 與 Critical Path：AOE network 是 activity-on-edge，edge 代表活動與 duration，vertex 代表事件完成點。Critical path 是最長路徑，代表整個 project 最快完成時間。Earliest event time 用 topological order 往前算最大值，latest event time 用 reverse topological order 往回算最小值。若 activity 的 slack = latest - earliest = 0，就是 critical activity。
  • Static Timing Analysis：投影片附錄把 combinational logic network 視為 DAG，vertex 代表 gate/input/output，edge 代表 signal wire。Arrival time 是信號最早到達時間，required time 是不違反總延遲限制下最晚需要到達的時間，slack = required - arrival。Slack 為 0 或負值代表 critical 或 timing violation，這與 AOE critical path 的觀念高度相似。
  • 考試常見錯誤：Dijkstra 不能用在有負邊的圖；Bellman-Ford 可以處理負邊但不能接受 negative cycle；Floyd-Warshall 是 O(n^3)，不是 O(E log V)；topological sort 只適用於 DAG；AOV 的 vertex 是 activity，但 AOE 的 edge 是 activity；critical path 是最長路徑，不是最短路徑。

三、演算法步驟分析

  1. 先判斷題目類型：shortest path、reachability、topological order、critical path 或 STA。
  2. 若是 shortest path，檢查是否有負邊；無負邊用 Dijkstra，有負邊用 Bellman-Ford。
  3. 若是 all-pairs，使用 Floyd-Warshall 三層迴圈，以 k 當中繼點。
  4. 若是 transitive closure，把加法與 min 改成 AND 與 OR。
  5. 若是 topological sort，先計算 indegree，再反覆輸出 0-indegree vertices。
  6. 若是 critical path，先算 earliest time，再反向算 latest time，最後找 slack=0。
  7. 作答最後一定補上時間複雜度、適用限制與是否需要 DAG。

  • Dijkstra 的核心：Dijkstra 維護 dist[u]，代表目前從 source 到 u 的最短距離估計。每次 ExtractMin 選出 dist 最小的 vertex，將它視為已確定，接著 relax 其 outgoing edges。如果 dist[u]+w(u,v) 小於 dist[v]，就更新 dist[v] 與 parent[v]。Dijkstra 與 Prim 很像，但 Dijkstra 更新的是 source 到該點的總路徑長；Prim 更新的是把該點接進 MST 的最小邊。
  • Bellman-Ford 的核心：Bellman-Ford 的想法是如果沒有 negative cycle，最短路徑最多只需要 n-1 條邊。因此可以重複進行 edge relaxation，逐步允許使用更多條邊。投影片中的 dist_k[u] 表示使用最多 k 條邊可得到的最短距離。考試常問它為什麼能處理負邊，答案是它不依賴 greedy 的永久確定，而是反覆改善。
  • All-Pairs 與 Transitive Closure：Floyd-Warshall 定義 a^k[i][j]：從 i 到 j，只允許經過 index ≤ k 的中繼點時的最短距離。轉移式是 min(a[i][j], a[i][k]+a[k][j])。Transitive closure 則把距離問題改成可達性問題，更新式改成 a[i][j] = a[i][j] OR (a[i][k] AND a[k][j])。前者求最短距離，後者回答是否有 path。
  • AOV Network：AOV network 是 activity-on-vertex，vertex 代表工作或課程，edge 代表 precedence relation。例如課程先修關係中，C1 和 C2 必須在 C3 前完成。AOV 必須是 DAG，因為若有 cycle，就代表某些工作互相依賴，無法排出合法執行順序。這也導向 topological sort。
  • Topological Sort：Topological order 是 DAG 的一種線性排序，若有 edge <i,j>，則 i 必須出現在 j 前面。它不一定唯一。常見演算法是使用 indegree：先找 indegree 為 0 的 vertex，輸出它，刪除其 outgoing edges，更新其他 vertex 的 indegree。若最後無法輸出所有 vertex，表示 graph 有 cycle。
  • AOE 與 Critical Path：AOE network 是 activity-on-edge，edge 代表活動與 duration，vertex 代表事件完成點。Critical path 是最長路徑，代表整個 project 最快完成時間。Earliest event time 用 topological order 往前算最大值，latest event time 用 reverse topological order 往回算最小值。若 activity 的 slack = latest - earliest = 0，就是 critical activity。
  • Static Timing Analysis：投影片附錄把 combinational logic network 視為 DAG，vertex 代表 gate/input/output，edge 代表 signal wire。Arrival time 是信號最早到達時間，required time 是不違反總延遲限制下最晚需要到達的時間，slack = required - arrival。Slack 為 0 或負值代表 critical 或 timing violation，這與 AOE critical path 的觀念高度相似。
  • 考試常見錯誤：Dijkstra 不能用在有負邊的圖；Bellman-Ford 可以處理負邊但不能接受 negative cycle；Floyd-Warshall 是 O(n^3)，不是 O(E log V)；topological sort 只適用於 DAG；AOV 的 vertex 是 activity，但 AOE 的 edge 是 activity；critical path 是最長路徑，不是最短路徑。

三、演算法步驟分析

  1. 先判斷題目類型：shortest path、reachability、topological order、critical path 或 STA。
  2. 若是 shortest path，檢查是否有負邊；無負邊用 Dijkstra，有負邊用 Bellman-Ford。
  3. 若是 all-pairs，使用 Floyd-Warshall 三層迴圈，以 k 當中繼點。
  4. 若是 transitive closure，把加法與 min 改成 AND 與 OR。
  5. 若是 topological sort，先計算 indegree，再反覆輸出 0-indegree vertices。
  6. 若是 critical path，先算 earliest time，再反向算 latest time，最後找 slack=0。
  7. 作答最後一定補上時間複雜度、適用限制與是否需要 DAG。

四、考試重點與老師常考題型分析

  • 給 weighted digraph，要求用 Dijkstra 算 dist[] 每一輪變化。
  • 給含負邊圖，要求用 Bellman-Ford 算 dist_k[] 表格。
  • 給 adjacency matrix，要求用 Floyd-Warshall 更新 all-pairs shortest paths。
  • 給 unweighted digraph，要求算 transitive closure A+ 或 A*。

  • 給 AOE network，要求算 earliest time、latest time、slack 與 critical path。
  • 給 logic DAG，要求找 arrival time、required time、slack 與 critical path delay。

  • All-Pairs 與 Transitive Closure：Floyd-Warshall 定義 a^k[i][j]：從 i 到 j，只允許經過 index ≤ k 的中繼點時的最短距離。轉移式是 min(a[i][j], a[i][k]+a[k][j])。Transitive closure 則把距離問題改成可達性問題，更新式改成 a[i][j] = a[i][j] OR (a[i][k] AND a[k][j])。前者求最短距離，後者回答是否有 path。
  • AOV Network：AOV network 是 activity-on-vertex，vertex 代表工作或課程，edge 代表 precedence relation。例如課程先修關係中，C1 和 C2 必須在 C3 前完成。AOV 必須是 DAG，因為若有 cycle，就代表某些工作互相依賴，無法排出合法執行順序。這也導向 topological sort。
  • Topological Sort：Topological order 是 DAG 的一種線性排序，若有 edge <i,j>，則 i 必須出現在 j 前面。它不一定唯一。常見演算法是使用 indegree：先找 indegree 為 0 的 vertex，輸出它，刪除其 outgoing edges，更新其他 vertex 的 indegree。若最後無法輸出所有 vertex，表示 graph 有 cycle。
  • AOE 與 Critical Path：AOE network 是 activity-on-edge，edge 代表活動與 duration，vertex 代表事件完成點。Critical path 是最長路徑，代表整個 project 最快完成時間。Earliest event time 用 topological order 往前算最大值，latest event time 用 reverse topological order 往回算最小值。若 activity 的 slack = latest - earliest = 0，就是 critical activity。
  • Static Timing Analysis：投影片附錄把 combinational logic network 視為 DAG，vertex 代表 gate/input/output，edge 代表 signal wire。Arrival time 是信號最早到達時間，required time 是不違反總延遲限制下最晚需要到達的時間，slack = required - arrival。Slack 為 0 或負值代表 critical 或 timing violation，這與 AOE critical path 的觀念高度相似。
  • 考試常見錯誤：Dijkstra 不能用在有負邊的圖；Bellman-Ford 可以處理負邊但不能接受 negative cycle；Floyd-Warshall 是 O(n^3)，不是 O(E log V)；topological sort 只適用於 DAG；AOV 的 vertex 是 activity，但 AOE 的 edge 是 activity；critical path 是最長路徑，不是最短路徑。

三、演算法步驟分析

  1. 先判斷題目類型：shortest path、reachability、topological order、critical path 或 STA。
  2. 若是 shortest path，檢查是否有負邊；無負邊用 Dijkstra，有負邊用 Bellman-Ford。
  3. 若是 all-pairs，使用 Floyd-Warshall 三層迴圈，以 k 當中繼點。
  4. 若是 transitive closure，把加法與 min 改成 AND 與 OR。
  5. 若是 topological sort，先計算 indegree，再反覆輸出 0-indegree vertices。
  6. 若是 critical path，先算 earliest time，再反向算 latest time，最後找 slack=0。
  7. 作答最後一定補上時間複雜度、適用限制與是否需要 DAG。

四、考試重點與老師常考題型分析

  • 給 weighted digraph，要求用 Dijkstra 算 dist[] 每一輪變化。
  • 給含負邊圖，要求用 Bellman-Ford 算 dist_k[] 表格。
  • 給 adjacency matrix，要求用 Floyd-Warshall 更新 all-pairs shortest paths。
  • 給 unweighted digraph，要求算 transitive closure A+ 或 A*。
  • 給課程先修圖，要求找 topological order 或判斷是否有 cycle。
  • 給 AOE network，要求算 earliest time、latest time、slack 與 critical path。
  • 給 logic DAG，要求找 arrival time、required time、slack 與 critical path delay。

  6. 若是 critical path，先算 earliest time，再反向算 latest time，最後找 slack=0。
