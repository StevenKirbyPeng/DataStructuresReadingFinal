題目原文

Explain how the Königsberg Bridge Problem can be abstracted into a graph. What are the vertices and edges? Why is there no Eulerian walk?

中文翻譯

說明如何把科尼格茲堡橋問題抽象成圖形。什麼是 vertex？什麼是 edge？為什麼沒有 Eulerian walk？

核心觀念

abstractionvertexedgeEulerian walk

動態圖解

解題步驟

  1. 把陸地區域抽象成 vertex。
  2. 把橋抽象成 edge。
  3. Eulerian walk 要每條 edge 走一次且回到起點，所有 vertex 的 degree 必須是偶數。
  4. 科尼格茲堡橋抽象圖中有奇數 degree 的 vertex，所以不可能。
完整答案：The land areas become vertices, and the bridges become edges. Since an Eulerian walk returning to the start requires every vertex to have even degree, but this graph contains vertices of odd degree, no Eulerian walk exists.
考試容易錯：不要把橋當 vertex；橋是 edge，陸地才是 vertex。

題目原文

Given V = {0,1,2,3} and E = {(0,1),(0,2),(1,2),(2,3)}.
a. Is this graph directed or undirected?
b. List all vertices adjacent to vertex 2.
c. If the graph were directed with edges <0,1>, <0,2>, <1,2>, <2,3>, what are the outgoing neighbors of 2?

中文翻譯

給定 V 與 E，判斷是有向圖或無向圖，列出 vertex 2 的相鄰點；若改成 directed graph，求 2 的 outgoing neighbors。

圖解

解題步驟

  1. 使用小括號 (u,v) 表示無向邊時，(u,v) 與 (v,u) 相同。
  2. vertex 2 連到 0、1、3，所以 adjacent vertices 是 0、1、3。
  3. 若是 directed edges，從 2 出發只有 <2,3>，所以 outgoing neighbor 是 3。
完整答案：a. Undirected graph. b. Adjacent to 2: {0,1,3}. c. In the directed version, outgoing neighbors of 2: {3}。
考試容易錯：Directed graph 要分 outgoing 與 incoming；undirected graph 則不用分方向。

題目原文

For a simple graph with n vertices, how many edges are in a complete undirected graph and a complete directed graph? If n = 5, compute both values. Also explain the degree-sum formula for an undirected graph.

中文翻譯

n 個頂點的 simple graph，如果是 complete undirected graph 與 complete directed graph，各有幾條邊？n=5 時計算答案。並說明無向圖 degree-sum 公式。

核心觀念

complete graphdegreedegree-sum formula

圖解

解題步驟

  1. Complete undirected graph：任兩點之間一條無向邊，所以 C(n,2)=n(n−1)/2。
  2. Complete directed graph：每對不同頂點有兩個方向，所以 n(n−1)。
  3. n=5：undirected = 5×4/2=10；directed = 5×4=20。
  4. 無向圖每條邊會貢獻兩個 incident degree，所以所有 degree 總和 = 2e。
完整答案：Complete undirected graph has n(n−1)/2 edges; complete directed graph has n(n−1) edges. When n=5, they are 10 and 20. For undirected graphs, ∑degree(v)=2|E|。

a. Is 0,1,2,0 a cycle?
b. Is the graph connected?

根據邊集合判斷路徑是否形成 cycle、整張圖是否 connected，以及 connected components 有幾個。

題目原文

Construct the adjacency matrix of the following directed graph:
V = {0,1,2,3}
E = {<0,1>, <0,2>, <1,2>, <2,0>, <2,3>, <3,3>}
Then find the in-degree and out-degree of vertex 2.

中文翻譯

請建立 directed graph 的 adjacency matrix，並計算 vertex 2 的 in-degree 與 out-degree。

圖解與矩陣

解題步驟

  1. Directed matrix 中，a[i][j]=1 表示有邊 i→j。
  2. 第 2 列代表 vertex 2 出去的邊：2→0、2→3，所以 out-degree(2)=2。
  3. 第 2 欄代表進入 vertex 2 的邊：0→2、1→2，所以 in-degree(2)=2。
  4. 3→3 是 self loop，矩陣 a[3][3]=1。
完整答案：Matrix rows 0..3 are [0,1,1,0], [0,0,1,0], [1,0,0,1], [0,0,0,1]。in-degree(2)=2，out-degree(2)=2。
考試容易錯：Directed graph 的 in-degree 看「欄 column」，out-degree 看「列 row」。

題目原文

For the undirected graph with edges:
(0,1), (0,2), (1,2), (1,3), (2,3)
a. Write its adjacency list.
b. How many edge nodes are needed in an adjacency multilist?
c. Which representation is better for a sparse graph?

中文翻譯

給定無向圖，寫出 adjacency list，判斷 adjacency multilist 需要多少 edge node，並說明 sparse graph 較適合哪種表示法。

圖解

解題步驟

題目原文

Starting from vertex 0, perform DFS on the graph below.
When there is a choice, always visit the smallest-numbered unvisited neighbor first.
Edges: (0,1), (0,2), (1,3), (1,4), (2,5), (2,6), (3,7), (4,7), (5,7), (6,7)
Write the DFS traversal order and draw the DFS spanning tree.

中文翻譯

從 0 開始做 DFS；若有多個選擇，先走編號最小的未拜訪鄰點。請寫出 DFS 順序並畫出 DFS spanning tree。

DFS 動態圖解

解題步驟

  1. DFS 是一路往深處走，走到底再 backtrack。
  2. 從 0 開始：0→1→3→7。
  3. 7 的未拜訪鄰點依序可到 4、5、6；先走 4，再回來走 5，再走 2，再走 6。
  4. 依照最小編號鄰點規則，完整 order 如答案。
完整答案：DFS order: 0, 1, 3, 7, 4, 5, 2, 6。DFS tree edges: (0,1), (1,3), (3,7), (7,4), (7,5), (5,2), (2,6)。

完整 C++ 參考程式碼

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

class Graph {
private:
    int n;                         // 頂點數量
    vector<vector<int>> adj;       // adjacency list
    vector<bool> visited;          // 紀錄是否拜訪過

    // DFS 的遞迴工作函式
    void dfsWork(int v) {
        visited[v] = true;         // 標記目前頂點已拜訪
        cout << v << " ";         // 輸出 DFS 拜訪順序

        // 依照題目規定：小編號鄰點優先
        for (int w : adj[v]) {
            if (!visited[w]) {
                dfsWork(w);        // 往尚未拜訪的鄰點深入
            }
        }
    }

public:
    Graph(int vertices) : n(vertices), adj(vertices), visited(vertices, false) {}

    // 加入無向邊
    void addEdge(int u, int v) {
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    // 排序所有 adjacency list，保證小編號優先
    void sortNeighbors() {
        for (auto &list : adj) sort(list.begin(), list.end());
    }

    // DFS driver function
    void DFS(int start) {
        fill(visited.begin(), visited.end(), false);
        dfsWork(start);
        cout << endl;
    }
};

函式說明

addEdge(u,v)
加入無向邊，因此 u 的 list 加 v，v 的 list 也加 u。
sortNeighbors()
把每個 adjacency list 排序，符合「小編號優先」。
dfsWork(v)
遞迴拜訪 v，再往未拜訪鄰點深入。
DFS(start)
初始化 visited，並從 start 開始。

題目原文

Using the same graph as Question 7, perform BFS starting from vertex 0.
When there is a choice, always visit the smallest-numbered unvisited neighbor first.
Write the BFS traversal order and draw the BFS spanning tree.

中文翻譯

使用第 7 題同一張圖，從 0 做 BFS；若有多個選擇，先拜訪編號最小的未拜訪鄰點。請寫出 BFS 順序與 BFS spanning tree。

BFS 動態圖解

解題步驟

  1. BFS 使用 queue，一層一層往外擴散。
  2. 從 0 開始，先放入 1、2。
  3. 處理 1 時加入 3、4；處理 2 時加入 5、6。
  4. 最後處理 3 時加入 7。
完整答案：BFS order: 0, 1, 2, 3, 4, 5, 6, 7。BFS tree edges: (0,1), (0,2), (1,3), (1,4), (2,5), (2,6), (3,7)。

完整 C++ 參考程式碼

#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>
using namespace std;

void BFS(const vector<vector<int>>& adj, int start) {
    vector<bool> visited(adj.size(), false); // 紀錄是否拜訪過
    queue<int> q;                            // BFS 使用 queue

    visited[start] = true;                    // 起點先標記
    q.push(start);                            // 起點進入 queue

    while (!q.empty()) {
        int v = q.front();                    // 取出最早進入 queue 的頂點
        q.pop();
        cout << v << " ";

        // 依序檢查 v 的鄰點
        for (int w : adj[v]) {
            if (!visited[w]) {
                visited[w] = true;            // 入 queue 前就標記，避免重複入隊
                q.push(w);
            }
        }
    }
}
考試容易錯：BFS 是「進 queue 時」就標記 visited，不是出 queue 才標記，否則同一節點可能重複入隊。

題目原文

Explain the difference between a DFS spanning tree and a BFS spanning tree. In an undirected graph, what is the difference between back edges and cross edges with respect to a spanning tree?

中文翻譯

說明 DFS spanning tree 與 BFS spanning tree 的差異。對於無向圖，相對於 spanning tree，back edge 與 cross edge 有何差別？

圖解

解題步驟

  1. DFS tree 像 preorder tree traversal，會一路深入。
  2. BFS tree 像 level-order traversal，會一層一層展開。
  3. Back edge：非 tree edge 連到 ancestor / descendant。
  4. Cross edge：非 tree edge 連到彼此沒有 ancestor-descendant 關係的節點。
完整答案：DFS spanning tree is depth-oriented and built by recursive exploration; BFS spanning tree is level-oriented and built by queue expansion. A back edge connects a vertex to its ancestor or descendant in the spanning tree, while a cross edge connects vertices with no ancestor-descendant relationship。
考試容易錯：投影片中特別強調：DFS spanning tree 沒有 cross edges；BFS spanning tree 沒有 back edges。

題目原文

Given the following DFS tree information:
Vertex:  0  1  2  3  4  5  6
Parent: -  0  1  1  3  3  5
dfn:     1  2  3  4  5  6  7
low:     1  1  3  2  5  4  7
Which non-root vertices are articulation points? Use the rule low(w) >= dfn(u).

中文翻譯

根據 DFS tree、dfn 與 low 值，使用 low(w) ≥ dfn(u) 判斷哪些非 root vertex 是 articulation point。

動態分析

解題步驟

  1. 非 root vertex u 是 articulation point，若存在 child w 使 low(w) ≥ dfn(u)。
  2. u=1 有 children 2,3。low(2)=3 ≥ dfn(1)=2，所以 1 是 articulation point。
  3. u=3 有 children 4,5。low(4)=5 ≥ dfn(3)=4，所以 3 是 articulation point。
  4. u=5 有 child 6。low(6)=7 ≥ dfn(5)=6，所以 5 是 articulation point。
完整答案：Articulation points are vertices 1, 3, and 5。

完整 C++ 參考程式碼

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

class ArticulationFinder {
private:
    int n, timer;
    vector<vector<int>> adj;
    vector<int> dfn, low, parent;
    vector<bool> isArt;

    void dfs(int u) {
        dfn[u] = low[u] = ++timer;       // 設定 DFS 拜訪順序與初始 low
        int childCount = 0;              // root 需要用 child 數判斷

        for (int w : adj[u]) {
            if (dfn[w] == 0) {           // tree edge：w 尚未拜訪
                parent[w] = u;
                childCount++;
                dfs(w);
                low[u] = min(low[u], low[w]);

                // 非 root：存在 child w 使 low[w] >= dfn[u]
                if (parent[u] != -1 && low[w] >= dfn[u]) {
                    isArt[u] = true;
                }
            } else if (w != parent[u]) { // back edge：不能把 parent edge 算進去
                low[u] = min(low[u], dfn[w]);
            }
        }

        // root：至少有兩個 DFS children 才是 articulation point
        if (parent[u] == -1 && childCount >= 2) {
            isArt[u] = true;
        }
    }

public:
    ArticulationFinder(int vertices) : n(vertices), timer(0), adj(vertices),
        dfn(vertices, 0), low(vertices, 0), parent(vertices, -1), isArt(vertices, false) {}

    void addEdge(int u, int v) {
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    void findArticulationPoints() {
        for (int i = 0; i < n; i++) {
            if (dfn[i] == 0) dfs(i);
        }
        for (int i = 0; i < n; i++) {
            if (isArt[i]) cout << i << " ";
        }
    }
};

函式說明

dfn[u]
DFS 拜訪 u 的順序。
low[u]
u 或其 descendants 經由最多一條 back edge 可到達的最小 dfn。
low[w] ≥ dfn[u]
代表 w 的子樹無法回到 u 的祖先，所以 u 是關節點。
root case
DFS root 若有兩個以上 children，才是 articulation point。

題目原文

Use Kruskal's algorithm to find an MST of the weighted graph:
Edges: (0,5,10), (2,3,12), (1,6,14), (1,2,16), (3,6,18), (3,4,22), (4,6,24), (4,5,25), (0,1,28)
Show the selected edges and total cost.

中文翻譯

請使用 Kruskal 演算法找出 weighted graph 的 MST，列出被選的邊與總成本。

Kruskal 動態圖解

解題步驟

  1. Kruskal 先將所有邊依 weight 由小到大排序。
  2. 每次選最小邊，但不能形成 cycle。
  3. 選到 |V|−1 條邊就完成。
  4. 需要用 disjoint set / union-find 判斷兩端點是否已在同一集合。
完整答案：Selected edges: (0,5,10), (2,3,12), (1,6,14), (1,2,16), skip (3,6,18), select (3,4,22), skip (4,6,24), select (4,5,25)。Total cost = 10+12+14+16+22+25 = 99。

完整 C++ 參考程式碼

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

struct Edge {
    int u, v, w; // u、v 為端點，w 為權重
};

class DisjointSet {
private:
    vector<int> parent;
public:
    DisjointSet(int n) : parent(n, -1) {}

    // collapsing find：找 root，並做路徑壓縮
    int find(int x) {
        if (parent[x] < 0) return x;
        return parent[x] = find(parent[x]);
    }

    // weighted union：小集合接到大集合下
    bool unite(int a, int b) {
        int ra = find(a), rb = find(b);
        if (ra == rb) return false; // 同集合，加入會形成 cycle
        if (parent[ra] > parent[rb]) swap(ra, rb); // ra 集合較大或相等
        parent[ra] += parent[rb];
        parent[rb] = ra;
        return true;
    }
};

int main() {
    int V = 7;
    vector<Edge> edges = {
        {0,5,10}, {2,3,12}, {1,6,14}, {1,2,16}, {3,6,18},
        {3,4,22}, {4,6,24}, {4,5,25}, {0,1,28}
    };

    sort(edges.begin(), edges.end(), [](const Edge& a, const Edge& b) {
        return a.w < b.w; // 由小到大排序
    });

    DisjointSet ds(V);
    int total = 0, count = 0;

    for (const Edge& e : edges) {
        if (ds.unite(e.u, e.v)) { // 若不形成 cycle，選取此邊
            cout << "Select (" << e.u << "," << e.v << ") weight=" << e.w << endl;
            total += e.w;
            count++;
            if (count == V - 1) break;
        }
    }
    cout << "Total cost = " << total << endl;
    return 0;
}

題目原文

Compare Prim's algorithm and Sollin's algorithm for constructing a minimum spanning tree. Explain how each algorithm grows the MST and give the time complexity of Prim's algorithm when implemented with a priority queue.

中文翻譯

比較 Prim 與 Sollin 演算法如何建立 MST，並說明 Prim 使用 priority queue 時的時間複雜度。

圖解比較

解題步驟

  1. Prim：從單一 vertex 開始，每次加入一條連接 T 與非 T 的最小安全邊。
  2. Sollin：一開始每個 vertex 是一個 component，每一輪每個 component 選一條最便宜的 outgoing edge。
  3. Prim 一次通常長一條邊；Sollin 一輪可以同時加入多條邊並 merge components。
  4. Prim 使用 priority queue 時，常見時間複雜度為 O(E log V)。
完整答案：Prim grows one tree from a chosen start vertex by repeatedly adding the minimum safe edge from the current tree to an outside vertex. Sollin starts with many components and lets each component choose its cheapest outgoing edge, then merges components. Prim with a priority queue runs in O(E log V)。
考試容易錯：Prim 不是先排序所有 edges；Kruskal 才是直接依 edge weight 排序。
