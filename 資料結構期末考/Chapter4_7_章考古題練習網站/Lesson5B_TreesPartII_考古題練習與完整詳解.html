題目原文

A max priority queue supports Top(), Push(e), and Pop(). Explain the purpose of each operation and compare the insertion and deletion time of unordered array, sorted array, unordered linked list, sorted linked list, and heap.

中文翻譯

最大優先佇列支援 Top()、Push(e)、Pop()。請說明每個操作的目的，並比較不同表示法的插入與刪除時間。

核心觀念

priority queueMaxPQheaptime complexity

整理表

RepresentationInsertionDeletion of max說明
Unordered arrayΘ(1)Θ(n)插入容易，刪除最大值要掃描
Sorted arrayΘ(n)Θ(1)插入要維持排序，刪除最大值容易
Unordered linked listΘ(1)Θ(n)插入到任意位置，刪除最大值要找
Sorted linked listΘ(n)Θ(1)插入要找位置，最大值可放在端點
HeapΘ(log n)Θ(log n)插入 bubble up，刪除 trickle down

解題步驟

  1. Top()：回傳目前最大 key，但不刪除。
  2. Push(e)：加入一個新元素。
  3. Pop()：刪除目前最大 key。
  4. Heap 在插入與刪除都維持 Θ(log n)，因此適合 priority queue。
完整答案：Top() returns the maximum key; Push(e) inserts a new element; Pop() deletes the maximum key. Heap is usually the balanced choice because both Push and Pop take Θ(log n), while unordered structures make deletion slow and sorted structures make insertion slow.
老師出題提醒：這種題常出成 True/False 或比較表，尤其會問「heap 為什麼適合 priority queue」。

題目原文

Given the array representation with index starting from 1: A = [ -, 20, 15, 18, 10, 12, 9, 8 ]. Is it a max heap? If not, explain which parent-child relation violates the max heap property.

中文翻譯

給定從 index 1 開始的陣列，判斷是否為 max heap；若不是，指出哪個 parent-child 關係違反規則。

核心觀念

max heapcomplete binary treeparent ≥ child

圖解

解題步驟

  1. Max heap 必須同時是 complete binary tree 與 max tree。
  2. Complete binary tree 的陣列表示中，left child = 2i，right child = 2i+1。
  3. 檢查 parent ≥ children：20 ≥ 15、18；15 ≥ 10、12；18 ≥ 9、8。
  4. 全部符合，所以是 max heap。
完整答案：Yes, it is a max heap. Every parent is greater than or equal to its children.
常見錯誤：Max heap 不是 BST，所以不要求左小右大；只要求 parent 比 children 大。

題目原文

Given a max heap stored as A = [ -, 20, 15, 2, 14, 10, 1 ], show the array after Push(21). Explain the bubble-up process.

中文翻譯

給定 max heap 陣列，執行 Push(21)，請列出 bubble-up 過程與最後陣列。

核心觀念

Pushbubble uparray representation

動態示範

解題步驟

  1. 先把 21 放到 heap 的最後一格 index 7。
  2. 21 與 parent index 3 的 2 比較，21 較大，往上交換。
  3. 21 到 index 3 後，再與 parent index 1 的 20 比較，21 較大，再往上交換。
  4. 21 到 root，停止。

C++ 程式碼與中文註解

template <class T>
void MaxHeap<T>::Push(const T& e) {
    // 如果 heap 陣列已滿，就擴大容量
    if (heapSize == capacity) {
        ChangeSize1D(heap, capacity, 2 * capacity);
        capacity *= 2;
    }

    // 新節點先放在最後一個位置
    int currentNode = ++heapSize;

    // bubble up：只要新元素比 parent 大，就把 parent 往下移
    while (currentNode != 1 && heap[currentNode / 2] < e) {
        heap[currentNode] = heap[currentNode / 2];
        currentNode /= 2;
    }

    // 找到正確位置後放入 e
    heap[currentNode] = e;
}
完整答案：最後陣列為 [ -, 21, 15, 20, 14, 10, 1, 2 ]。
老師出題提醒：這題很像 PPT 第 9 頁 Push(21)，老師常要求畫出每一步 array 與 tree 狀態。

題目原文

Given a max heap stored as A = [ -, 21, 15, 20, 14, 10, 1, 2 ], show the array after Pop(). Explain the trickle-down process.

中文翻譯

給定 max heap，執行 Pop() 刪除最大值，請列出 trickle-down 過程與最後陣列。

核心觀念

Poptrickle downdelete max

動態示範

解題步驟

  1. 刪除 root 21。
  2. 把最後一個元素 2 暫時拿來補 root。
  3. 比較左右 child：15 與 20，選較大的 20 往上。
  4. 2 下沉到 index 3，再與 child 1 比較，2 ≥ 1，停止。

C++ 程式碼與函式說明

template <class T>
void MaxHeap<T>::Pop() {
    if (IsEmpty()) throw "Heap is empty! Cannot delete.";

    // 刪除最大值，也就是 root
    heap[1].~T();

    // 取出最後一個元素，準備拿來重新放回 heap
    T lastE = heap[heapSize--];

    int currentNode = 1; // 從 root 開始往下找位置
    int child = 2;       // currentNode 的左 child

    while (child <= heapSize) {
        // 如果右 child 存在且比左 child 大，就改選右 child
        if (child < heapSize && heap[child] < heap[child + 1])
            child++;

        // 如果 lastE 已經大於等於較大的 child，代表位置正確
        if (lastE >= heap[child]) break;

        // 否則把較大的 child 往上移
        heap[currentNode] = heap[child];
        currentNode = child;
        child *= 2;
    }

    heap[currentNode] = lastE;
}
完整答案：Pop() 後陣列為 [ -, 20, 15, 2, 14, 10, 1 ]。

題目原文

Build a max heap from A = [ -, 4, 1, 3, 2, 16, 9, 10, 14, 8, 7 ] by bottom-up heapify. Then give the final ascending order after heap sort.

中文翻譯

使用 bottom-up heapify 建立 max heap，接著使用 heap sort 排成升冪。

核心觀念

bottom-up heapifyheap sortO(n log n)

動態示範：Build Heap

解題步驟

  1. 把陣列視為 complete binary tree。
  2. 從最後一個非葉節點 i = ⌊n/2⌋ 開始往 root 做 Adjust。
  3. 每次選較大的 child 往上，讓較小的 parent 往下沉。
  4. 建好 max heap 後，每次 swap root 與最後一格，再縮小 heap，重新 Adjust root。
完整答案：Build max heap 後為 [ -, 16, 14, 10, 8, 7, 9, 3, 2, 4, 1 ]。Heap sort 升冪結果為 [ -, 1, 2, 3, 4, 7, 8, 9, 10, 14, 16 ]。
常見錯誤：Build heap 不是從左到右插入；PPT 的方式是 bottom-up heapify，從 n/2 往 1 調整。

題目原文

Explain why a heap is suitable for a priority queue but not suitable for a dictionary. Compare search, min/max, rank, insert, and delete operations between a min/max heap and a binary search tree.

中文翻譯

說明為什麼 heap 適合 priority queue，但不適合 dictionary，並比較 heap 與 BST 的常見操作時間。

核心觀念

dictionaryheapBSTO(h)

OperationMin/Max HeapBinary Search Tree
SearchO(n)O(h)
Min/MaxO(1)O(h)
RankO(n)O(h) 若有 leftSize
InsertO(h)O(h)
DeleteRoot: O(h), arbitrary: O(n)O(h)

解題步驟

  1. Heap 只保證 parent 與 children 的大小關係。
  2. Heap 不保證左子樹全部小於 root、右子樹全部大於 root。
  3. 因此 heap 找任意 key 可能要掃全部節點。
  4. BST 因為有 left smaller / right larger，所以適合 search、rank、dictionary operations。
完整答案：Heap is good for priority queues because Top(), Push(), and Pop() are efficient. However, it is not good for dictionaries because searching an arbitrary key is O(n). A BST supports dictionary operations in O(h), provided the height h is well controlled.

題目原文

Given a BST with keys 7, 6, 8, 10, 14, 12. Trace the search path for key 9, then insert 9 into the BST.

中文翻譯

給定 BST，先追蹤搜尋 9 的路徑，再將 9 插入 BST。

核心觀念

BST searchBST insertpath tracing

動態示範

解題步驟

  1. 9 > 7，所以往右到 8。
  2. 9 > 8，所以往右到 10。
  3. 9 < 10，所以往左；左子樹為空。
  4. 因此 9 插入為 10 的 left child。

C++ 插入程式與中文註解

template <class K, class E>
void BST<K, E>::Insert(const pair<K, E>& thePair) {
    TreeNode<pair<K, E>> *p = root;
    TreeNode<pair<K, E>> *pp = 0; // pp 記錄 p 的 parent

    // 先用搜尋方式找到要插入的位置
    while (p) {
        pp = p;
        if (thePair.key < p->data.key)
            p = p->leftChild;
        else if (thePair.key > p->data.key)
            p = p->rightChild;
        else {
            // key 重複時，更新 element
            p->data.element = thePair.element;
            return;
        }
    }

    // 建立新節點
    p = new TreeNode<pair<K, E>>(thePair);

    // 根據 key 大小接到 parent 的左邊或右邊
    if (root) {
        if (thePair.key < pp->data.key) pp->leftChild = p;
        else pp->rightChild = p;
    } else {
        root = p;
    }
}
完整答案：搜尋路徑為 7 → 8 → 10 → null。插入後，9 成為 10 的 left child。

題目原文

A BST node stores an extra field leftSize = 1 + the number of nodes in its left subtree. Given the following tree, find the 4th smallest key. Show the rank-search process.

中文翻譯

BST 每個節點多存 leftSize。請找第 4 小的 key，並寫出 rank search 過程。

leftSizerankorder statistic tree

解題步驟

題目原文

Given a forest F consisting of three trees: T1 has root A with children B, C, D; T2 has root E with child F; T3 has root G with children H, I. Convert the forest to its corresponding binary tree and write the forest preorder traversal.

中文翻譯

給定三棵樹組成的 forest，請轉成對應 binary tree，並寫出 forest preorder。

核心觀念

forestdegree-two treeleft child right siblingpreorder

動態示範

解題步驟

  1. Forest 的第一棵樹 root A 成為 binary tree 的 root。
  2. A 的第一個 child B 成為 A 的 left child。
  3. B、C、D 是 siblings，所以用 right child 串成 B → C → D。
  4. T2 的 root E 是 T1 的 sibling，所以接在 A 的 right child 路徑上，也就是 D 的 right child。
  5. Forest preorder：先 visit 第一棵 root，再其 subtrees，再 remaining trees。
完整答案：Forest preorder = A B C D E F G H I。轉成 binary tree 時，left child 表示第一個孩子，right child 表示下一個 sibling。

題目原文

Eight sorted runs have current first keys: 10, 9, 20, 6, 8, 9, 90, 17. Build the first level and root winner of a winner tree. Which run wins? Explain why the total merge time is O(n log k).

中文翻譯

8 個 sorted runs 的第一個 key 分別為 10, 9, 20, 6, 8, 9, 90, 17。請建立 winner tree 的比較結果，指出哪個 run 勝出，並解釋合併時間。

核心觀念

selection treewinner treek-way merge

圖解

解題步驟

  1. 兩兩比較：min(10,9)=9；min(20,6)=6；min(8,9)=8；min(90,17)=17。
  2. 第二層比較：min(9,6)=6；min(8,17)=8。
  3. Root 比較：min(6,8)=6。
  4. 所以 run4 的 key 6 是整體 winner。
  5. 每取出一個元素，只需沿著該 leaf 到 root 更新，約 log k 次比較；總共 n 個元素，所以 O(n log k)。
完整答案：Winner 是 run4，key = 6。Total merging time is O(n log k)。
老師出題提醒：Selection tree 常考「下一個 run 更新後，winner tree 狀態怎麼變」。要會手動畫 tournament。

題目原文

Initially parent[i] = -1 for i = 0,...,7. Perform WeightedUnion(0,1), WeightedUnion(2,3), WeightedUnion(4,5), WeightedUnion(6,7), WeightedUnion(0,2), WeightedUnion(4,6), WeightedUnion(0,4). Show the final parent array.

中文翻譯

一開始每個元素都是獨立集合，parent[i] = -1。依序執行 WeightedUnion，請列出最後 parent array。

核心觀念

disjoint setweighted unionparent array

動態示範

解題步驟

  1. parent[root] 存負數，代表該 set 的節點數。
  2. WeightedUnion 把節點數較少的樹接到較多的樹底下。
  3. 節點數相同時，依程式 else，把 j 接到 i。
  4. 照順序合併後，0 成為代表 root，parent[0] = -8。

C++ WeightedUnion 程式碼

void Sets::WeightedUnion(int i, int j) {
    // parent[i] 與 parent[j] 是負數，代表集合大小
    // 例如 parent[i] = -4 表示 root i 的集合有 4 個節點
    int temp = parent[i] + parent[j];

    if (parent[i] > parent[j]) {
        // i 的負數比較大，代表 i 的集合比較小
        // 所以把 i 接到 j 底下
        parent[i] = j;
        parent[j] = temp;
    } else {
        // j 比較小，或兩者一樣大
        // 把 j 接到 i 底下
        parent[j] = i;
        parent[i] = temp;
    }
}
完整答案：最後 parent array = [-8, 0, 0, 2, 0, 4, 4, 6]。
常見錯誤：root 的 parent 是負數，不是 -1 永遠不變；合併後要更新為總節點數的負值。

題目原文

Given parent = [-8, 0, 0, 2, 0, 4, 4, 6], perform CollapsingFind(7). Show the returned root and the updated parent array.

中文翻譯

給定 parent array，執行 CollapsingFind(7)，請列出回傳 root 與更新後 parent array。

核心觀念

collapsing findpath compressionroot

動態示範

解題步驟

  1. 從 7 開始找 root：7 → 6 → 4 → 0。
  2. parent[0] = -8，所以 root 是 0。
  3. Collapsing rule 把路徑上的節點直接接到 root。
  4. 因此 parent[7]、parent[6]、parent[4] 都應直接變成 0；其中 parent[4] 原本已是 0。

C++ CollapsingFind 程式碼

int Sets::CollapsingFind(int i) {
    // 第一段：沿著 parent 找到 root
    int r;
    for (r = i; parent[r] >= 0; r = parent[r]) {
        // 空迴圈：只負責往上找 root
    }

    // 第二段：把 i 到 root 路徑上的所有節點直接接到 root
    while (i != r) {
        int s = parent[i]; // 先記住原本 parent
        parent[i] = r;     // 將 i 直接接到 root
        i = s;             // 移動到下一個節點
    }

    return r;
}
完整答案：CollapsingFind(7) 回傳 root = 0。更新後 parent = [-8, 0, 0, 2, 0, 4, 0, 0]。

題目原文

In a B*-tree floorplan representation, explain the meaning of the left child and the right child of a node. Given block 1 at coordinate (x1, y1) with width w1, what is the x-coordinate of its left child and right child?

中文翻譯

在 B*-tree floorplanning 表示法中，說明 left child 與 right child 的意義。若 block 1 的座標為 (x1, y1)，寬度為 w1，請寫出其 left child 與 right child 的 x 座標。

核心觀念

B*-treefloorplanningleft childright childcontour

圖解

解題步驟

  1. B*-tree 用 ordered binary tree 表示 non-slicing floorplan。
  2. Left child 是右邊相鄰且最低的 block，所以 xj = xi + wi。
  3. Right child 是在上方且相同 x-coordinate 的第一個 block，所以 xj = xi。
  4. y-coordinate 通常用 horizontal contour 計算。
完整答案：Left child represents the lowest adjacent block on the right, so xj = xi + wi. Right child represents the first block above with the same x-coordinate, so xj = xi.
老師出題提醒：這一段偏進階，若考試出現，多半是觀念題或圖形判斷題，不太會要求完整實作 floorplanner。
