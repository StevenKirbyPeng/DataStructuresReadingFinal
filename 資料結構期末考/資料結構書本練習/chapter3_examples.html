  14. 如果程式使用佇列，重點是 FIFO：最早放入的資料最早取出。這種特性常用於 BFS、排程、等待線與層序走訪。trace 時要同時記錄 front 與 rear，避免混淆佇列頭尾。

  16. 如果程式使用圖，重點是 visited 陣列與相鄰節點。visited 可以避免重複拜訪或無限循環；相鄰表示法會影響時間複雜度。相鄰矩陣適合密集圖，相鄰串列適合稀疏圖。
  17. 本程式的時間複雜度可以從「主要迴圈執行幾次」估計。如果只有單層掃描通常是 O(n)；雙層迴圈常是 O(n²)；每次把資料範圍折半常是 O(log n)；樹高控制得好時常接近 O(log n)。

#include <stdlib.h>
#define MAX 20
using namespace std;  // 宣告變數，用來保存演算法執行過程中的資料。

class In_to_post {  // 定義資料結構類別，集中管理資料成員與操作函式。

{

    cout << "Postfix expression: ";  // 輸出結果，將資料顯示或寫入檔案。

    for (ctr = 0; ctr <= rear; ctr++) {  // for 迴圈，用固定範圍重複走訪資料。

class Stack {  // 定義資料結構類別，集中管理資料成員與操作函式。

    Stack();

    if (top >= MAX-1)   // 當堆疊已滿，則顯示錯誤  // 條件判斷，依照目前資料狀態選擇不同處理路徑。

        cin.getline(item[top], 21);  // 讀入資料，可能來自鍵盤或檔案。

}

    if (top < 0)  // 當堆疊沒有資料存在，則顯示錯誤  // 條件判斷，依照目前資料狀態選擇不同處理路徑。

    else {  // 否則分支，處理上一個條件不成立的情況。
        cout << "\n Item " << item[top] << " deleted\n";  // 釋放動態記憶體，避免記憶體洩漏。

    }

}
