  7. 程式中的函式分工通常可以分成三類：第一類負責初始化資料，第二類負責執行核心演算法，第三類負責輸出或檢查結果。把函式依角色分類後，整支程式就不會看起來像一大段難懂的程式碼。
  8. 變數命名通常反映它在演算法中的任務。例如 size、n、capacity 多半代表資料數量或容量；top、front、rear、current 多半代表目前位置；temp、p、q、prev 多半代表暫存或指標移動。
  9. 核心迴圈是本程式最重要的地方。它通常會反覆檢查條件、更新索引、比較資料或搬移資料。理解迴圈時，要先問三件事：迴圈從哪裡開始、什麼時候停止、每次執行後哪些變數會改變。
  10. 判斷式 if / else 是演算法決策點。它通常負責處理特殊情況，例如資料為空、找到目標、索引越界、需要交換、需要遞迴、需要往左或往右走。考試常會要求你說明某個 if 條件為什麼必要。
  11. 如果程式使用遞迴，必須先找終止條件。終止條件通常對應最簡單情況，例如資料為空、左右邊界交錯、節點為 NULL 或只剩一個元素。沒有正確終止條件就會造成無窮遞迴。
  12. 如果程式使用排序或搜尋，必須注意資料是否需要事先有序。像二元搜尋必須建立在排序後的資料上；若資料未排序，即使程式語法正確，結果仍可能錯誤。這也是資料結構題目常見陷阱。
  13. 如果程式使用堆疊，重點是 LIFO：最後放入的資料最先取出。這種特性常用於括號配對、函式呼叫、DFS、回溯與運算式轉換。trace 時可以畫出 top 從哪裡移動到哪裡。
  14. 如果程式使用佇列，重點是 FIFO：最早放入的資料最早取出。這種特性常用於 BFS、排程、等待線與層序走訪。trace 時要同時記錄 front 與 rear，避免混淆佇列頭尾。
  15. 如果程式使用樹，必須注意每個節點與子節點的關係。二元搜尋樹依照 key 大小決定往左或往右；heap 則依照父子大小維持優先權；一般二元樹走訪則依照 LVR、VLR、LRV 決定輸出順序。
  16. 如果程式使用圖，重點是 visited 陣列與相鄰節點。visited 可以避免重複拜訪或無限循環；相鄰表示法會影響時間複雜度。相鄰矩陣適合密集圖，相鄰串列適合稀疏圖。
  17. 本程式的時間複雜度可以從「主要迴圈執行幾次」估計。如果只有單層掃描通常是 O(n)；雙層迴圈常是 O(n²)；每次把資料範圍折半常是 O(log n)；樹高控制得好時常接近 O(log n)。
  18. 空間複雜度要看是否建立額外資料結構。若只使用少數變數，通常是 O(1)；若建立與輸入大小相同的陣列、堆疊、佇列或 visited，通常是 O(n)；二維矩陣常是 O(mn)。
  19. 記憶體配置是本程式的重要觀察點。靜態陣列大小固定但簡單；動態陣列可依需求配置但要注意 delete；鏈結串列彈性高，但每個節點都需要額外指標空間。

  21. 常見錯誤包含：陣列索引越界、指標沒有初始化、刪除節點後仍使用原指標、遞迴沒有終止條件、忘記更新 front/rear/top、比較條件寫反、或是沒有處理空資料結構。

  9. 核心迴圈是本程式最重要的地方。它通常會反覆檢查條件、更新索引、比較資料或搬移資料。理解迴圈時，要先問三件事：迴圈從哪裡開始、什麼時候停止、每次執行後哪些變數會改變。
  10. 判斷式 if / else 是演算法決策點。它通常負責處理特殊情況，例如資料為空、找到目標、索引越界、需要交換、需要遞迴、需要往左或往右走。考試常會要求你說明某個 if 條件為什麼必要。
  11. 如果程式使用遞迴，必須先找終止條件。終止條件通常對應最簡單情況，例如資料為空、左右邊界交錯、節點為 NULL 或只剩一個元素。沒有正確終止條件就會造成無窮遞迴。
  12. 如果程式使用排序或搜尋，必須注意資料是否需要事先有序。像二元搜尋必須建立在排序後的資料上；若資料未排序，即使程式語法正確，結果仍可能錯誤。這也是資料結構題目常見陷阱。
  13. 如果程式使用堆疊，重點是 LIFO：最後放入的資料最先取出。這種特性常用於括號配對、函式呼叫、DFS、回溯與運算式轉換。trace 時可以畫出 top 從哪裡移動到哪裡。
  14. 如果程式使用佇列，重點是 FIFO：最早放入的資料最早取出。這種特性常用於 BFS、排程、等待線與層序走訪。trace 時要同時記錄 front 與 rear，避免混淆佇列頭尾。
  15. 如果程式使用樹，必須注意每個節點與子節點的關係。二元搜尋樹依照 key 大小決定往左或往右；heap 則依照父子大小維持優先權；一般二元樹走訪則依照 LVR、VLR、LRV 決定輸出順序。
  16. 如果程式使用圖，重點是 visited 陣列與相鄰節點。visited 可以避免重複拜訪或無限循環；相鄰表示法會影響時間複雜度。相鄰矩陣適合密集圖，相鄰串列適合稀疏圖。
  17. 本程式的時間複雜度可以從「主要迴圈執行幾次」估計。如果只有單層掃描通常是 O(n)；雙層迴圈常是 O(n²)；每次把資料範圍折半常是 O(log n)；樹高控制得好時常接近 O(log n)。
  18. 空間複雜度要看是否建立額外資料結構。若只使用少數變數，通常是 O(1)；若建立與輸入大小相同的陣列、堆疊、佇列或 visited，通常是 O(n)；二維矩陣常是 O(mn)。

  17. 本程式的時間複雜度可以從「主要迴圈執行幾次」估計。如果只有單層掃描通常是 O(n)；雙層迴圈常是 O(n²)；每次把資料範圍折半常是 O(log n)；樹高控制得好時常接近 O(log n)。
  18. 空間複雜度要看是否建立額外資料結構。若只使用少數變數，通常是 O(1)；若建立與輸入大小相同的陣列、堆疊、佇列或 visited，通常是 O(n)；二維矩陣常是 O(mn)。
  19. 記憶體配置是本程式的重要觀察點。靜態陣列大小固定但簡單；動態陣列可依需求配置但要注意 delete；鏈結串列彈性高，但每個節點都需要額外指標空間。

  18. 空間複雜度要看是否建立額外資料結構。若只使用少數變數，通常是 O(1)；若建立與輸入大小相同的陣列、堆疊、佇列或 visited，通常是 O(n)；二維矩陣常是 O(mn)。
  19. 記憶體配置是本程式的重要觀察點。靜態陣列大小固定但簡單；動態陣列可依需求配置但要注意 delete；鏈結串列彈性高，但每個節點都需要額外指標空間。
  20. 程式執行結果要和演算法目的互相對照。輸出不是只看有沒有跑出東西，而是要確認每一筆結果是否符合資料結構規則。例如排序結果是否遞增、搜尋位置是否正確、樹走訪順序是否合理。
  21. 常見錯誤包含：陣列索引越界、指標沒有初始化、刪除節點後仍使用原指標、遞迴沒有終止條件、忘記更新 front/rear/top、比較條件寫反、或是沒有處理空資料結構。

  23. 面試或考試常問這個程式的優缺點。優點通常是觀念清楚、容易實作、適合作為基本資料結構練習；缺點則可能是效率不一定最佳、記憶體使用較多、或遇到極端輸入時需要額外處理。

            if (adjlist[vj]->count == 0)  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
                Push(&Stack1, vj);  // 呼叫資料結構或演算法操作，改變元素位置或集合內容。

            ptr = ptr->link;  // 指定或更新變數值，改變目前資料狀態。
        }

    }

        adjlist[vi]->vertex = vi;  // 指定或更新變數值，改變目前資料狀態。
        adjlist[vi]->link = NULL;  // 指定或更新變數值，改變目前資料狀態。
    }
    /*讀取節點資料*/
    while (!fin.eof()) {  // while 迴圈，只要條件成立就持續執行。
	    fin >> vi >> vj;	  // 宣告變數，用來保存演算法執行過程中的資料。
        node = (Node *)new Node;  // 動態配置記憶體，建立新的陣列或節點。
        node->vertex = vj;  // 指定或更新變數值，改變目前資料狀態。
        node->link = NULL;  // 指定或更新變數值，改變目前資料狀態。
        if (adjlist[vi]->link == NULL)  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
            adjlist[vi]->link = node;  // 指定或更新變數值，改變目前資料狀態。
        else {  // 否則分支，處理上一個條件不成立的情況。
            lastnode = searchlast(adjlist[vi]);  // 指定或更新變數值，改變目前資料狀態。
            lastnode->link = node;  // 指定或更新變數值，改變目前資料狀態。
        }
    }
    fin.close();
}

void TopologicSort::topSort(int k)  // 函式宣告或函式開始，負責執行一段可重複使用的演算法步驟。
{
    Node *ptr;
    int w;  // 宣告變數，用來保存演算法執行過程中的資料。
    
    visited[k] = TRUE;          /*設定v頂點為已拜訪過*/  // 指定或更新變數值，改變目前資料狀態。
    ptr = adjlist[k]->link;     /*拜訪v相鄰頂點*/  // 指定或更新變數值，改變目前資料狀態。
    while (ptr != NULL) {  // while 迴圈，只要條件成立就持續執行。
        w = ptr->vertex;        /* w 為v的立即後繼者 */   // 指定或更新變數值，改變目前資料狀態。
        if (!visited[w])  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
            topSort(w);
        ptr = ptr->link;  // 指定或更新變數值，改變目前資料狀態。
    }
    TopOrder[--place] = k;  // 指定或更新變數值，改變目前資料狀態。
}
