0. 程式原理清楚說明（約一千字）

  1. 本程式 binarySearchTree.cpp 的主要目的，是把「CHAPTER 7 二元搜尋樹 範例程式完整教學」中的核心觀念轉成可以執行、可以追蹤、可以驗證的 C++ 範例。閱讀時不要只記語法，而要先看它想解決什麼問題，以及資料在程式中如何被建立、保存、更新與輸出。
  2. 這個範例的核心資料結構可理解為：搜尋、索引範圍、比較策略與成功/失敗條件。程式會先準備必要的陣列、節點、索引、指標或輔助變數，接著透過主要函式把資料一步一步處理成最後結果。
  3. 開始閱讀程式時，建議先找到 main 函式，因為 main 通常是整支程式的入口。main 會負責建立測試資料、呼叫核心函式、接收回傳結果，最後把結果印出來，因此它可以視為整個流程圖的最上層。
  4. 若程式中有 class 或 struct，代表作者把資料與操作包裝在一起。資料成員負責記錄狀態，成員函式負責改變狀態。這種設計符合資料抽象與封裝觀念，也方便之後擴充或重複使用。
  5. 若程式使用陣列，重點要放在索引範圍。陣列通常從 0 或 1 開始，迴圈條件若寫錯，很容易造成少處理一格或超出邊界。因此 trace 時要特別記錄每一次 i、j、k 或 current 的變化。
  6. 若程式使用指標或 linked node，重點要放在 link 的方向。新增節點時通常要先配置新節點，再設定資料欄位，最後接上前後節點；刪除節點時則要先保留要刪除的位置，再修改連結，最後釋放記憶體。
  7. 程式中的函式分工通常可以分成三類：第一類負責初始化資料，第二類負責執行核心演算法，第三類負責輸出或檢查結果。把函式依角色分類後，整支程式就不會看起來像一大段難懂的程式碼。
  8. 變數命名通常反映它在演算法中的任務。例如 size、n、capacity 多半代表資料數量或容量；top、front、rear、current 多半代表目前位置；temp、p、q、prev 多半代表暫存或指標移動。
  9. 核心迴圈是本程式最重要的地方。它通常會反覆檢查條件、更新索引、比較資料或搬移資料。理解迴圈時，要先問三件事：迴圈從哪裡開始、什麼時候停止、每次執行後哪些變數會改變。
  10. 判斷式 if / else 是演算法決策點。它通常負責處理特殊情況，例如資料為空、找到目標、索引越界、需要交換、需要遞迴、需要往左或往右走。考試常會要求你說明某個 if 條件為什麼必要。
  11. 如果程式使用遞迴，必須先找終止條件。終止條件通常對應最簡單情況，例如資料為空、左右邊界交錯、節點為 NULL 或只剩一個元素。沒有正確終止條件就會造成無窮遞迴。
  12. 如果程式使用排序或搜尋，必須注意資料是否需要事先有序。像二元搜尋必須建立在排序後的資料上；若資料未排序，即使程式語法正確，結果仍可能錯誤。這也是資料結構題目常見陷阱。
  13. 如果程式使用堆疊，重點是 LIFO：最後放入的資料最先取出。這種特性常用於括號配對、函式呼叫、DFS、回溯與運算式轉換。trace 時可以畫出 top 從哪裡移動到哪裡。
  14. 如果程式使用佇列，重點是 FIFO：最早放入的資料最早取出。這種特性常用於 BFS、排程、等待線與層序走訪。trace 時要同時記錄 front 與 rear，避免混淆佇列頭尾。
  15. 如果程式使用樹，必須注意每個節點與子節點的關係。二元搜尋樹依照 key 大小決定往左或往右；heap 則依照父子大小維持優先權；一般二元樹走訪則依照 LVR、VLR、LRV 決定輸出順序。
  16. 如果程式使用圖，重點是 visited 陣列與相鄰節點。visited 可以避免重複拜訪或無限循環；相鄰表示法會影響時間複雜度。相鄰矩陣適合密集圖，相鄰串列適合稀疏圖。
  17. 本程式的時間複雜度可以從「主要迴圈執行幾次」估計。如果只有單層掃描通常是 O(n)；雙層迴圈常是 O(n²)；每次把資料範圍折半常是 O(log n)；樹高控制得好時常接近 O(log n)。
  18. 空間複雜度要看是否建立額外資料結構。若只使用少數變數，通常是 O(1)；若建立與輸入大小相同的陣列、堆疊、佇列或 visited，通常是 O(n)；二維矩陣常是 O(mn)。
  19. 記憶體配置是本程式的重要觀察點。靜態陣列大小固定但簡單；動態陣列可依需求配置但要注意 delete；鏈結串列彈性高，但每個節點都需要額外指標空間。
  20. 程式執行結果要和演算法目的互相對照。輸出不是只看有沒有跑出東西，而是要確認每一筆結果是否符合資料結構規則。例如排序結果是否遞增、搜尋位置是否正確、樹走訪順序是否合理。
  21. 常見錯誤包含：陣列索引越界、指標沒有初始化、刪除節點後仍使用原指標、遞迴沒有終止條件、忘記更新 front/rear/top、比較條件寫反、或是沒有處理空資料結構。
  22. 老師出 trace code 題目時，通常會要求你寫出每一步變數變化。建議用表格記錄迴圈次數、目前索引、比較對象、資料結構內容與輸出結果，這樣最不容易漏掉中間狀態。
  23. 面試或考試常問這個程式的優缺點。優點通常是觀念清楚、容易實作、適合作為基本資料結構練習；缺點則可能是效率不一定最佳、記憶體使用較多、或遇到極端輸入時需要額外處理。
  24. 實際應用上，這類程式可延伸到資料整理、搜尋系統、排程管理、模擬問題、路徑探索、資料索引與演算法效能比較。資料結構的重點不是背程式，而是知道什麼情境該使用什麼結構。

// 處理二元搜尋樹，將新增資料加入至二元搜尋樹中
void Bintree::access(int id, char name[], int score)  // 函式宣告或函式開始，負責執行一段可重複使用的演算法步驟。
{
    Node_type *node, *prev;
    if (search(id) != NULL) {  // 資料已存在則顯示錯誤  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
        cout << "Student id: " << id << " has existed!\n";  // 輸出結果，將資料顯示或寫入檔案。
        return;  // 回傳函式結果，結束目前函式。
    }
    ptr = new Node_type;  // 動態配置記憶體，建立新的陣列或節點。
    ptr->id = id;  // 指定或更新變數值，改變目前資料狀態。
    strcpy(ptr->name, name);
    ptr->score = score;  // 指定或更新變數值，改變目前資料狀態。
    ptr->llink = ptr->rlink = NULL;  // 指定或更新變數值，改變目前資料狀態。
    if(root == NULL)  // 當根節點為 NULL 的狀況  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
        root = ptr;  // 指定或更新變數值，改變目前資料狀態。
    else {            // 當根節點不為NULL的狀況  // 否則分支，處理上一個條件不成立的情況。
        node = root;  // 指定或更新變數值，改變目前資料狀態。
        while (node != NULL) {  // 搜尋資料插入點  // while 迴圈，只要條件成立就持續執行。
            prev = node;  // 指定或更新變數值，改變目前資料狀態。
            if (ptr->id < node->id)  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
                node = node->llink;  // 指定或更新變數值，改變目前資料狀態。
            else  // 否則分支，處理上一個條件不成立的情況。
                node = node->rlink;  // 指定或更新變數值，改變目前資料狀態。
        }
        if (ptr->id < prev->id)  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
            prev->llink = ptr;  // 指定或更新變數值，改變目前資料狀態。
        else  // 否則分支，處理上一個條件不成立的情況。
            prev->rlink = ptr;  // 指定或更新變數值，改變目前資料狀態。
    }
}

// 將資料從二元搜尋樹中移除
void Bintree::removing(int del_id)  // 函式宣告或函式開始，負責執行一段可重複使用的演算法步驟。
{
    Node_type *del_node;
    
    if ((del_node = search(del_id)) == NULL) { // 找不到資料則顯示錯誤  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
        cout << "Student id: " << del_id << " not found!\n";  // 輸出結果，將資料顯示或寫入檔案。
        return;  // 回傳函式結果，結束目前函式。
    }
    // 節點不為樹葉節點的狀況
    if (del_node->llink != NULL || del_node->rlink != NULL)  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
        del_node = replace(del_node);  // 指定或更新變數值，改變目前資料狀態。
    else if (del_node == root)     // 節點為樹葉節點的狀況  // 否則分支，處理上一個條件不成立的情況。
        root = NULL;  // 指定或更新變數值，改變目前資料狀態。
    else  // 否則分支，處理上一個條件不成立的情況。
        connect(del_node, 'n');
    delete del_node;  // 釋放記憶體  // 釋放動態記憶體，避免記憶體洩漏。
    cout << "student id: " << del_id << " deleted!\n";  // 釋放動態記憶體，避免記憶體洩漏。
}

// 尋找刪除非樹葉節點的替代節點
Node_type *Bintree::replace(Node_type *node)
{
    Node_type *re_node;
    // 當右子樹找不到替代節點，會搜尋左子樹是否存在替代節點
    if ((re_node = search_re_r(node->rlink)) == NULL)  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
        re_node = search_re_L(node->llink);  // 指定或更新變數值，改變目前資料狀態。
    if (re_node->rlink != NULL)  // 當替代節點有右子樹存在的狀況  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
        connect(re_node, 'r');
    else if (re_node->llink != NULL)  // 當替代節點有左子樹存在的狀況  // 否則分支，處理上一個條件不成立的情況。
        connect(re_node, 'L');
    else  // 當替代節點為樹葉節點的狀況  // 否則分支，處理上一個條件不成立的情況。
        connect(re_node, 'n');
    node->id = re_node->id;  // 指定或更新變數值，改變目前資料狀態。
    strcpy(node->name, re_node->name);
    node->score = re_node->score;  // 指定或更新變數值，改變目前資料狀態。
    return re_node;  // 回傳函式結果，結束目前函式。
}

// 調整二元搜尋樹的鏈結，link為r表示處理右鏈結，為l表處理左鏈結，
// 為m則將鏈結指向NULL
void Bintree::connect(Node_type *node, char link)  // 函式宣告或函式開始，負責執行一段可重複使用的演算法步驟。
{
    Node_type *parent;
    parent = search_p(node);     // 搜尋父節點  // 指定或更新變數值，改變目前資料狀態。
    if (node->id < parent->id)   // 取代的節點在父節點的左側   // 條件判斷，依照目前資料狀態選擇不同處理路徑。
        if (link == 'r')         // link為r  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
            parent->llink = node->rlink;  // 指定或更新變數值，改變目前資料狀態。
        else if (link == 'L')      // 函式宣告或函式開始，負責執行一段可重複使用的演算法步驟。
		// link為 l
            parent->llink = node->llink;  // 指定或更新變數值，改變目前資料狀態。
        else  // link為 n  // 否則分支，處理上一個條件不成立的情況。
            parent->llink = NULL;  // 指定或更新變數值，改變目前資料狀態。
    else  // 取代節點在父節點的右側   // 否則分支，處理上一個條件不成立的情況。
        if (link == 'r')  // link為 r  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
            parent->rlink = node->rlink;  // 指定或更新變數值，改變目前資料狀態。
        else  // 否則分支，處理上一個條件不成立的情況。
            if (link == 'L')  // link為 l  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
                parent->rlink = node->llink;  // 指定或更新變數值，改變目前資料狀態。
            else  // link為 n  // 否則分支，處理上一個條件不成立的情況。
                parent->rlink = NULL;  // 指定或更新變數值，改變目前資料狀態。
}

// 搜尋target所在節點
Node_type *Bintree::search(int target_id)
{
    Node_type *node;
    node = root;  // 指定或更新變數值，改變目前資料狀態。
    while (node != NULL) {  // while 迴圈，只要條件成立就持續執行。
        if (target_id == node->id)  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
            return node;  // 回傳函式結果，結束目前函式。
        else  // 否則分支，處理上一個條件不成立的情況。
            if (target_id < node->id)  // target_id 小於目前節點，往左搜尋  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
                node = node->llink;  // 指定或更新變數值，改變目前資料狀態。
            else  // target_id大於目前節點，往右搜尋  // 否則分支，處理上一個條件不成立的情況。
                node = node->rlink;  // 指定或更新變數值，改變目前資料狀態。
    }
    return node;  // 回傳函式結果，結束目前函式。
}
