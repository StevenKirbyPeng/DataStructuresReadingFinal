0. 程式原理清楚說明（約一千字）

  1. 本程式 heapTree.cpp 的主要目的，是把「CHAPTER 8 堆積 範例程式完整教學」中的核心觀念轉成可以執行、可以追蹤、可以驗證的 C++ 範例。閱讀時不要只記語法，而要先看它想解決什麼問題，以及資料在程式中如何被建立、保存、更新與輸出。
  2. 這個範例的核心資料結構可理解為：堆積、完全二元樹、上浮下沉與優先佇列。程式會先準備必要的陣列、節點、索引、指標或輔助變數，接著透過主要函式把資料一步一步處理成最後結果。
  3. 開始閱讀程式時，建議先找到 main 函式，因為 main 通常是整支程式的入口。main 會負責建立測試資料、呼叫核心函式、接收回傳結果，最後把結果印出來，因此它可以視為整個流程圖的最上層。
  4. 若程式中有 class 或 struct，代表作者把資料與操作包裝在一起。資料成員負責記錄狀態，成員函式負責改變狀態。這種設計符合資料抽象與封裝觀念，也方便之後擴充或重複使用。
  5. 若程式使用陣列，重點要放在索引範圍。陣列通常從 0 或 1 開始，迴圈條件若寫錯，很容易造成少處理一格或超出邊界。因此 trace 時要特別記錄每一次 i、j、k 或 current 的變化。
  6. 若程式使用指標或 linked node，重點要放在 link 的方向。新增節點時通常要先配置新節點，再設定資料欄位，最後接上前後節點；刪除節點時則要先保留要刪除的位置，再修改連結，最後釋放記憶體。
  7. 程式中的函式分工通常可以分成三類：第一類負責初始化資料，第二類負責執行核心演算法，第三類負責輸出或檢查結果。把函式依角色分類後，整支程式就不會看起來像一大段難懂的程式碼。
  8. 變數命名通常反映它在演算法中的任務。例如 size、n、capacity 多半代表資料數量或容量；top、front、rear、current 多半代表目前位置；temp、p、q、prev 多半代表暫存或指標移動。
  9. 核心迴圈是本程式最重要的地方。它通常會反覆檢查條件、更新索引、比較資料或搬移資料。理解迴圈時，要先問三件事：迴圈從哪裡開始、什麼時候停止、每次執行後哪些變數會改變。
  10. 判斷式 if / else 是演算法決策點。它通常負責處理特殊情況，例如資料為空、找到目標、索引越界、需要交換、需要遞迴、需要往左或往右走。考試常會要求你說明某個 if 條件為什麼必要。
  11. 如果程式使用遞迴，必須先找終止條件。終止條件通常對應最簡單情況，例如資料為空、左右邊界交錯、節點為 NULL 或只剩一個元素。沒有正確終止條件就會造成無窮遞迴。
  12. 如果程式使用排序或搜尋，必須注意資料是否需要事先有序。像二元搜尋必須建立在排序後的資料上；若資料未排序，即使程式語法正確，結果仍可能錯誤。這也是資料結構題目常見陷阱。
  13. 如果程式使用堆疊，重點是 LIFO：最後放入的資料最先取出。這種特性常用於括號配對、函式呼叫、DFS、回溯與運算式轉換。trace 時可以畫出 top 從哪裡移動到哪裡。
  14. 如果程式使用佇列，重點是 FIFO：最早放入的資料最早取出。這種特性常用於 BFS、排程、等待線與層序走訪。trace 時要同時記錄 front 與 rear，避免混淆佇列頭尾。
  15. 如果程式使用樹，必須注意每個節點與子節點的關係。二元搜尋樹依照 key 大小決定往左或往右；heap 則依照父子大小維持優先權；一般二元樹走訪則依照 LVR、VLR、LRV 決定輸出順序。
  16. 如果程式使用圖，重點是 visited 陣列與相鄰節點。visited 可以避免重複拜訪或無限循環；相鄰表示法會影響時間複雜度。相鄰矩陣適合密集圖，相鄰串列適合稀疏圖。
  17. 本程式的時間複雜度可以從「主要迴圈執行幾次」估計。如果只有單層掃描通常是 O(n)；雙層迴圈常是 O(n²)；每次把資料範圍折半常是 O(log n)；樹高控制得好時常接近 O(log n)。
  18. 空間複雜度要看是否建立額外資料結構。若只使用少數變數，通常是 O(1)；若建立與輸入大小相同的陣列、堆疊、佇列或 visited，通常是 O(n)；二維矩陣常是 O(mn)。
  19. 記憶體配置是本程式的重要觀察點。靜態陣列大小固定但簡單；動態陣列可依需求配置但要注意 delete；鏈結串列彈性高，但每個節點都需要額外指標空間。
  20. 程式執行結果要和演算法目的互相對照。輸出不是只看有沒有跑出東西，而是要確認每一筆結果是否符合資料結構規則。例如排序結果是否遞增、搜尋位置是否正確、樹走訪順序是否合理。
  21. 常見錯誤包含：陣列索引越界、指標沒有初始化、刪除節點後仍使用原指標、遞迴沒有終止條件、忘記更新 front/rear/top、比較條件寫反、或是沒有處理空資料結構。
  22. 老師出 trace code 題目時，通常會要求你寫出每一步變數變化。建議用表格記錄迴圈次數、目前索引、比較對象、資料結構內容與輸出結果，這樣最不容易漏掉中間狀態。
  23. 面試或考試常問這個程式的優缺點。優點通常是觀念清楚、容易實作、適合作為基本資料結構練習；缺點則可能是效率不一定最佳、記憶體使用較多、或遇到極端輸入時需要額外處理。
  24. 實際應用上，這類程式可延伸到資料整理、搜尋系統、排程管理、模擬問題、路徑探索、資料索引與演算法效能比較。資料結構的重點不是背程式，而是知道什麼情境該使用什麼結構。
  25. 讀完本程式後，最重要的練習是自己改一組輸入，手動推一次流程，再執行程式比較答案。只要能說清楚每個函式的任務、每個變數的角色、每個判斷式的目的，就代表你真正理解這支程式。

class HeapTree {  // 定義資料結構類別，集中管理資料成員與操作函式。
private:
    int heap_tree[MAX];    // 堆積樹陣列

    void display_f(void);  // 輸出函數
    void create(int id_number);      // 建立資料於堆積樹
    void removes(int index_temp);    // 從堆積樹中刪除資料
    void show(char op);              // 印出資料於螢幕

    void exchange(int *id1, int *id2);      // 交換資料
    int search(int id_number);   // 搜尋資料
};

void HeapTree::create(int id_number)  // ID_NUMBER為新增資料
{
    heap_tree[++last_index] = id_number;  // 將資料新增於最後  // 指定或更新變數值，改變目前資料狀態。
    adjust_u(heap_tree, last_index);    // 調整新增資料
}

void HeapTree::removes(int index_temp)  // INDEX_TEMP為欲刪除資料之INDEX
{
    // 以最後一筆資料代替刪除資料
    heap_tree[index_temp] = heap_tree[last_index];  // 指定或更新變數值，改變目前資料狀態。
    heap_tree[last_index--] = 0;  // 指定或更新變數值，改變目前資料狀態。
    if (last_index > 1) { // 當資料筆數大於1筆，則做調整  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
        // 當替代資料大於其PARENT NODE，則往上調整
        if (heap_tree[index_temp] > heap_tree[index_temp / 2] && index_temp > 1)  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
            adjust_u(heap_tree, index_temp);
        else // 替代資料小於其CHILDEN NODE，則往下調整  // 否則分支，處理上一個條件不成立的情況。
            adjust_d(heap_tree, index_temp, last_index-1);
    }
}

void HeapTree::show(char op)  // 函式宣告或函式開始，負責執行一段可重複使用的演算法步驟。
{
    int heap_temp[MAX+1];
    int c_index;  // 宣告變數，用來保存演算法執行過程中的資料。
    // 將堆積樹資料複製到另一個陣列作排序工作
    for (c_index = 1; c_index <= last_index; c_index++)  // for 迴圈，用固定範圍重複走訪資料。
        heap_temp[c_index] = heap_tree[c_index];  // 指定或更新變數值，改變目前資料狀態。
    // 將陣列調整為由小到大排列
    for (c_index = last_index-1; c_index > 0; c_index--) {  // for 迴圈，用固定範圍重複走訪資料。
        exchange(&heap_temp[1], &heap_temp[c_index+1]);
        adjust_d(heap_temp, 1, c_index);
    }
    cout << "\n       ID number\n";  // 輸出結果，將資料顯示或寫入檔案。
    cout << " =====================\n";  // 輸出結果，將資料顯示或寫入檔案。
    // 選擇第一種方式輸出，以遞增方式輸出--使用堆疊
    // 選擇第二種方式輸出，以遞減方式輸出--使用佇列
    switch(op) {  // 多分支判斷，依照不同值執行對應處理。
        case '1':  // 多分支判斷，依照不同值執行對應處理。
            for (c_index = 1; c_index <= last_index; c_index++) {  // for 迴圈，用固定範圍重複走訪資料。
                cout.width(14);  // 輸出結果，將資料顯示或寫入檔案。
                cout << heap_temp[c_index] << "\n";  // 輸出結果，將資料顯示或寫入檔案。
            }
            break;
        case '2':  // 多分支判斷，依照不同值執行對應處理。
            for (c_index = last_index; c_index > 0; c_index--) {  // for 迴圈，用固定範圍重複走訪資料。
                cout.width(14);  // 輸出結果，將資料顯示或寫入檔案。
                cout << heap_temp[c_index] << "\n";  // 輸出結果，將資料顯示或寫入檔案。
            }
            break;
    }
    cout << " =====================\n";  // 輸出結果，將資料顯示或寫入檔案。
    cout << " Total member: " << last_index << "\n";  // 輸出結果，將資料顯示或寫入檔案。
}

void HeapTree::adjust_u(int temp[], int index) // INDEX為目前資料在陣列之INDEX
{
    while (index > 1) { // 將資料往上調整至根為止  // while 迴圈，只要條件成立就持續執行。
        if(temp[index] <= temp[index/2])  // 資料調整完畢就跳出，否則交換資料  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
            break;
        else  // 否則分支，處理上一個條件不成立的情況。
            exchange(&temp[index], &temp[index/2]);
        index /= 2;  // 指定或更新變數值，改變目前資料狀態。
    }
}

// INDEX1為目前資料在陣列之INDEX，INDEX2為最後一筆資料在陣列之INDEX
void HeapTree::adjust_d(int temp[], int index1, int index2)  // 函式宣告或函式開始，負責執行一段可重複使用的演算法步驟。
{
    // ID_NUMBER記錄目前資料，INDEX_TEMP則是目前資料之CHILDEN NODE的INDEX
    int id_number, index_temp;  // 宣告變數，用來保存演算法執行過程中的資料。
    id_number = temp[index1];  // 指定或更新變數值，改變目前資料狀態。
    index_temp = index1 * 2;  // 指定或更新變數值，改變目前資料狀態。
    // 當比較資料之INDEX不大於最後一筆資料之INDEX，則繼續比較
    while (index_temp <= index2) {  // while 迴圈，只要條件成立就持續執行。
        if ((index_temp < index2) && (temp[index_temp] < temp[index_temp+1]))  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
            index_temp++;  // INDEX_TEMP記錄目前資料之CHILDEN NODE中較大者
        if (id_number >= temp[index_temp])  // 比較完畢則跳出，否則交換資料  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
            break;
        else {  // 否則分支，處理上一個條件不成立的情況。
            temp[index_temp/2] = temp[index_temp];  // 指定或更新變數值，改變目前資料狀態。
            index_temp *= 2;  // 指定或更新變數值，改變目前資料狀態。
        }
    }
    temp[index_temp/2] = id_number;  // 指定或更新變數值，改變目前資料狀態。
}

void HeapTree::exchange(int *id1, int *id2)  // 交換傳來之ID1及ID2儲存之資料
{
    int id_number;  // 宣告變數，用來保存演算法執行過程中的資料。
    id_number = *id1;  // 指定或更新變數值，改變目前資料狀態。
    *id1 = *id2;
    *id2 = id_number;
}

int HeapTree::search(int id_number)  // 尋找陣列中ID_NUMBER所在
{
    int c_index;  // 宣告變數，用來保存演算法執行過程中的資料。
    for (c_index = 1; c_index <= MAX; c_index++)  // for 迴圈，用固定範圍重複走訪資料。
        if (id_number == heap_tree[c_index])  // 條件判斷，依照目前資料狀態選擇不同處理路徑。
            return c_index;  // 找到則回傳資料在陣列中之INDEX  // 回傳函式結果，結束目前函式。
    return 0;  // 沒找到則回傳0  // 回傳函式結果，結束目前函式。
}
