01 Sorting Problem

非常詳細主題說明

說明排序問題的輸入是一組 records，每筆資料有 key，輸出必須依 key 形成非遞減順序。動畫用資料列、key 標示與結果區分，強調排序不一定只移動單一數字，而是以 key 決定 record 的相對順序。

主題補充說明

一、主題背景

Sorting Problem 的核心是把一組資料依照 key value 排成非遞減順序。投影片強調輸入不是只有單純整數，也可以是 record；真正被比較的是 key，而輸出是資料的重新排列結果。這個主題是所有排序演算法的共同起點，後面的 Bubble、Selection、Insertion、Merge、Quick 都是在解決同一個抽象問題。

二、核心概念

  • record
  • key value
  • permutation
  • nondecreasing order
  • input / output specification
  • 排序結果不唯一

三、運作原理

先定義資料集合，再指定比較依據 key，最後產生一個排列，使所有 key 依照由小到大排列。若有相同 key，排序結果可能不唯一，因此後面會延伸出 stable sorting 的觀念。

四、複雜度分析

此主題本身是在定義問題，不固定對應某一個時間複雜度；不同演算法會有 O(n²)、O(n log n) 等差異。考試時要先看題目要求的是「排序問題定義」還是「特定排序演算法」。

五、優缺點

  • 優點：能把所有排序演算法放在同一個規格下比較；容易判斷輸入、輸出與正確性。
  • 缺點：只定義問題，不說明如何排序；實際效率必須看採用哪一種演算法。

六、老師上課重點

老師通常會要求學生分清楚 record 與 key，不要把排序看成只是在移動數字。

七、期末考重點

常考輸入輸出定義、nondecreasing order、相同 key 時輸出不唯一，以及排序結果是否符合規格。

#include <iostream>
#include <vector>
#include <string>
#include <algorithm>
using namespace std;

// Record 代表一筆資料；key 是排序依據，name 是資料內容
struct Record {
    int key;
    string name;
};

// 根據 key 由小到大排序
void sortRecords(vector<Record>& records) {
    sort(records.begin(), records.end(), [](const Record& a, const Record& b) {
        return a.key < b.key; // 只比較 key，不直接比較整筆資料
    });
}

// 印出目前 records 的順序
void printRecords(const vector<Record>& records) {
    for (const auto& r : records) {
        cout << "(" << r.key << ", " << r.name << ") ";
    }
    cout << endl;
}

int main() {
    vector<Record> records = {{30, "R1"}, {10, "R2"}, {20, "R3"}};
    cout << "Before sorting: ";
    printRecords(records);
    sortRecords(records);
    cout << "After sorting : ";
    printRecords(records);
    return 0;
}

輸出結果

After sorting : (10, R2) (20, R3) (30, R1)

02 In-place and Stable Sorting

非常詳細主題說明

說明 in-place 與 stable 的差異：in-place 重點是額外記憶體少，stable 重點是相同 key 的原始先後順序不被破壞。動畫用相同 key 標籤與工作區對照，幫助分辨兩個性質不是同一件事。

主題補充說明

一、主題背景

In-place 與 Stable 是排序演算法的兩個重要性質。In-place 關心額外空間是否為常數；Stable 關心相同 key 的資料是否保持原本相對順序。這兩個性質常用來比較 Bubble、Insertion、Selection、Merge、Quick 等排序。

二、核心概念

  • in-place
  • stable sorting
  • extra space
  • relative order
  • identical key
  • permutation

三、運作原理

In-place 排序只允許使用少量額外變數，主要在原陣列中交換或移動元素。Stable 排序在遇到相同 key 時，不會把後出現的資料排到前面資料之前。

四、複雜度分析

In-place 通常看空間複雜度是否為 O(1)；stable 不一定影響時間複雜度，但若為了穩定性而額外配置陣列，空間可能上升到 O(n)。

五、優缺點

  • 優點：In-place 節省記憶體；Stable 適合多欄位排序，例如先依姓名排，再依分數排。
  • 缺點：有些高效率排序不容易同時 in-place 且 stable；強行維持穩定可能讓實作變複雜。

六、老師上課重點

老師容易考哪個排序是 stable、哪個是 in-place，以及相同 key 的相對位置是否改變。

七、期末考重點

常考判斷題、表格比較題，或給一串含重複 key 的資料要求判斷排序是否 stable。

完整 C++ 程式碼

#include <iostream>
#include <vector>
#include <string>
using namespace std;

struct Item {
    int key;
    string id; // 用 id 檢查相同 key 的先後順序是否被保留
};

// 穩定版 insertion sort；相同 key 不會互相交換
void stableInsertionSort(vector<Item>& a) {
    for (int i = 1; i < (int)a.size(); i++) {
        Item keyItem = a[i];
        int j = i - 1;
        // 只移動 key 較大的元素；相等時不移動，所以 stable
        while (j >= 0 && a[j].key > keyItem.key) {
            a[j + 1] = a[j];
            j--;
        }
        a[j + 1] = keyItem;
    }
}

void printItems(const vector<Item>& a) {
    for (const auto& x : a) cout << x.key << x.id << " ";
    cout << endl;
}

int main() {
    vector<Item> a = {{2,"A"}, {1,"B"}, {2,"C"}, {1,"D"}};
    cout << "Before: "; printItems(a);
    stableInsertionSort(a);
    cout << "After : "; printItems(a);
    return 0;
}

輸出結果

Before: 2A 1B 2C 1D
After : 1B 1D 2A 2C

函式說明

變數說明

變數說明

keyItem目前要插入到左側已排序區的元素。

03 Internal vs External Sorting

非常詳細主題說明

說明 internal sorting 是資料可放入主記憶體中排序，external sorting 則需處理磁碟或外部儲存的大量資料。動畫以記憶體區與外部檔案區表示資料搬移。

主題補充說明

一、主題背景

Internal Sorting 指所有資料都能放在主記憶體中處理；External Sorting 則處理資料量太大，必須依賴磁碟或外部儲存。這個主題讓學生理解排序不只看 CPU 比較次數，也要考慮 I/O 成本。

二、核心概念

  • internal sorting
  • external sorting
  • main memory
  • disk I/O
  • run
  • merge

三、運作原理

內部排序可直接在陣列中比較與交換；外部排序通常先把資料分批排序成 runs，再利用多路合併逐步產生完整排序結果。

四、複雜度分析

Internal sorting 常分析比較次數與交換次數；External sorting 更重視磁碟讀寫次數。若資料量遠大於記憶體，I/O 成本會主導整體效率。

五、優缺點

  • 優點：Internal sorting 實作簡單、速度快；External sorting 可處理超大型資料。
  • 缺點：Internal sorting 受記憶體容量限制；External sorting 實作複雜且 I/O 成本高。

六、老師上課重點

老師可能會問為什麼資料庫或大型檔案不能只用普通陣列排序。

七、期末考重點

常考 internal/external 定義差異、使用情境，以及為什麼 merge sort 適合 external sorting。

完整 C++ 程式碼

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

// Internal sorting：全部資料都放在記憶體 vector 中排序
void internalSort(vector<int>& data) {
    sort(data.begin(), data.end());
}

// 模擬 external sorting 的第一步：把大量資料切成小 run 後各自排序
vector<vector<int>> makeSortedRuns(vector<int> data, int runSize) {
    vector<vector<int>> runs;
    for (int i = 0; i < (int)data.size(); i += runSize) {
        vector<int> run;
        for (int j = i; j < (int)data.size() && j < i + runSize; j++) run.push_back(data[j]);
        sort(run.begin(), run.end());
        runs.push_back(run);
    }
    return runs;
}

int main() {
    vector<int> data = {9, 4, 7, 1, 6, 3};
    auto runs = makeSortedRuns(data, 2);
    cout << "Sorted runs: ";
    for (auto& run : runs) {
        cout << "[";
        for (int x : run) cout << x << " ";
        cout << "] ";
    }
    cout << endl;
    internalSort(data);
    cout << "Internal sorted: ";
    for (int x : data) cout << x << " ";
    return 0;
}

輸出結果

Sorted runs: [4 9 ] [1 7 ] [3 6 ]
Internal sorted: 1 3 4 6 7 9

makeSortedRuns(data, runSize)模擬外部排序，先切成小段 run 並各自排序。

變數說明

變數說明

04 Comparison Sort Overview

非常詳細主題說明

說明 comparison sort 只透過兩兩 key 比較決定順序，因此 Bubble、Selection、Insertion、Quick、Merge 都屬於比較排序。動畫用比較箭頭呈現每一次比較會產生 yes/no 決策。

主題補充說明

一、主題背景

Comparison Sort 是只透過比較 key 大小來決定順序的排序類型。Bubble、Selection、Insertion、Merge、Quick、Heap 都屬於 comparison sort。這個總覽主題重點在建立不同排序法的比較架構。

二、核心概念

  • comparison
  • swap
  • shift
  • divide and conquer
  • lower bound
  • O(n log n)

三、運作原理

所有 comparison sort 都透過「比較兩個元素」逐步縮小可能排列。不同演算法差異在於比較的順序、交換方式、是否分治、是否使用額外空間。

四、複雜度分析

簡單排序常見 O(n²)；較進階排序如 merge sort、heap sort 平均或最壞可達 O(n log n)。Comparison sort 的最壞情況下界是 Ω(n log n)。

五、優缺點

  • 優點：模型通用，適合比較各種資料型態，只要 key 可比較即可。
  • 缺點：受 comparison lower bound 限制，無法在一般比較模型中突破 Ω(n log n) 最壞下界。

六、老師上課重點

老師常用這個主題連接各種排序演算法與 decision tree lower bound。

七、期末考重點

常考各排序法的時間、空間、stable、in-place 表格，以及 comparison sort 下界。

05 Bubble Sort

非常詳細主題說明

Bubble Sort 的重點是只比較相鄰元素，若順序錯誤就交換；每一趟會把目前未排序區中的最大值像泡泡一樣推到右邊。這版動畫特別用相鄰箭頭、紅色比較格與綠色完成後綴呈現，不再只是一般長條圖。

主題補充說明

一、主題背景

Bubble Sort 是最直觀的交換式排序。它反覆比較相鄰元素，若順序錯誤就交換，使較大的元素像氣泡一樣逐輪移到右端。

二、核心概念

  • adjacent comparison
  • swap
  • pass
  • early stop
  • stable
  • in-place

三、運作原理

每一輪從左到右比較相鄰元素，若 a[j] > a[j+1] 就交換。第一輪會把最大值移到最後，第二輪把次大值移到倒數第二個位置，逐步縮小未排序區。

四、複雜度分析

最壞與平均時間 O(n²)；若加入 swapped 旗標，最佳情況可為 O(n)。空間複雜度 O(1)。

五、優缺點

  • 優點：觀念簡單、容易 trace、可 stable、可 in-place。
  • 缺點：效率差，大量資料不適合；交換次數可能很多。

六、老師上課重點

老師常用 Bubble Sort 考相鄰比較、每一輪結果、是否 early stop。

七、期末考重點

常考給一組資料，要求畫出每一 pass 後的陣列狀態與比較交換次數。

#include <iostream>
#include <vector>
using namespace std;

void bubbleSort(vector<int>& a) {
    int n = a.size();
    for (int pass = 0; pass < n - 1; pass++) {
        for (int j = 0; j < n - 1 - pass; j++) {
            // 只比較相鄰兩格
            if (a[j] > a[j + 1]) swap(a[j], a[j + 1]);
        }
    }
}

int main() {
    vector<int> a = {5, 1, 4, 2};
    bubbleSort(a);
    cout << "Bubble sorted: ";
    for (int x : a) cout << x << " ";
    return 0;
}

輸出結果

Bubble sorted: 1 2 4 5

函式說明

06 Selection Sort

非常詳細主題說明

Selection Sort 的重點是每一輪在未排序區掃描出最小值，再把它交換到已排序區尾端。這版動畫會標出 scan 位置、目前 min 位置與已排序 prefix，和 Bubble Sort 的相鄰交換完全不同。

主題補充說明

一、主題背景

Selection Sort 每一輪從未排序區中選出最小值，放到目前位置。它強調「選擇最小」而不是頻繁交換。

二、核心概念

  • minimum selection
  • unsorted region
  • sorted prefix
  • swap once per pass
  • not stable by default
  • in-place

三、運作原理

第 i 輪在 a[i..n-1] 中找最小元素的位置 minIndex，最後與 a[i] 交換。完成後 a[0..i] 成為已排序前綴。

四、複雜度分析

無論資料是否接近排序，時間通常都是 O(n²)，因為每輪仍要掃描未排序區；空間 O(1)。

五、優缺點

  • 優點：交換次數少，最多 n-1 次；實作簡單，適合說明 sorted/unsorted boundary。
  • 缺點：比較次數固定偏高；一般實作不穩定。

六、老師上課重點

老師常比較 Selection Sort 與 Bubble Sort：Selection 交換少，但比較仍多。

七、期末考重點

常考每輪 minIndex、交換後陣列、是否 stable，以及 O(n²) 原因。

#include <iostream>
#include <vector>
using namespace std;

void selectionSort(vector<int>& a) {
    int n = a.size();
    for (int i = 0; i < n - 1; i++) {
        int minIndex = i;
        for (int j = i + 1; j < n; j++) {
            if (a[j] < a[minIndex]) minIndex = j;
        }
        swap(a[i], a[minIndex]); // 把最小值放到前面
    }
}

07 Insertion Sort

非常詳細主題說明

Insertion Sort 的重點是把第一個未排序元素當作 key，插入左側已排序 prefix 的正確位置。動畫用 key 被取出、較大元素右移、空位插入來表現，能對應手算題目中每一輪插入的狀態。

主題補充說明

一、主題背景

Insertion Sort 模擬整理撲克牌的方式，逐一把新元素插入已排序區的正確位置。它對接近排序的資料很有效。

二、核心概念

  • sorted prefix
  • key
  • shift
  • insert position
  • stable
  • adaptive

三、運作原理

從第二個元素開始，把目前 key 暫存，往左比較已排序區元素，較大的元素右移，直到找到插入位置，再放回 key。

四、複雜度分析

最壞與平均 O(n²)；若資料已排序，最佳 O(n)。空間 O(1)。

五、優缺點

  • 優點：簡單、stable、in-place，對小資料或 nearly sorted data 效率好。
  • 缺點：若資料反向排序，移動次數很多，效率退化。

六、老師上課重點

老師常要求 trace key 的移動與已排序區如何擴張。

七、期末考重點

常考每次插入後的陣列狀態、移動次數、最佳情況為何是 O(n)。

#include <iostream>
#include <vector>
using namespace std;

void insertionSort(vector<int>& a) {
    for (int i = 1; i < (int)a.size(); i++) {
        int key = a[i];
        int j = i - 1;
        // 將比 key 大的元素往右移
        while (j >= 0 && a[j] > key) {
            a[j + 1] = a[j];
            j--;
        }
        a[j + 1] = key; // 把 key 插入正確位置
    }
}

int main() {
    vector<int> a = {5, 2, 4, 6, 1};
    insertionSort(a);
    cout << "Insertion sorted: ";
    for (int x : a) cout << x << " ";
    return 0;
}

08 Quick Sort Partition

非常詳細主題說明

Quick Sort Partition 的重點是選 pivot，使用 i/j 指標把小於 pivot 與大於 pivot 的資料分到兩側。動畫用 pivot 黃色標示、i/j 指標與左右分區呈現，不會和 Merge Sort 或 Bubble Sort 混淆。

主題補充說明

一、主題背景

Partition 是 Quick Sort 的核心步驟。它選定 pivot，將資料分成小於 pivot 與大於 pivot 的兩側，使 pivot 最後位於正確位置。

二、核心概念

  • pivot
  • partition
  • left pointer
  • right pointer
  • swap
  • divide and conquer

三、運作原理

選取 pivot 後，透過指標掃描或 Lomuto/Hoare 方法，把較小元素移到左側、較大元素移到右側。Partition 完成後，pivot 左右兩邊可分別遞迴排序。

四、複雜度分析

單次 partition 需要線性掃描，時間 O(n)，空間視實作可為 O(1)。整體 quick sort 效率取決於 partition 是否平衡。

五、優缺點

  • 優點：核心步驟快速，通常 cache-friendly，平均效率很好。
  • 缺點：若 pivot 選不好，切分極度不平衡，整體可能退化 O(n²)。

六、老師上課重點

老師常要求學生畫出 partition 過程中 i、j 或 pivot 的位置變化。

七、期末考重點

常考 partition 結果是否合法、pivot 最後位置、以及左右子陣列如何形成。

#include <iostream>
#include <vector>
using namespace std;

int partition(vector<int>& a, int low, int high) {
    int pivot = a[high];
    int i = low - 1;
    for (int j = low; j < high; j++) {
        if (a[j] <= pivot) {
            i++;
            swap(a[i], a[j]);
        }
    }
    swap(a[i + 1], a[high]);
    return i + 1;
}

int main() {
    vector<int> a = {9, 3, 7, 1, 6};
    int p = partition(a, 0, a.size() - 1);

    cout << "After partition: ";
    for (int x : a) cout << x << " ";
    return 0;
}

#include <iostream>
#include <vector>
using namespace std;

int partition(vector<int>& a, int low, int high) {
    int pivot = a[high], i = low - 1;
    for (int j = low; j < high; j++) {
        if (a[j] <= pivot) swap(a[++i], a[j]);
    }
    swap(a[i + 1], a[high]);
    return i + 1;
}

void quickSort(vector<int>& a, int low, int high) {
    if (low >= high) return;
    int p = partition(a, low, high);
    quickSort(a, low, p - 1);   // 遞迴處理左半
    quickSort(a, p + 1, high);  // 遞迴處理右半
}

int main() {

    quickSort(a, 0, a.size() - 1);
    cout << "Quick sorted: ";
    for (int x : a) cout << x << " ";
    return 0;
}

10 Decision Tree Lower Bound

非常詳細主題說明

Decision Tree Lower Bound 說明比較排序可以視為二元決策樹，每個 leaf 對應一種可能排列，因此高度至少需要容納 n! 個結果，得到 Ω(n log n) 下界。動畫用比較節點與 permutation leaf 呈現。

主題補充說明

一、主題背景

Decision Tree Lower Bound 用決策樹模型證明 comparison sort 最壞情況至少需要 Ω(n log n) 次比較。這是排序理論的重點。

二、核心概念

  • decision tree
  • comparison sort
  • leaf
  • permutation
  • height
  • lower bound

三、運作原理

每一次比較只有兩種結果，因此可用二元決策樹表示排序過程。n 個元素有 n! 種可能排列，因此樹至少要有 n! 個葉節點，樹高至少 log₂(n!)，約為 Ω(n log n)。

四、複雜度分析

此主題不是某個演算法，而是證明所有 comparison sort 在最壞情況下無法比 Ω(n log n) 更好。

五、優缺點

  • 優點：能從理論上解釋為什麼 merge/heap/quick 平均 O(n log n) 已經接近比較模型極限。
  • 缺點：只適用於 comparison sort，不限制 counting sort、radix sort 等非比較排序。

六、老師上課重點

老師常用它說明「不是演算法寫不好，而是模型有下界」。

七、期末考重點

常考 n! 葉節點、decision tree height、Ω(n log n) 推導與適用範圍。

#include <iostream>
using namespace std;

long long factorial(int n) {
    long long ans = 1;
    for (int i = 2; i <= n; i++) ans *= i;
    return ans;
}

int lowerBoundComparisons(int n) {
    long long leaves = factorial(n); // 排列數 n!
    int h = 0;
    long long capacity = 1;
    while (capacity < leaves) {
        capacity *= 2; // 高度 h 的二元決策樹最多 2^h 個 leaf
        h++;
    }
    return h;
}

int main() {
    int n = 4;
    cout << "n! = " << factorial(n) << endl;
    cout << "Minimum comparisons >= " << lowerBoundComparisons(n) << endl;
    return 0;
}

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

void iterativeMergeSort(vector<int>& a) {
    int n = a.size();
    vector<int> temp(n);
    for (int width = 1; width < n; width *= 2) {
        for (int left = 0; left < n; left += 2 * width) {
            int mid = min(left + width, n);
            int right = min(left + 2 * width, n);
            merge(a.begin() + left, a.begin() + mid,
                  a.begin() + mid, a.begin() + right,
                  temp.begin() + left);
        }
        a = temp;
    }
}

int main() {
    vector<int> a = {7, 3, 5, 1};
    iterativeMergeSort(a);
    cout << "Iterative merge sorted: ";
    for (int x : a) cout << x << " ";
    return 0;
}

#include <iostream>
#include <vector>
using namespace std;

vector<int> mergeTwoRuns(const vector<int>& left, const vector<int>& right) {
    vector<int> out;
    int i = 0, j = 0;
    while (i < (int)left.size() && j < (int)right.size()) {
        if (left[i] <= right[j]) out.push_back(left[i++]);
        else out.push_back(right[j++]);
    }
    while (i < (int)left.size()) out.push_back(left[i++]);
    while (j < (int)right.size()) out.push_back(right[j++]);
    return out;
}

int main() {
    vector<int> left = {1, 4, 8};
    vector<int> right = {2, 3, 9};
    vector<int> merged = mergeTwoRuns(left, right);
    cout << "Merged: ";
    for (int x : merged) cout << x << " ";
    return 0;
}

一、主題背景

Recursive Merge Sort 是典型 divide and conquer 演算法。它先把陣列切成左右兩半，遞迴排序後再 merge。

二、核心概念

  • divide and conquer
  • mid
  • recursive call
  • base case
  • merge
  • stable

三、運作原理

若區段長度大於 1，就切成 left half 與 right half；分別遞迴排序後，呼叫 merge 合併。base case 是單一元素自然有序。

四、複雜度分析

時間 O(n log n)，空間 O(n) 加上遞迴堆疊 O(log n)。最壞、平均、最佳時間都為 O(n log n)。

五、優缺點

  • 優點：穩定、效能穩定、適合證明分治與遞迴。
  • 缺點：需要額外空間；遞迴與 merge 實作比簡單排序複雜。

六、老師上課重點

老師常用遞迴樹說明每層成本 O(n)，層數 log n。

七、期末考重點

常考切割順序、merge 回來的順序、遞迴終止條件與複雜度。

#include <iostream>
#include <vector>
using namespace std;

void mergeRange(vector<int>& a, int l, int m, int r) {
    vector<int> temp;
    int i = l, j = m + 1;
    while (i <= m && j <= r) {
        if (a[i] <= a[j]) temp.push_back(a[i++]);
        else temp.push_back(a[j++]);
    }
    while (i <= m) temp.push_back(a[i++]);
    while (j <= r) temp.push_back(a[j++]);
    for (int k = 0; k < (int)temp.size(); k++) a[l + k] = temp[k];
}

void mergeSort(vector<int>& a, int l, int r) {
    if (l >= r) return;
    int m = (l + r) / 2;
    mergeSort(a, l, m);      // 左半切半排序
    mergeSort(a, m + 1, r);  // 右半切半排序
    mergeRange(a, l, m, r);  // 合併左右半
}

int main() {
    vector<int> a = {6, 2, 8, 1, 5};
    mergeSort(a, 0, a.size() - 1);
    cout << "Recursive merge sorted: ";
    for (int x : a) cout << x << " ";
    return 0;
}

輸出結果

Recursive merge sorted: 1 2 5 6 8

mergeSort(a, l, r)遞迴切半，再呼叫 mergeRange 合併。
mergeRange(a, l, m, r)合併 a[l..m] 與 a[m+1..r]。

變數說明

14 Sorting Summary

非常詳細主題說明

Sorting Summary 用表格比較各排序法的平均時間、穩定性與是否 in-place。動畫逐步 highlight 不同演算法，適合考前整理與快速比較。

主題補充說明

一、主題背景

Sorting Summary 用來統整各種排序演算法的特性。重點不是只背名稱，而是能依資料量、穩定性、空間限制與最壞情況選擇合適排序法。

二、核心概念

  • time complexity
  • space complexity
  • stable
  • in-place
  • adaptive
  • comparison lower bound

三、運作原理

將 Bubble、Selection、Insertion、Merge、Quick 等排序放入同一張比較表，從最佳、平均、最壞時間、額外空間、是否穩定、是否 in-place 進行分析。

四、複雜度分析

簡單排序多為 O(n²)；Merge Sort 穩定 O(n log n) 但需 O(n) 空間；Quick Sort 平均 O(n log n) 但最壞 O(n²)。

五、優缺點

  • 優點：方便建立考試判斷框架，也能理解實務選擇排序法的理由。
  • 缺點：若只背表格而不會 trace，遇到手算題仍容易錯。

六、老師上課重點

老師通常會把 summary 當作期末考選擇題、配合題與簡答題來源。

七、期末考重點

常考排序法比較表、stable/in-place 判斷、最壞情況、以及哪種資料適合哪種排序。

#include <iostream>
#include <iomanip>
#include <string>
using namespace std;

struct SortInfo {
    string name;
    string avgTime;
    string stable;
    string inPlace;
};

void printSummary(SortInfo table[], int n) {
    cout << left << setw(16) << "Algorithm" << setw(12) << "Avg" 
         << setw(10) << "Stable" << "In-place" << endl;
    for (int i = 0; i < n; i++) {
        cout << left << setw(16) << table[i].name << setw(12) << table[i].avgTime
             << setw(10) << table[i].stable << table[i].inPlace << endl;
    }
}

int main() {
    SortInfo table[] = {
        {"Bubble", "O(n^2)", "Yes", "Yes"},
        {"Selection", "O(n^2)", "No", "Yes"},
        {"Insertion", "O(n^2)", "Yes", "Yes"},
        {"Merge", "O(nlogn)", "Yes", "No"},
        {"Quick", "O(nlogn)", "No", "Yes"}
    };
    printSummary(table, 5);
    return 0;
}
