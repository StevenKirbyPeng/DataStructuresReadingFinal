📖 主題詳細說明：Priority Queue 與 ADT MaxPQ

一、主題背景

Priority Queue 是一種依照 priority/key value 決定刪除順序的抽象資料型態。一般 queue 是 FIFO，先進先出；priority queue 則是每次刪除最高或最低 priority 的元素。投影片中 MaxPQ 以抽象類別定義 IsEmpty、Top、Push、Pop，重點是把「規格」和「實作」分開。 Trees Part II 的重點是把 Trees Part I 的基本節點、遞迴與 traversal 延伸到更實用的資料結構。這些主題常出現在期末考，因為它們可以同時考觀念、圖形、演算法步驟、複雜度與 C++ 類別設計。

本主題要用「定義 → 圖形 → 操作 → 複雜度 → 考試題型」的方式理解。只背名詞通常不夠，因為老師很常要求學生依照一組輸入資料畫出 heap、BST、winner tree 或 union-find forest 的最後狀態。

二、核心概念

  • priority
  • key value
  • MaxPQ
  • MinPQ
  • Top
  • Push
  • Pop
  • ADT

三、運作原理

操作時應先確認資料結構的不變條件。例如 heap 要保持 complete binary tree 與 heap property；BST 要保持左小右大的排序規則；forest 轉 binary tree 要保持第一孩子與下一兄弟的對應；Union-Find 要保持每個集合只有一個代表 root。

程式實作時，要特別注意 index、pointer 與 parent/child 關係。若是陣列表示，通常靠公式找到父節點與子節點；若是 linked representation，就必須正確更新 link。若是演算法動畫，則每一個 frame 都代表一次狀態改變，不能只看最後結果。

四、複雜度與比較

分析項目重點
時間複雜度若每次操作沿著樹高移動，通常是 O(h) 或 O(log n)；若必須掃描全部節點，通常是 O(n)。
空間複雜度主要取決於是否需要額外陣列、遞迴堆疊、輔助 queue/stack，或是否只改變原有指標。
考試判斷先寫出不變條件，再逐步 trace。若每步能說明為何合法，圖通常就不會畫錯。

五、優點與缺點

  • 優點：樹狀結構能把資料的階層、排序、優先權或集合代表清楚表示出來。
  • 優點：許多操作只需要沿著一條 root-to-leaf path，因此在平衡情況下效率很好。
  • 缺點：若樹高過高或表示法不當，操作可能退化成線性時間。
  • 缺點：指標或陣列索引錯誤會導致整個結構失效。

六、老師上課與期末考重點

  • 會畫操作前後的樹形。
  • 會說明每一步為何要交換、連接、刪除或壓縮。
  • 會追蹤 C++ 程式碼中的變數與指標。
  • 會分析 worst case 與 average/common case。

💻 C++ 程式範例：Priority Queue 與 ADT MaxPQ

這段程式碼是針對本下拉式主題設計的演算法範例，保留中文註解，方便對照投影片中的操作流程、資料結構不變條件與期末考 trace code 題型。

完整程式碼

#include <iostream>
#include <vector>
#include <stdexcept>
using namespace std;

// MaxPQ：以陣列模擬最大優先佇列，數值越大 priority 越高
class MaxPQ {
private:
    vector<int> data; // 儲存所有元素，這裡用線性陣列示範 ADT 行為
public:
    // 判斷 priority queue 是否為空
    bool IsEmpty() const { return data.empty(); }

    // 插入一個元素，代表 Push 操作
    void Push(int x) { data.push_back(x); }

    // 回傳目前 priority 最大的元素，代表 Top 操作
    int Top() const {
        if (IsEmpty()) throw runtime_error("PQ is empty");
        int best = data[0];
        for (int x : data) if (x > best) best = x;
        return best;
    }

    // 刪除 priority 最大的元素，代表 Pop 操作
    void Pop() {
        if (IsEmpty()) return;
        int bestIndex = 0;
        for (int i = 1; i < (int)data.size(); ++i)
            if (data[i] > data[bestIndex]) bestIndex = i;
        data.erase(data.begin() + bestIndex);
    }
};

int main() {
    MaxPQ pq;
    pq.Push(30); pq.Push(10); pq.Push(50); pq.Push(20);
    cout << "Top = " << pq.Top() << "\n";
    pq.Pop();
    cout << "After Pop, Top = " << pq.Top() << "\n";
    cout << "IsEmpty = " << (pq.IsEmpty() ? "true" : "false") << "\n";
}

輸出結果

Top = 50
After Pop, Top = 30
IsEmpty = false

函式說明

  • IsEmpty()：判斷 MaxPQ 是否沒有元素。
  • Push(x)：把新元素放入 priority queue。
  • Top()：掃描目前資料並回傳 priority 最大的元素。
  • Pop()：刪除 priority 最大的元素。

變數說明

  • data：儲存 priority queue 內所有元素。
  • best / bestIndex：目前找到的最大 priority 元素與位置。

  • Frame 1：建立 MaxPQ<T> 抽象介面，先看到 IsEmpty、Top、Push、Pop 四個核心操作。

💻 C++ 程式範例：Max Heap 插入 Push

這段程式碼是針對本下拉式主題設計的演算法範例，保留中文註解，方便對照投影片中的操作流程、資料結構不變條件與期末考 trace code 題型。

完整程式碼

#include <iostream>
#include <vector>
using namespace std;

class MaxHeap {
private:
    vector<int> heap; // 1-based array，方便使用 i/2 找 parent
public:
    MaxHeap() { heap.push_back(-1); }

    // Push：先放到最後，再一路 bubble up 修正 heap property
    void Push(int x) {
        heap.push_back(x);
        int i = heap.size() - 1;
        while (i > 1 && heap[i] > heap[i / 2]) {
            swap(heap[i], heap[i / 2]);
            i /= 2;
        }
    }

    void Print() const {
        for (int i = 1; i < (int)heap.size(); ++i) cout << heap[i] << ' ';
        cout << "\n";
    }
};

int main() {
    MaxHeap h;
    for (int x : {20, 15, 2, 14, 10, 1}) h.Push(x);
    cout << "Before Push(21): "; h.Print();
    h.Push(21);
    cout << "After Push(21): "; h.Print();
}

Before Push(21): 20 15 2 14 10 1 
After Push(21): 21 15 20 14 10 1 2

函式說明

  • Push(x)：將新元素放在陣列最後，再用 bubble up 維持 max heap。
  • Print()：輸出目前 heap 的陣列內容。

變數說明

  • heap：1-based max heap array。
  • i：新節點 bubble up 時所在位置。
  • x：準備插入的新 key。

#include <iostream>
#include <vector>
using namespace std;

class MaxHeap {
private:
    vector<int> heap; // heap[1] 是最大值

    // DownHeap：root 被最後元素取代後，往下調整
    void DownHeap(int i) {
        int n = heap.size() - 1;
        while (2 * i <= n) {
            int child = 2 * i;
            if (child + 1 <= n && heap[child + 1] > heap[child]) child++;
            if (heap[i] >= heap[child]) break;
            swap(heap[i], heap[child]);
            i = child;
        }
    }
public:
    MaxHeap() { heap = {-1, 21, 15, 20, 14, 10, 1, 2}; }

    int Pop() {
        int maxValue = heap[1];
        heap[1] = heap.back();
        heap.pop_back();
        if (heap.size() > 1) DownHeap(1);
        return maxValue;
    }

    void Print() const {
        for (int i = 1; i < (int)heap.size(); ++i) cout << heap[i] << ' ';
        cout << "\n";
    }
};

int main() {
    MaxHeap h;
    cout << "Before Pop: "; h.Print();
    cout << "Pop max = " << h.Pop() << "\n";
    cout << "After Pop: "; h.Print();
}

#include <vector>
using namespace std;

// 對 index i 執行 MaxHeapify，使 i 以下重新符合 max heap
void MaxHeapify(vector<int>& a, int i, int n) {
    while (2 * i <= n) {
        int child = 2 * i;

}

void Print(const vector<int>& a) {
    for (int i = 1; i < (int)a.size(); ++i) cout << a[i] << ' ';
    cout << "\n";
}

int main() {

    BuildMaxHeap(a);
    cout << "After BuildMaxHeap: "; Print(a);
}

這段程式碼是針對本下拉式主題設計的演算法範例，保留中文註解，方便對照投影片中的操作流程、資料結構不變條件與期末考 trace code 題型。

完整程式碼

#include <iostream>
#include <vector>
using namespace std;

void Heapify(vector<int>& a, int n, int i) {
    while (true) {
        int largest = i, l = 2 * i + 1, r = 2 * i + 2;
        if (l < n && a[l] > a[largest]) largest = l;
        if (r < n && a[r] > a[largest]) largest = r;
        if (largest == i) break;
        swap(a[i], a[largest]);
        i = largest;
    }
}

// HeapSort：先建 max heap，再把最大值逐一交換到最後
void HeapSort(vector<int>& a) {
    int n = a.size();
    for (int i = n / 2 - 1; i >= 0; --i) Heapify(a, n, i);
    for (int end = n - 1; end > 0; --end) {
        swap(a[0], a[end]);
        Heapify(a, end, 0);
    }
}

int main() {
    vector<int> a = {5, 1, 4, 2, 8};
    HeapSort(a);
    cout << "Sorted: ";
    for (int x : a) cout << x << ' ';
    cout << "\n";
}

  • n：目前 heap 有效範圍。

    else if (key > root->key) root->right = Insert(root->right, key);

    if (!root) return false;
    if (key == root->key) return true;
    if (key < root->key) return SearchRecursive(root->left, key);
    return SearchRecursive(root->right, key);
}

    while (cur) {
        if (key == cur->key) return true;
        cur = (key < cur->key) ? cur->left : cur->right;
    }
    return false;
}

    Node* left;
    Node* right;
    Node(int k) : key(k), size(1), left(nullptr), right(nullptr) {}

Node* Insert(Node* root, int key) {

    if (key < root->key) root->left = Insert(root->left, key);
    else root->right = Insert(root->right, key);
    root->size = 1 + Size(root->left) + Size(root->right);
    return root;

    if (k == leftSize + 1) return root->key;

    return Select(root->right, k - leftSize - 1);
}

int Height(Node* root) {
    if (!root) return 0;
    return 1 + max(Height(root->left), Height(root->right));
}

    cout << "BST height = " << Height(root) << "\n";

Union-Find 的效率取決於樹高。Weighting rule 讓較小的樹接到較大的樹，避免高度快速增加；Collapsing rule 在 Find 時把路徑上的節點直接連到 root，也稱 path compression。 Trees Part II 的重點是把 Trees Part I 的基本節點、遞迴與 traversal 延伸到更實用的資料結構。這些主題常出現在期末考，因為它們可以同時考觀念、圖形、演算法步驟、複雜度與 C++ 類別設計。

二、核心概念

  • weighted union
  • count field
  • path compression
  • collapsing rule
  • amortized

三、運作原理

操作時應先確認資料結構的不變條件。例如 heap 要保持 complete binary tree 與 heap property；BST 要保持左小右大的排序規則；forest 轉 binary tree 要保持第一孩子與下一兄弟的對應；Union-Find 要保持每個集合只有一個代表 root。

程式實作時，要特別注意 index、pointer 與 parent/child 關係。若是陣列表示，通常靠公式找到父節點與子節點；若是 linked representation，就必須正確更新 link。若是演算法動畫，則每一個 frame 都代表一次狀態改變，不能只看最後結果。

分析項目重點

💻 C++ 程式範例：Weighting Rule 與 Collapsing Rule

這段程式碼是針對本下拉式主題設計的演算法範例，保留中文註解，方便對照投影片中的操作流程、資料結構不變條件與期末考 trace code 題型。

完整程式碼

#include <iostream>
#include <vector>
using namespace std;

class WeightedDSU {
private:
    vector<int> parent, sz; // sz[root] 記錄集合大小，用於 weighting rule
public:
    WeightedDSU(int n) : parent(n), sz(n, 1) {
        for (int i = 0; i < n; ++i) parent[i] = i;
    }

    // Collapsing rule：Find 時順便做 path compression
    int Find(int x) {
        if (parent[x] == x) return x;
        return parent[x] = Find(parent[x]);
    }

    // Weighting rule：小集合掛到大集合底下
    void Union(int a, int b) {
        int ra = Find(a), rb = Find(b);
        if (ra == rb) return;
        if (sz[ra] < sz[rb]) swap(ra, rb);
        parent[rb] = ra;
        sz[ra] += sz[rb];
    }

    void Print() {
        for (int i = 0; i < (int)parent.size(); ++i)
            cout << i << "->" << Find(i) << ' ';
        cout << "\n";
    }
};

int main() {
    WeightedDSU dsu(8);
    dsu.Union(0, 1); dsu.Union(2, 3); dsu.Union(0, 2); dsu.Union(4, 5);
    dsu.Print();
}

輸出結果

0->0 1->0 2->0 3->0 4->4 5->4 6->6 7->7

函式說明

  • Find(x)：使用 collapsing rule 做 path compression。
  • Union(a,b)：使用 weighting rule，把小集合掛到大集合。
  • Print()：輸出每個元素的集合代表。

變數說明

  • parent：parent array。
  • sz：每個 root 所代表集合的大小。
  • ra / rb：合併前兩個集合的 root。

完整程式碼

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

struct Block {
    char name;
    int w, h;
    int x, y;
};

// 簡化 B*-Tree placement：依序把 block 放到右側，示範 floorplanning 面積計算
int ComputeArea(vector<Block>& blocks) {
    int curX = 0, maxH = 0;
    for (auto& b : blocks) {
        b.x = curX;
        b.y = 0;
        curX += b.w;
        maxH = max(maxH, b.h);
    }
    return curX * maxH;
}

void PrintBlocks(const vector<Block>& blocks) {
    for (auto b : blocks)
        cout << b.name << " at (" << b.x << "," << b.y << ") size=" << b.w << "x" << b.h << "\n";
}

int main() {
    vector<Block> blocks = {{'A', 3, 2}, {'B', 2, 4}, {'C', 4, 1}};
    int area = ComputeArea(blocks);
    PrintBlocks(blocks);
    cout << "Total area = " << area << "\n";
}
