Sequential vs. Linked Representation
非常詳細主題說明：
比較連續記憶體與鏈結記憶體；示範插入造成搬移或改 pointer；理解 O(n) 與 O(1) 差異。本主題屬於 Linked Lists 章節，重點在理解資料不一定連續放在記憶體中，而是透過 link/pointer 將節點串起來。考試常要求畫出操作前後的指標變化、判斷時間複雜度，以及說明 first、last、header、front、rear、iterator 等角色。
主題背景：
本主題比較 sequential representation 與 linked representation。Sequential 以陣列連續儲存，能用 index 直接存取；linked 則用節點與 pointer 串接，資料在記憶體中不必連續。Linked Lists 章節的核心就是理解「位置關係」不一定來自實體相鄰，而可以由 link 欄位描述。

核心概念：
  • sequential mapping
  • array index
  • linked representation
  • node
  • pointer/link
  • insert/delete cost
運作原理：
陣列插入中間元素時，後方資料通常要整批右移；刪除時也要左移補洞。Linked list 插入或刪除某節點後方資料時，只要找到前一個節點並更新 link，即可改變串列順序。

複雜度分析：
陣列隨機存取為 O(1)，但中間插入刪除常為 O(n)。Linked list 若已知前一節點，插入刪除為 O(1)，但搜尋第 i 個節點通常要 O(n)。

優缺點：
  • 優點：Linked representation 適合頻繁插入刪除，不必搬移大量資料。
  • 缺點：Linked list 不能直接用 index 隨機存取，且每個節點需要額外 pointer 空間。
老師上課重點：
老師通常會用 EAT、FAT、HAT、GAT 這類圖示說明陣列搬移與 pointer 改接的差異。

期末考重點：
期末常考操作前後圖形、時間複雜度比較，以及為什麼 linked insert/delete 可以是 O(1)。
非常詳細動畫說明：
動畫目的：比較陣列連續儲存與 linked list 非連續儲存的差異。Frame 1 先呈現原始資料列；Frame 2 強調 sequential representation 可用 index 找元素；Frame 3 顯示中間插入時陣列後方元素需要搬移；Frame 4 對照 linked representation 只要改變 link；Frame 5 到 Frame 8 逐步標示 first、node、link 與 NULL 的關係。觀察重點是「陣列靠位置」、「串列靠指標」。考試常要求畫出插入 GAT 或刪除 GAT 後，陣列搬移與 link 改接的差異。
C++ 程式範例：
#include <iostream>
using namespace std;

class Chain;
class ChainNode {
    friend class Chain;
public:
    ChainNode(int element = 0, ChainNode* next = 0) {
        data = element;
        link = next;
    }
private:
    int data;
    ChainNode* link;
};

class Chain {
public:
    Chain() { first = 0; }
    void Create2();
    void Insert50(ChainNode* x);
    void Delete(ChainNode* x, ChainNode* y);
    ChainNode* First() { return first; }
    void Print() const {
        for (ChainNode* p = first; p != 0; p = p->link) cout << p->data << ' ';
        cout << '\n';
    }
private:
    ChainNode* first;
};

void Chain::Create2() {
    ChainNode* second = new ChainNode(20, 0);
    first = new ChainNode(10, second);
}

void Chain::Insert50(ChainNode* x) {
    if (first) x->link = new ChainNode(50, x->link);
    else first = new ChainNode(50, 0);
}

void Chain::Delete(ChainNode* x, ChainNode* y) {
    if (x == first) first = first->link;
    else y->link = x->link;
    delete x;
}

int main() {
    Chain c;
    c.Create2();
    c.Insert50(c.First());
    c.Print();
    return 0;
}

Array 插入後: EAT FAT GAT HAT 
Linked 插入後: EAT -> FAT -> GAT -> HAT -> NULL

arrayInsertDemo() 示範陣列插入；linkedInsertDemo() 示範 linked list 改接概念；main() 執行兩種表示法比較。

變數說明：
a 是陣列資料；x 是輸出的字串元素。

Singly Linked Chain
非常詳細主題說明：
建立 first 指標；逐節點沿 link 前進；最後遇到 NULL 停止。本主題屬於 Linked Lists 章節，重點在理解資料不一定連續放在記憶體中，而是透過 link/pointer 將節點串起來。考試常要求畫出操作前後的指標變化、判斷時間複雜度，以及說明 first、last、header、front、rear、iterator 等角色。
主題背景：
Singly Linked Chain 是最基本的 linked list。每個節點包含 data 與 link，link 指向下一個節點，first 指向第一個節點，最後節點指向 NULL。

核心概念：
  • Node
  • data field
  • link field
  • first pointer
  • NULL
  • chain traversal
運作原理：
從 first 開始，透過 current = current->link 一個節點一個節點往後走，直到 current == NULL。插入與刪除時要小心保存下一個節點位置，避免斷鏈。

複雜度分析：
走訪整條 chain 為 O(n)。若已知插入位置前一節點，插入為 O(1)；若要先搜尋位置，總時間為 O(n)。空間為 O(n)。

優缺點：
  • 優點：結構簡單，適合理解 linked list 的所有 pointer 操作。
  • 缺點：只能單方向前進，無法直接回到前一節點。
老師上課重點：
老師重點通常在 first、link、NULL 的角色，以及每次 current 移動後指到哪裡。

期末考重點：
常考畫出節點與 link、trace traversal，以及刪除節點時 link 更新順序。
非常詳細動畫說明：
動畫目的：說明單向 linked chain 的節點如何由 data 與 link 組成。Frame 1 建立 head/first；Frame 2 加入第一個節點；Frame 3 顯示 link 指向下一節點；Frame 4 到 Frame 6 逐步形成 EAT → FAT → HAT → NULL；Frame 7 強調最後節點 link 為 NULL；Frame 8 完成整條 chain。觀察重點是每個節點只有一個向後 link，因此只能由 first 往後走，不能直接往回。
C++ 程式範例：
#include <iostream>
using namespace std;

class ThreeLetterChain;
class ThreeLetterNode {
    friend class ThreeLetterChain;
private:
    char data[4];
    ThreeLetterNode* link;
public:
    ThreeLetterNode(const char* s = "", ThreeLetterNode* next = 0) : link(next) {
        data[0]=s[0]; data[1]=s[1]; data[2]=s[2]; data[3]='\0';
    }
};

class ThreeLetterChain {
public:
    ThreeLetterChain() : first(0) {}
    void InsertFront(const char* s) { first = new ThreeLetterNode(s, first); }
    void Print() const {
        for (ThreeLetterNode* p = first; p; p = p->link) cout << p->data << ' ';
        cout << '\n';
    }
private:
    ThreeLetterNode* first;
};

int main(){ ThreeLetterChain c; c.InsertFront("WAT"); c.InsertFront("CAT"); c.InsertFront("BAT"); c.Print(); }
輸出結果：
EAT -> FAT -> HAT -> NULL
函式說明：
printChain(first) 從 first 開始沿 link 走訪並輸出節點。

變數說明：
first 指向第一個節點；current 是走訪指標；data 是節點資料；link 指向下一節點。

Node Class and Has-a Relation
非常詳細主題說明：
節點分成 data 與 link；Chain 物件 has-a first pointer；說明封裝與節點關係。本主題屬於 Linked Lists 章節，重點在理解資料不一定連續放在記憶體中，而是透過 link/pointer 將節點串起來。考試常要求畫出操作前後的指標變化、判斷時間複雜度，以及說明 first、last、header、front、rear、iterator 等角色。
主題背景：
本主題說明如何用 C++ class 定義節點，以及 linked list 和 node 之間的 has-a 關係。Node 不是完整串列，而是串列的組成單位。

核心概念：
  • class Node
  • data member
  • link pointer
  • has-a relation
  • encapsulation
  • friend / private
運作原理：
節點類別通常包含資料欄位與指向下一節點的指標。List 類別透過 first 指標持有多個 Node，因此 List has Node。良好設計會把節點細節封裝起來，避免外部任意破壞 link。

複雜度分析：
Node 建立為 O(1)，串列整體空間為節點數 O(n)。封裝不改變漸近複雜度，但會提高程式安全性。

優缺點：
  • 優點：類別設計清楚，能把資料與指標封裝成可重用結構。
  • 缺點：若權限或建構子設計不好，容易造成外部無法操作或過度暴露細節。
老師上課重點：
老師可能會問 data 與 link 是 Node 的欄位，而 first 是 List 的欄位。

期末考重點：
常考 class 宣告、private/public、Node* link 的意義，以及 has-a 和 is-a 差異。
非常詳細動畫說明：
動畫目的：說明 C++ node class 的欄位設計與 has-a 關係。Frame 1 顯示 Node 物件；Frame 2 將 data 欄位標出；Frame 3 將 link pointer 標出；Frame 4 說明 node has data、has link；Frame 5 到 Frame 8 展示多個 Node 物件如何透過 link 形成結構。重點是理解 class 不是只有資料，也包含指向其他物件的關係。
C++ 程式範例：
#include <iostream>
using namespace std;

class PC { public: string processor="CPU", memory="RAM", disk="SSD"; };
class hi_tech_firm { public: string CEO="golf"; string Manager[2]={"hi","pay"}; };

int main(){
    PC pc;
    hi_tech_firm firm;
    cout << "PC HAS-A " << pc.processor << ", " << pc.memory << ", " << pc.disk << '\n';
    cout << "Firm HAS-A CEO: " << firm.CEO << '\n';
}
輸出結果：
10 20 30
函式說明：
insertFront(value) 在串列前端新增節點；print() 走訪 Chain。

變數說明：
first 是 Chain 持有的開頭指標；newNode 是新配置節點；data/link 是 Node 欄位。

Pointer Manipulation
非常詳細主題說明：
先保存 next；把新節點指向後方；再讓前節點指向新節點。本主題屬於 Linked Lists 章節，重點在理解資料不一定連續放在記憶體中，而是透過 link/pointer 將節點串起來。考試常要求畫出操作前後的指標變化、判斷時間複雜度，以及說明 first、last、header、front、rear、iterator 等角色。
主題背景：
Pointer Manipulation 是 linked list 最容易出錯也最常考的部分。插入、刪除、串接、反轉都依賴正確的 pointer 更新順序。

核心概念：
  • pointer assignment
  • temporary pointer
  • prev
  • current
  • next
  • memory safety
運作原理：
修改 link 前通常要先保存會被覆蓋的位址。例如刪除 current 時，要先讓 prev->link 指向 current->link，再釋放 current。若順序錯誤，可能遺失整段串列。

複雜度分析：
單次 pointer 指派為 O(1)，但找到目標位置通常需要 O(n)。

優缺點：
  • 優點：更新 pointer 可以快速完成結構調整，不需要搬移資料。
  • 缺點：容易出現 lost node、dangling pointer、memory leak 等錯誤。
老師上課重點：
老師常要求一步一步畫出 pointer 改變，而不是只寫最後結果。

期末考重點：
常考插入與刪除的正確語句順序，尤其是 temp 指標何時需要保存。
非常詳細動畫說明：
動畫目的：示範 linked list 操作最容易出錯的 pointer 改接。Frame 1 顯示原始 prev → current → next；Frame 2 建立 newNode；Frame 3 先令 newNode->link 指向 current 或 next；Frame 4 再令 prev->link 指向 newNode；Frame 5 到 Frame 8 檢查是否有節點遺失。重點是 pointer 更新順序，若先覆蓋 prev->link，可能失去後續節點位址。
C++ 程式範例：
#include <iostream>
using namespace std;

struct Node { int data; Node* link; Node(int d=0, Node* l=0):data(d),link(l){} };

int main(){
    Node* y = new Node(20,0);
    Node* x = new Node(10,0);
    cout << "before x = y: x=" << x->data << ", y=" << y->data << '\n';
    x = y;
    cout << "after  x = y: x=" << x->data << ", y=" << y->data << '\n';
    Node* a = new Node(1,0); Node* b = new Node(2,0);
    *a = *b;
    cout << "after *a = *b: a=" << a->data << ", b=" << b->data << '\n';
}
輸出結果：
10 20 30
函式說明：
insertAfter(prev,newNode) 在 prev 後方插入新節點。

變數說明：
prev 是插入位置前一節點；newNode 是新節點；link 保存節點連接。

Create Insert Delete Chain
非常詳細主題說明：
從空串列建立；插入新節點；刪除時先接線再釋放。本主題屬於 Linked Lists 章節，重點在理解資料不一定連續放在記憶體中，而是透過 link/pointer 將節點串起來。考試常要求畫出操作前後的指標變化、判斷時間複雜度，以及說明 first、last、header、front、rear、iterator 等角色。
主題背景：
建立、插入、刪除是 singly linked list 的基本操作。這個主題把節點配置、資料填入與 pointer 接回串列結合起來。

核心概念：
  • new node
  • insert after
  • delete after
  • first
  • prev
  • current
  • delete
運作原理：
建立節點時配置記憶體並設定 data。插入時讓新節點先指向原本下一個節點，再讓前一節點指向新節點。刪除時先繞過目標節點，再釋放記憶體。

複雜度分析：
若已知位置，插入刪除為 O(1)；若必須搜尋值或位置，為 O(n)。空間依節點數為 O(n)。

優缺點：
  • 優點：能動態增減資料量，不需事先固定容量。
  • 缺點：必須手動處理記憶體與指標，容易出現錯誤。
老師上課重點：
老師會強調插入順序不能反，刪除前要保留欲刪節點位址。

期末考重點：
常考給一段程式碼判斷刪除後串列結果，或補上缺少的 pointer 語句。
非常詳細動畫說明：
動畫目的：完整示範建立、插入與刪除 chain。Frame 1 是空串列；Frame 2 建立第一個節點；Frame 3 到 Frame 4 追加節點；Frame 5 插入新節點；Frame 6 顯示刪除前需找到前一節點；Frame 7 改接 link 跳過刪除節點；Frame 8 完成並釋放或忽略被刪節點。考試要能畫出每一步指標變化。
C++ 程式範例：
#include <iostream>
using namespace std;

template <class T> class Chain;
template <class T>
class ChainNode {
    friend class Chain<T>;
public:
    ChainNode(const T& e=T(), ChainNode<T>* n=0):data(e),link(n){}
private:
    T data;
    ChainNode<T>* link;
};

template <class T>
class Chain {
public:
    Chain(){ first = 0; }
    void InsertFront(const T& e){ first = new ChainNode<T>(e, first); }
    void Print() const { for(auto p=first;p;p=p->link) cout << p->data << ' '; cout << '\n'; }
private:
    ChainNode<T>* first;
};

int main(){ Chain<int> intlist; intlist.InsertFront(20); intlist.InsertFront(10); intlist.Print(); }
輸出結果：
10 30
函式說明：
deleteAfter(prev) 刪除 prev 後面的節點。

變數說明：
prev 是欲刪節點前一個；target 是要刪除的節點；first 是串列開頭。

Template Chain Iterator
非常詳細主題說明：
泛型 T 存放資料；iterator 指向目前節點；++ 代表沿 link 移動。本主題屬於 Linked Lists 章節，重點在理解資料不一定連續放在記憶體中，而是透過 link/pointer 將節點串起來。考試常要求畫出操作前後的指標變化、判斷時間複雜度，以及說明 first、last、header、front、rear、iterator 等角色。
主題背景：
Template Chain Iterator 把 linked list 擴充為可處理多種資料型態，並用 iterator 提供走訪介面。這也是 C++ STL list 與 iterator 概念的前置基礎。

核心概念：
  • template<class T>
  • iterator
  • operator*
  • operator++
  • generic programming
  • current pointer
運作原理：
Template 讓 Node<T> 可存 int、string 或自訂型態。Iterator 內部保存 current pointer，operator* 回傳目前資料，operator++ 讓 current 前進到下一節點。

複雜度分析：
iterator 前進一步為 O(1)，走訪全部為 O(n)。Template 不改變時間複雜度，只增加泛型能力。

優缺點：
  • 優點：程式可重用，串列不再限制單一資料型態。
  • 缺點：語法較複雜，iterator 若指到 NULL 再解參考會錯誤。
老師上課重點：
老師常把 template 和 iterator 與 STL 概念連在一起。

期末考重點：
常考 iterator 如何移動、*it 代表什麼，以及 template 參數 T 的意義。
非常詳細動畫說明：
動畫目的：說明 template chain 與 iterator 如何把節點走訪泛型化。Frame 1 顯示 Chain<T>；Frame 2 顯示 first；Frame 3 顯示 iterator 指向第一個節點；Frame 4 到 Frame 7 iterator 逐步移動到下一個節點；Frame 8 到達 NULL 停止。重點是 iterator 封裝 pointer 移動，讓使用者像走訪容器一樣讀取 linked list。
C++ 程式範例：
#include <iostream>
#include <numeric>
using namespace std;

template <class T> class Chain;
template <class T>
class ChainNode { friend class Chain<T>; T data; ChainNode<T>* link; public: ChainNode(T e=T(), ChainNode<T>* l=0):data(e),link(l){} };

template <class T>
class Chain {
public:
    class ChainIterator {
    public:
        ChainIterator(ChainNode<T>* startNode=0){ current=startNode; }
        T& operator*() const { return current->data; }
        ChainIterator& operator++(){ current=current->link; return *this; }
        bool operator!=(const ChainIterator right) const { return current != right.current; }
    private: ChainNode<T>* current;
    };
    Chain(): first(0), last(0) {}
    void InsertBack(const T& e){ if(first){last->link=new ChainNode<T>(e); last=last->link;} else first=last=new ChainNode<T>(e); }
    ChainIterator begin(){ return ChainIterator(first); }
    ChainIterator end(){ return ChainIterator(0); }
private:
    ChainNode<T>* first; ChainNode<T>* last;
};

int main(){ Chain<int> y; y.InsertBack(1); y.InsertBack(2); y.InsertBack(3); int sum=0; for(auto it=y.begin(); it!=y.end(); ++it) sum += *it; cout << sum << '\n'; }
輸出結果：
A B C
函式說明：
printChain<T>(first) 使用 template 走訪不同型態的 chain。

變數說明：
T 是資料型態參數；first 是串列開頭；p 是走訪指標。

  • concatenate
  • reverse
  • prev/current/next
  • linear traversal
運作原理：
InsertBack 若有 last 指標可 O(1) 接到尾端。Concatenate 將第一條串列尾端指向第二條開頭。Reverse 則逐步將每個節點 link 反向，通常需要 prev、current、next 三個指標。

複雜度分析：
InsertBack 有 last 為 O(1)，沒有 last 需 O(n)。Concatenate 若已知尾端為 O(1)，否則 O(n)。Reverse 為 O(n)，空間 O(1)。

優缺點：
  • 優點：能有效操作整條串列，特別是 reverse 可原地完成。
  • 缺點：反轉時若沒有先保存 next，容易斷掉尚未處理的節點。
老師上課重點：
老師常要求畫出 reverse 每一步 prev、current、next 的位置。

期末考重點：
常考反轉後的 link 方向、concatenate 後 first/last 應如何更新。
非常詳細動畫說明：
動畫目的：把三個常見操作放在同一個 linked list 情境中。Frame 1 顯示原串列；Frame 2 InsertBack 找到尾端；Frame 3 將最後節點 link 指向新節點；Frame 4 Concatenate 將第一條串列尾端接到第二條串列 first；Frame 5 到 Frame 8 Reverse 逐步把 link 方向反轉。觀察重點是 last、first、prev、cur、next 的角色。
C++ 程式範例：
#include <iostream>
using namespace std;

template <class T> class Chain;
template <class T>
class ChainNode { friend class Chain<T>; public: ChainNode(T e=T(), ChainNode<T>* l=0):data(e),link(l){} private: T data; ChainNode<T>* link; };

template <class T>
class Chain {
public:
    Chain(): first(0), last(0) {}
    void InsertBack(const T& e){ if(first){ last->link = new ChainNode<T>(e); last = last->link; } else first = last = new ChainNode<T>(e); }
    void Concatenate(Chain<T>& b){ if(first){ last->link=b.first; last=b.last; } else { first=b.first; last=b.last; } b.first=b.last=0; }
    void Reverse(){ ChainNode<T>* current=first; ChainNode<T>* previous=0; last=first; while(current){ ChainNode<T>* r=previous; previous=current; current=current->link; previous->link=r; } first=previous; }
    void Print(){ for(auto p=first;p;p=p->link) cout << p->data << ' '; cout << '\n'; }
private: ChainNode<T>* first; ChainNode<T>* last;
};

int main(){ Chain<int> a,b; a.InsertBack(1); a.InsertBack(2); b.InsertBack(3); a.Concatenate(b); a.Reverse(); a.Print(); }
輸出結果：
3 2 1
函式說明：
reverse(first) 使用 prev/current/next 原地反轉串列。

變數說明：
prev 指向已反轉部分；current 指向目前節點；next 保存尚未處理部分。

非常詳細主題說明：
last 指向尾節點；last->link 回到第一個節點；用 last 可快速找到頭尾。本主題屬於 Linked Lists 章節，重點在理解資料不一定連續放在記憶體中，而是透過 link/pointer 將節點串起來。考試常要求畫出操作前後的指標變化、判斷時間複雜度，以及說明 first、last、header、front、rear、iterator 等角色。
主題背景：
Circular List 讓最後節點 link 回到第一個節點，而不是 NULL。若只保存 last 指標，就能透過 last->link 找到 first。

核心概念：
  • circular list
  • last pointer
  • last->link as first
  • no NULL end
  • cycle traversal
運作原理：
在 circular list 中，last->link 指向第一個節點。插入開頭或尾端時，要同時維護 last 與 last->link 的正確關係。走訪時不能用 NULL 停止，必須回到起點時停止。

複雜度分析：

優缺點：
  • 優點：從 last 可快速找到 first，適合輪流處理或環狀資料。
  • 缺點：走訪停止條件較難，若寫錯可能無限迴圈。
老師上課重點：
老師常問為什麼 circular list 可以只用 last 表示整條串列。

期末考重點：
常考 last->link 指向誰，以及插入第一個節點與一般節點時的差異。
非常詳細動畫說明：
動畫目的：說明 circular list 使用 last pointer 的好處。Frame 1 顯示線性串列尾端指向 NULL；Frame 2 將尾端 link 改成指回 first；Frame 3 標示 last；Frame 4 顯示 first 可由 last->link 取得；Frame 5 到 Frame 8 展示插入尾端與走訪一圈後回到起點。重點是 circular list 沒有 NULL 結尾，停止條件要改成是否回到起始節點。
C++ 程式範例：
#include <iostream>
using namespace std;

template <class T> struct ChainNode { T data; ChainNode<T>* link; ChainNode(T e=T()):data(e),link(0){} };
template <class T>
class CircularList {
public:
    CircularList(): last(0) {}

printCircular(last) 從 last->link 開始輸出 circular list。

變數說明：
last 指向最後節點；first = last->link；p 是走訪指標。

Linked Stack and Queue
非常詳細主題說明：
Stack 在 top push/pop；Queue 在 rear 插入、front 刪除；比較 LIFO 與 FIFO。本主題屬於 Linked Lists 章節，重點在理解資料不一定連續放在記憶體中，而是透過 link/pointer 將節點串起來。考試常要求畫出操作前後的指標變化、判斷時間複雜度，以及說明 first、last、header、front、rear、iterator 等角色。
主題背景：
Stack 與 Queue 可用 linked list 實作。Stack 採 LIFO，通常在 top 插入與刪除；Queue 採 FIFO，通常用 front 與 rear 管理兩端。

核心概念：
  • stack
  • queue
  • LIFO
  • FIFO
  • top
  • front
  • rear
  • push/pop/enqueue/dequeue
運作原理：
Linked stack 的 push 在 top 前端插入，pop 刪除 top。Linked queue 的 enqueue 接到 rear 後方，dequeue 從 front 移除，空佇列時 front 與 rear 都要正確更新。

複雜度分析：
若維護 top/front/rear，push/pop/enqueue/dequeue 都可 O(1)。空間為 O(n)。

優缺點：
  • 優點：容量可動態成長，不需固定陣列大小。
  • 缺點：每個元素有額外 pointer 成本，且要小心空結構操作。
老師上課重點：
老師常比較 stack 與 queue 操作端點不同。

期末考重點：
常考操作序列後 top/front/rear 指向誰，以及空佇列刪除最後一個元素後如何更新。
非常詳細動畫說明：
動畫目的：比較 linked stack 與 linked queue 的操作端點。Frame 1 顯示 stack top；Frame 2 push 新節點到 top；Frame 3 pop 移除 top；Frame 4 顯示 queue front/rear；Frame 5 enqueue 接到 rear；Frame 6 dequeue 從 front 移除；Frame 7 到 Frame 8 比較 LIFO 與 FIFO。考試常考 top、front、rear 更新是否正確。
C++ 程式範例：
#include <iostream>
#include <stdexcept>
using namespace std;

template <class T> struct ChainNode { T data; ChainNode<T>* link; ChainNode(T e=T(), ChainNode<T>* l=0):data(e),link(l){} };
template <class T>
class LinkedStack {
public:
    LinkedStack(){ top=0; }
    bool IsEmpty() const { return top==0; }
    void Push(const T& e){ top = new ChainNode<T>(e, top); }
    void Pop(){ if(IsEmpty()) throw runtime_error("Stack is empty. Cannot delete."); ChainNode<T>* delNode=top; top=top->link; delete delNode; }
    T Top() const { return top->data; }
private: ChainNode<T>* top;
};
int main(){ LinkedStack<int> s; s.Push(10); s.Push(20); cout << s.Top() << '\n'; s.Pop(); cout << s.Top() << '\n'; }

Stack pop: 20
Queue pop: 10
函式說明：
main() 使用 STL stack 與 queue 示範 LIFO/FIFO。

變數說明：
st 是 stack；q 是 queue；top/front 分別代表可刪除端。

Available Space List
非常詳細主題說明：
刪除節點進入 av list；new node 可從 av list 取回；降低反覆配置成本。本主題屬於 Linked Lists 章節，重點在理解資料不一定連續放在記憶體中，而是透過 link/pointer 將節點串起來。考試常要求畫出操作前後的指標變化、判斷時間複雜度，以及說明 first、last、header、front、rear、iterator 等角色。
主題背景：
Available Space List 是管理可用節點的 free list。當程式頻繁配置與釋放節點時，可把刪除的節點收回可用串列以便再次利用。

  • reuse node
  • new/delete cost
  • memory management

刪除節點時不一定立刻交還系統，而是放入 available list。需要新節點時先從 available list 取出，若沒有可用節點再配置新記憶體。

老師可能會把 available list 與多項式、稀疏矩陣節點回收連結。

Polynomial Addition
非常詳細主題說明：
比較兩多項式指標；相同 exponent 相加；不同 exponent 複製較大項。本主題屬於 Linked Lists 章節，重點在理解資料不一定連續放在記憶體中，而是透過 link/pointer 將節點串起來。考試常要求畫出操作前後的指標變化、判斷時間複雜度，以及說明 first、last、header、front、rear、iterator 等角色。
主題背景：
Polynomial Addition 是 linked list 的經典應用。兩個已依 exponent 排序的多項式可以像 merge 一樣相加。

核心概念：
  • two polynomial chains
  • same exponent combine
  • merge-like scan
  • zero coefficient removal
  • term comparison
運作原理：
用兩個指標分別掃描 A 與 B。若 exponent 相同，就係數相加；若一邊 exponent 較大，就把該項複製到結果。若相加係數為 0，該項不放入結果。

複雜度分析：
若 A 有 m 項、B 有 n 項，時間 O(m+n)，結果空間最多 O(m+n)。

優缺點：
  • 優點：效率高，能利用已排序 exponent 線性完成。
  • 缺點：若原多項式未排序，必須先排序或插入時維持排序。
老師上課重點：
老師常用 polynomial addition 測試 merge 思維與 linked list 建立新節點。

期末考重點：
常考同次方合併、係數為 0 時刪除，以及最後結果 chain。
非常詳細動畫說明：
動畫目的：示範兩個 polynomial chain 如何相加。Frame 1 顯示 A 與 B 兩條多項式；Frame 2 使用兩個 pointer 指向目前項；Frame 3 比較 exponent；Frame 4 exponent 相同則係數相加；Frame 5 exponent 較大者先輸出；Frame 6 到 Frame 7 移動 pointer；Frame 8 得到結果多項式。重點是類似 merge 的雙指標流程。
C++ 程式範例：
#include <iostream>
#include <vector>
using namespace std;

struct Term { int coef; int exp; Term Set(int c,int e){coef=c;exp=e;return *this;} };
class Polynomial {
public:
    vector<Term> poly;
    Polynomial operator+(const Polynomial& b) const {
        Term temp; Polynomial c; size_t ai=0, bi=0;
        while (ai < poly.size() && bi < b.poly.size()) {
            if (poly[ai].exp == b.poly[bi].exp) { int sum=poly[ai].coef+b.poly[bi].coef; if(sum) c.poly.push_back(temp.Set(sum, poly[ai].exp)); ai++; bi++; }
            else if (poly[ai].exp < b.poly[bi].exp) { c.poly.push_back(b.poly[bi++]); }
            else { c.poly.push_back(poly[ai++]); }
        }
        while (ai < poly.size()) c.poly.push_back(poly[ai++]);
        while (bi < b.poly.size()) c.poly.push_back(b.poly[bi++]);
        return c;
    }
};
int main(){ Polynomial a,b; Term t; a.poly={t.Set(3,14),t.Set(2,8),t.Set(1,0)}; b.poly={t.Set(8,14),t.Set(-3,10),t.Set(10,6)}; Polynomial c=a+b; for(auto &x:c.poly) cout<<x.coef<<"x^"<<x.exp<<' '; }
輸出結果：
8x^2 + 5x^0
函式說明：
printPoly(p) 輸出結果；main() 示範相同指數項合併。

變數說明：
a、b 是兩個多項式；result 是相加結果；coef/exp 是項目資料。

Sparse Matrix Orthogonal List
非常詳細主題說明：
非零元素用節點表示；row link 串同列；col link 串同欄。本主題屬於 Linked Lists 章節，重點在理解資料不一定連續放在記憶體中，而是透過 link/pointer 將節點串起來。考試常要求畫出操作前後的指標變化、判斷時間複雜度，以及說明 first、last、header、front、rear、iterator 等角色。
主題背景：
Sparse Matrix Orthogonal List 用節點表示非零元素，並同時用 right 與 down 指標連結同行與同列。這是 linked list 在二維資料上的進階應用。

核心概念：
  • sparse matrix
  • orthogonal list
  • row header
  • column header
  • right pointer
  • down pointer
  • nonzero element
運作原理：
每個非零節點包含 row、col、value、right、down。right 連到同一列下一個非零元素，down 連到同一欄下一個非零元素。row header 與 column header 幫助快速進入某列或某欄。

複雜度分析：
空間與非零元素數量 nz 成正比。走訪所有非零元素為 O(nz)。若搜尋特定列或欄，成本取決於該列或欄非零數。

優缺點：
  • 優點：非常適合稀疏矩陣，節省大量零元素空間。
  • 缺點：結構比單鏈結串列複雜，需要同時維護 row 與 column 方向。
老師上課重點：
老師常要求學生分清楚 right 與 down 指標用途。

期末考重點：
常考由矩陣畫出 orthogonal list，或新增非零元素時兩個方向如何連接。
非常詳細動畫說明：
動畫目的：說明 sparse matrix 的 orthogonal list 同時有 row link 與 column link。Frame 1 顯示稀疏矩陣；Frame 2 建立非零元素節點；Frame 3 標示 row、col、value；Frame 4 加入 right link；Frame 5 加入 down link；Frame 6 到 Frame 8 顯示 row header 與 column header 如何連接所有非零元素。重點是同一個節點同時屬於某一列與某一行。
C++ 程式範例：
#include <iostream>
using namespace std;

struct MatrixNode {
    bool head;
    int row, col, value;
    MatrixNode *down, *right, *next;
    MatrixNode(): head(true), row(0), col(0), value(0), down(0), right(0), next(0) {}
};
int main(){ MatrixNode H; H.row=5; H.col=4; H.value=6; cout << "Sparse matrix: " << H.row << "x" << H.col << ", nonzero=" << H.value << '\n'; }
輸出結果：
Row 0: (0,1,5) (0,3,8)
函式說明：
main() 建立非零矩陣節點並以 right 指標走訪同一列。

變數說明：
row/col/value 是非零元素位置和值；right 連同列；down 連同欄。

Sparse Matrix Input and Destructor
非常詳細主題說明：
讀入 row col value；建立 row/col header；destructor 逐列釋放節點。本主題屬於 Linked Lists 章節，重點在理解資料不一定連續放在記憶體中，而是透過 link/pointer 將節點串起來。考試常要求畫出操作前後的指標變化、判斷時間複雜度，以及說明 first、last、header、front、rear、iterator 等角色。
主題背景：
本主題處理稀疏矩陣的輸入建立與 destructor 釋放。動態配置多個節點後，必須正確釋放，避免 memory leak。

核心概念：
  • input triples
  • row/col/value
  • dynamic allocation
  • destructor
  • delete nodes
  • memory leak
運作原理：
輸入通常以三元組 row、col、value 表示非零元素。建立時把節點插入對應 row 與 column chain。Destructor 需逐列或逐節點釋放所有配置的節點與 header。

複雜度分析：
建立成本與非零元素數 nz 及插入搜尋成本相關。釋放所有節點為 O(nz)。

優缺點：
  • 優點：能完整管理稀疏矩陣生命週期，符合 C++ 資源管理要求。
  • 缺點：釋放順序若錯誤可能重複 delete 或遺失節點。
老師上課重點：
老師可能會強調有 new 就要有對應 delete，尤其在複雜 linked structure 中。

期末考重點：
常考 destructor 為何需要逐節點釋放，以及 input triple 如何變成節點。
非常詳細動畫說明：
動畫目的：說明 sparse matrix 從輸入建立到 destructor 釋放記憶體。Frame 1 顯示輸入三元組；Frame 2 建立 header nodes；Frame 3 到 Frame 5 插入非零元素；Frame 6 顯示 row/column links；Frame 7 destructor 逐列釋放節點；Frame 8 完成清除。重點是動態配置結構必須正確釋放，避免 memory leak。
C++ 程式範例：
#include <iostream>
using namespace std;

template <class T> struct ChainNode{T data; ChainNode<T>* link; ChainNode(T e=T(),ChainNode<T>*l=0):data(e),link(l){}};
template <class T> class Chain{ public: Chain():first(0){} ~Chain(){ ChainNode<T>* next; for(; first; first=next){ next=first->link; delete first; } } void InsertFront(T e){ first=new ChainNode<T>(e,first);} private: ChainNode<T>* first; };
int main(){ Chain<int> c; c.InsertFront(1); c.InsertFront(2); cout << "destructor frees nodes\n"; }
輸出結果：
建立非零節點數: 2
destructor 釋放完成
函式說明：
destroy(first) 逐節點 delete，避免 memory leak。

變數說明：
nodes 保存配置節點；first 是釋放起點；next 保存下一個待釋放節點。

Doubly Linked List
非常詳細主題說明：
每節點有 left/right link；可雙向走訪；插入刪除需更新四條 link。本主題屬於 Linked Lists 章節，重點在理解資料不一定連續放在記憶體中，而是透過 link/pointer 將節點串起來。考試常要求畫出操作前後的指標變化、判斷時間複雜度，以及說明 first、last、header、front、rear、iterator 等角色。
主題背景：
Doubly Linked List 每個節點同時有 left/right 或 prev/next 指標，可以雙向走訪。它改善 singly linked list 無法直接回到前一節點的限制。

核心概念：
  • doubly linked list
  • prev
  • next
  • left link
  • right link
  • bidirectional traversal
運作原理：
每個節點的 next 指向後一節點，prev 指向前一節點。插入或刪除時必須同時更新前後兩邊指標，確保雙向關係一致。

複雜度分析：
已知節點時插入刪除為 O(1)，走訪為 O(n)。每個節點需要兩個 pointer，空間常數較高。

優缺點：
  • 優點：可以向前與向後走訪，刪除已知節點更方便。
  • 缺點：每次更新要處理兩個方向，指標錯誤機會增加。
老師上課重點：
老師常比較 singly 與 doubly linked list 的刪除差異。

期末考重點：
常考插入節點時四條 pointer 的更新順序，以及刪除節點後 prev/next 是否一致。
非常詳細動畫說明：
動畫目的：說明 doubly linked list 每個節點同時具有 llink 與 rlink。Frame 1 顯示單向串列；Frame 2 加入 backward link；Frame 3 插入新節點時設定 new->llink 與 new->rlink；Frame 4 更新前後節點的 link；Frame 5 到 Frame 7 示範刪除節點時前後改接；Frame 8 完成雙向走訪。重點是插入刪除要同時維護兩個方向。
C++ 程式範例：
#include <iostream>
using namespace std;

template <class T> struct DoubleNode { T data; DoubleNode<T>* left; DoubleNode<T>* right; DoubleNode(T e=T()):data(e),left(0),right(0){} };
int main(){ DoubleNode<int> a(1), b(2); a.right=&b; b.left=&a; cout << a.data << " <-> " << b.data << '\n'; }
輸出結果：
Forward: 10 20 30 
Backward: 30 20 10
函式說明：
main() 示範雙向連結與 forward/backward traversal。

變數說明：
prev 指向前一節點；next 指向下一節點；a/b/c 是三個節點。
