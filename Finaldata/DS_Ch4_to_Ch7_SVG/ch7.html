Merge sort 使用 divide and conquer：先把陣列切成左右兩半，各自排序後再 merge。Merge 時若相等元素先取左邊，排序就是 stable。時間複雜度穩定為 O(n log n)，但需要額外陣列空間 O(n)。投影片屬於比較排序，考試可能要求畫分割樹或合併過程。

正確：quick 核心是 partition。

詳細中文解說

Quick sort 選一個 pivot，把小於等於 pivot 的放左邊，大於 pivot 的放右邊，再遞迴處理左右子陣列。平均時間 O(n log n)，但 pivot 選不好時最壞 O(n^2)。它通常是 in-place，但不穩定。考試重點是 partition 後 pivot 必須在最終位置，左右兩側只保證相對於 pivot 的大小關係，不保證已完全排序。

Chapter 7 SortingHeap Sort

多步驟 SVG 分解講解

Step 1：陣列視為 complete binary tree
正確：heap sort 使用 heap 結構。
Step 2：Build max heap
正確：max heap root 最大。
Step 3：root 與最後一格交換
正確：最大值固定到右端。
Step 4：heap size 減一並 down-heap
正確：down-heap 恢復 heap order。

正確：O(n log n)，通常不穩定。

詳細中文解說

Heap sort 先把陣列建成 max heap，然後反覆把 heap[1] 最大值與最後一個元素交換，縮小 heapSize，再 Adjust root。投影片的 HeapSort/Adjust 常使用 1-based array，heap[0] 不用。本程式保留這個結構。Heap sort 是 in-place，時間 O(n log n)，但通常不 stable。

Chapter 7 SortingRadix Sort

多步驟 SVG 分解講解

Step 1：準備固定長度 key
正確：LSD radix 從最低位開始。
Step 2：從 LSD 開始分桶
正確：每位需穩定分配。
Step 3：依桶順序穩定收集
正確：收集順序影響下一輪。
Step 4：處理下一位數
正確：穩定性是關鍵。
Step 5：所有位數完成
正確：radix 不是 comparison sort。

詳細中文解說

Radix sort 不是比較排序。LSD Radix Sort 從 least significant digit 開始，一趟一趟用 stable sort 依某一位分桶，最後得到字典序或數值序。2025 期末題直接考 fixed-length strings 與 LSD Radix Sort，因此一定要會從最後一個字元排到第一個字元。穩定性是核心：如果每一趟不 stable，前一位已經建立的相對順序會被破壞。時間複雜度若長度 k、字母範圍 r、資料量 n，用 counting sort 約 O(k(n+r))。
