詳細中文解說

樹是階層式資料結構。Root 是最上層節點；degree 是某節點子樹數；leaf 是 degree 0 的節點；parent/children 表示上下層關係；siblings 有同一個 parent；level 在投影片中 root 從 1 開始；height 或 depth 是最大 level。這些名詞是樹章節所有題目的基礎，包含 parenthesization、level-order、forest 轉 binary tree、heap 與 BST。答題時先找 root，再看每個節點的 children，最後標出 leaf 與 level。

多步驟 SVG 分解講解

Step 1：Traversal 由 root 與左右子樹定義
正確：二元樹走訪是遞迴定義。
Step 2：Preorder：Root → Left → Right
正確：preorder 先輸出 root。
Step 3：Inorder：Left → Root → Right
正確：inorder 在 BST 會得到排序結果。
Step 4：Postorder：Left → Right → Root
正確：postorder 常用於釋放或運算樹。
Step 5：Level-order：Queue 逐層輸出
正確：level-order 使用 FIFO queue。

詳細中文解說

Binary tree 每個節點最多兩個 children。Traversal 是最常考的題型：Preorder 是 root-left-right；Inorder 是 left-root-right；Postorder 是 left-right-root；Level-order 需要 queue。樹的遞迴定義自然對應遞迴程式，因此投影片常強調 recursive algorithms。考試時可以在每個節點旁寫拜訪順序，避免把「走過」和「輸出」混淆。

詳細中文解說

Threaded binary tree 把原本為 null 的 child 指標改用來指向 inorder predecessor 或 successor。leftThread/rightThread 用來判斷指標是真 child 還是 thread。它的目的不是改變樹的邏輯結構，而是讓 inorder traversal 可以不使用 stack 或 recursion。InsertRight 題目常要求更新新節點、原節點右子樹與 thread 的關係，因此要先分清楚 rightChild 是真右子樹還是 successor thread。

Chapter 5 TreesHeap / MaxPQ

多步驟 SVG 分解講解

Step 1：MaxPQ 刪除最高 priority
正確：PQ 對應 Top/Push/Pop。
Step 2：Heap 是 complete binary tree
正確：complete tree 可用 array。
Step 3：Max heap：parent ≥ children
正確：max heap 最大值在 root。
Step 4：Push 往上調整
正確：插入後 up-heap。
Step 5：Pop 往下調整
正確：刪除後 down-heap。

詳細中文解說

Priority queue 中每個元素有 priority，只能刪除最高或最低 priority 的元素。投影片 ADT MaxPQ 定義 IsEmpty、Top、Push、Pop；MaxHeap 則用 complete binary tree 的 array representation 實作，heap[0] 通常不用。Max heap 滿足 parent key >= children key，所以最大值一定在 root，也就是 heap[1]。Push 往上調整，Pop 用最後一個元素補 root 後向下調整。

Chapter 5 TreesBinary Search Tree

多步驟 SVG 分解講解

Step 1：左子樹 key 小於 root
正確：BST 性質遞迴成立。
Step 2：右子樹 key 大於 root
正確：右子樹都大於 root。
Step 3：Search 比較後往左或右
正確：搜尋每步只走一邊。
Step 4：Insert 到 null 位置
正確：插入不破壞 ordering。
Step 5：Delete 分三種情況
正確：two-child delete 常用 successor/predecessor。

詳細中文解說

BST 的規則是 left subtree key < root key < right subtree key。搜尋時每次只需要往左或往右走一邊，因此平均效率好；但若樹很偏斜，最壞可能退化成 O(n)。BST 的 inorder traversal 會得到遞增序列，這是檢查 BST 是否正確的重要方法。插入時從 root 開始比較，小於走左，大於走右，遇到 null 就建立新節點。刪除則分 leaf、one child、two children 三種情況。
