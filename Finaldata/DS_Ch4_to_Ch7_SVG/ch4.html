Chapter 4 Linked ListsArray vs. List

詳細中文解說

投影片先比較 sequential representation 與 linked representation。Array 的優點是可用 index 或位址直接找到第 i 個元素，因此查詢任意位置通常是 O(1)；缺點是若要在中間插入或刪除，後面的元素常需要整批搬移，所以成本是 O(n)。Linked list 的節點由 data 與 link pointer 組成，first 指到第一個節點，最後節點指到 0。只要已經找到插入或刪除位置，操作本身主要是改變 link，因此可做到 O(1)。這個主題在考試常搭配圖形，要求判斷插入 GAT 或刪除 GAT 時哪些指標改變。

Chapter 4 Linked ListsSingly Linked List / Chain

詳細中文解說

單向串列 Chain 是第四章的主體。投影片中的 ChainNode 常用 private data 與 link，並讓 Chain 成為 friend class。這樣外部使用者不能直接碰節點細節，但 Chain 的成員函式可以操作節點。first 是整條鏈的入口，若 first=0 表示空串列。插入節點時要先 new 一個節點，填入 data，再把新節點 link 指向原本位置，最後修改前一個節點或 first。刪除節點時要保留被刪節點前一個節點，先跳過被刪節點，再 delete。

Chapter 4 Linked ListsDoubly Linked List

詳細中文解說

雙向串列在每個節點中同時保存 llink 與 rlink。rlink 指向下一個節點，llink 指向前一個節點，因此比單向串列更容易向前走、刪除目前節點也比較方便。不過每次插入或刪除要更新兩個方向的指標，所以要特別小心順序。投影片常用 header node 或 circular doubly linked list 減少邊界情況；基本概念仍是 data、llink、rlink 三欄。考試若畫圖，要把左右兩條箭頭都畫清楚。

第四章也連到 C++ STL list 與 iterator。list 是雙向串列，因此支援 bidirectional iterator：可以 ++ 往後，也可以 -- 往前，但不能像 vector 一樣用 it+5 做 random access。iterator 的考點常是知道不同容器的走訪能力，以及 insert/erase 後哪些 iterator 可能失效。若教學網站要讓學生理解，最好把 iterator 畫成一個指向節點的游標，而不是單純數字 index。

Chapter 4 Linked ListsApplications: Polynomial by linked list

詳細中文解說

串列的應用之一是多項式表示。每個節點可以存 coef 與 exp，代表一項，例如 3x^4。若多項式很稀疏，用 linked list 只存非零項，比陣列存每個次方更節省空間。合併兩個多項式時，通常比較兩個串列目前節點的 exp：次方大的先輸出，次方相同則係數相加。這也呼應投影片對 Array vs Linked 的比較：陣列查詢快，但稀疏資料可能浪費；串列適合動態項目與稀疏資料。
