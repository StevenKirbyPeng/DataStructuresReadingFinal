必備背景知識

一、LSD Radix Sort 的基本概念

LSD Radix Sort 的核心觀念是「多輪、穩定、由低位到高位」排序。本題給的資料是固定長度三個字元的字串，例如 cat、bat、ant、car、bar、arc、cab，因此每一個字串都可以看成有三個欄位：第 0 位是最左邊字母、第 1 位是中間字母、第 2 位是最右邊字母。若要依照字典序由小到大排序，直覺上最重要的是最左邊字母，但 LSD 方法反而先從最右邊字母開始。

二、為什麼要從最右邊字元開始

Radix Sort 每一次排序只處理一個 digit 或一個 character。第一輪依照最後一個字元排完後，只能保證最後一位由小到大；第二輪依照中間字元排序時，如果兩個字串中間字元相同，穩定性會讓它們仍維持第一輪依最後字元排好的順序；第三輪依照第一個字元排序時，如果第一個字元相同，前兩輪建立的中間字元與最後字元順序也會被保留下來。因此三輪結束後，整個字串就會符合完整字典序。

三、ASCII order 與字典序

此題特別要求 assume ASCII order，因此比較字母時可使用 a < b < c < ... 的順序。因為題目中的字母都是小寫英文字母，不需要處理大小寫 ASCII 的額外差異。若題目混合大寫與小寫，ASCII 中大寫字母與小寫字母的編碼順序會影響排序結果；但本題全部是小寫英文字母，所以排序規則可直接視為一般英文字母順序。

四、Stable Sort 是本題關鍵

題目特別要求 stable sort at each pass，這是本題最重要的陷阱。如果使用不穩定排序，例如在同一個 bucket 中任意顛倒相同 key 的元素，前面 pass 所排好的低位資訊就會被破壞，最後結果可能不是字典序。以 bat 與 bar 為例，它們第一個字母都是 b，第二個字母都是 a，真正決定先後的是第三個字母 r 與 t；第一輪最後字元排序會讓 bar 在 bat 前面。到了第三輪依第一個字元排序時，如果 stable，就會保留 bar、bat 的相對順序；如果不 stable，可能變成 bat、bar，答案就錯。

五、Linked Representation 的意義

題目附的程式是 linked-list radix sort。它不一定直接搬動字串本身，而是利用 link 陣列串起元素順序。這種設計的意義是：當資料項目很大時，搬動整筆資料成本高；若只改變 link，便能降低資料移動成本。程式中的 front 與 rear 用來記錄每個 bucket 的頭尾，link 用來表示下一個元素的位置，current 用來沿著目前串列逐一掃描。每一輪 pass 會根據目前 digit 把元素分配到不同 bucket，再按照 bucket 順序串回一條新的 linked list。

六、時間複雜度與小題判斷

時間複雜度通常寫成 O(d(n+r))。其中 d 是 digit 數，本題三個字元所以 d=3；n 是資料筆數，本題 n=7；r 是 bucket 數，如果只看小寫英文字母可視為 26。每一輪需要掃過 n 筆資料並處理 r 個 bucket，因此總成本是 d 輪乘上每輪 n+r。解題時必須依題目要求列出每一 pass 的排序結果，而不是只寫最後答案；因為此題考的是 LSD Radix Sort 的過程、stable 的效果，以及 linked bucket 合併後的相對順序是否正確。

// 輸出目前 link 串列代表的排序結果

void printByLink(const vector<string>& a, const vector<int>& link, int first) {
    for (int cur = first; cur != 0; cur = link[cur]) {
        cout << a[cur] << (link[cur] ? " " : "");
    }

// LSD Radix Sort：依照題目給的 linked-list bucket 概念撰寫
// a    ：1-based 字串陣列，a[0] 不使用，符合題目 index 0 空著的精神
// link ：用來串接目前排序順序的陣列
// d    ：每個字串長度，本題 d=3
// r    ：bucket 數量，小寫英文字母可用 r=26
// n    ：資料筆數，本題 n=7
int RadixSort(vector<string>& a, vector<int>& link, const int d, const int r, const int n) {
    vector<int> e(r), f(r);       // f[k] 是 bucket k 的第一個節點，e[k] 是 bucket k 的最後節點
    int first = 1;                // 初始串列從第 1 筆資料開始

    // 初始化 link：1 -> 2 -> 3 -> ... -> n -> 0
    for (int i = 1; i < n; i++) link[i] = i + 1;
    link[n] = 0;

    // 從最低位，也就是最右邊字元 d-1，逐步往左做到 0
    for (int i = d - 1; i >= 0; i--) {
        fill(f.begin(), f.end(), 0); // 每一輪重新清空 bucket first
        fill(e.begin(), e.end(), 0); // 每一輪重新清空 bucket end

        // 依照目前串列順序掃描，這可保證同 bucket 內維持穩定性
        for (int current = first; current; current = link[current]) {
            int k = digit(a[current], i, r);     // 計算目前字元所屬 bucket
            if (f[k] == 0) f[k] = current;       // bucket 空：目前節點成為第一個
            else link[e[k]] = current;           // bucket 不空：接在原本尾端後面
            e[k] = current;                      // 更新 bucket 尾端
        }

        int j = 0;
        while (j < r && !f[j]) j++;             // 找到第一個非空 bucket
        first = f[j];                            // 新串列從最小 bucket 的第一個節點開始
        int last = e[j];                         // last 記錄目前已串好串列的尾端

        // 依 bucket 編號由小到大串接所有非空 bucket
        for (int k = j + 1; k < r; k++) {
            if (f[k]) {
                link[last] = f[k];               // 把下一個非空 bucket 接到尾端
                last = e[k];                     // 更新尾端
            }
        }
        link[last] = 0;                          // 串列結尾設為 0

        cout << "Pass " << (d - i) << " after sorting index " << i << ": ";
        printByLink(a, link, first);
    }
    return first;
}

int main() {
    vector<string> a = {"", "cat", "bat", "ant", "car", "bar", "arc", "cab"};
    vector<int> link(a.size(), 0);
    int first = RadixSort(a, link, 3, 26, 7);

    cout << "Final result: ";
    printByLink(a, link, first);
    return 0;
}

b. Time Complexity 推導

共有 k 個字元位置，所以 Radix Sort 要做 k 輪。每一輪若用 counting sort，處理 n 個字串要 O(n)，建立與掃描 r 個 bucket 要 O(r)，所以單輪是 O(n+r)。總共 k 輪，因此時間複雜度為 O(k(n+r))。若字元集大小 r 視為常數，例如固定 ASCII 或小寫英文字母，則可簡化為 O(kn)。

c. Stability 推導

LSD Radix Sort 必須穩定，因為低位數先排好的相對關係，要在高位數排序時被保留下來。若不穩定，例如 car 與 cab 在前面 pass 已經依照後面字元建立正確關係，但下一輪排序相同 key 時亂改相對順序，最後字典序就可能錯誤。因此最終答案是：LSD Radix Sort 本身依賴 stable sorting subroutine；若每輪 stable，整體才會正確。

必備背景知識

一、圖的儲存方式

圖的儲存方式是資料結構考試中很常見的比較題。一般 adjacency list 對有向圖很直覺：每個頂點有一條串列，串列中放從此頂點出發的邊。若圖是無向圖，一條邊 (u,v) 同時屬於 u 的鄰接關係，也屬於 v 的鄰接關係，因此在標準 adjacency list 中通常會存兩份節點：一份出現在 u 的串列中，另一份出現在 v 的串列中。

二、Adjacency List 的優點與限制

Adjacency list 的優點是查詢某個頂點的鄰居很方便，空間也通常比 adjacency matrix 節省，尤其適合邊數遠小於頂點平方的 sparse graph。但在無向圖中，若要刪除一條邊、標記一條邊是否已拜訪、或在演算法中維護邊的狀態，就會發生「同一條邊有兩個副本」的問題，必須同步更新兩邊，容易出錯。

三、Adjacency Multilist 的核心設計

Adjacency Multilist 就是為了解決無向圖邊重複儲存而設計的結構。它讓每條無向邊只建立一個 edge node，edge node 中通常會有兩個端點 ivex、jvex，以及兩個指標 ilink、jlink。ivex 與 jvex 表示此邊連接哪兩個頂點；ilink 連到下一條也 incident to ivex 的邊；jlink 連到下一條也 incident to jvex 的邊。換句話說，同一個 edge node 會同時掛在兩個頂點的邊串列上，但實體只存在一份。

四、本題圖形對應方式

題目給 V={A,B,C,D}，E={(A,B),(A,C),(B,C),(C,D)}，因此 multilist 需要四個 edge node，分別代表 e1=AB、e2=AC、e3=BC、e4=CD。若用頂點順序 A=0、B=1、C=2、D=3，可以讓 A 的 firstedge 指到 AB，再透過 A 端的 link 接到 AC；B 的 firstedge 可指到 AB，再接到 BC；C 的 firstedge 可指到 AC，再接到 BC，再接到 CD；D 的 firstedge 指到 CD。

五、畫圖時要注意的指標方向

實際指標名稱依 edge node 中端點欄位而定。畫 adjacency multilist 時最容易錯的地方，是把每條邊畫成兩個 edge node，或只畫出某一端頂點的串列而漏掉另一端。正確觀念是：edge node 是邊的實體，頂點表只保存 firstedge 指標，邊節點透過 ilink 與 jlink 分別接入兩個不同頂點的 incident edge list。本題要求畫出 representation，因此答案重點不是只列出邊集合，而是要清楚表現 vertex node、edge node、ivex、jvex、ilink、jlink 之間的關係。

必備背景知識

一、Threaded Binary Tree 的目的

Threaded binary tree 的目的，是利用二元樹中原本為 null 的 child pointer 存放 traversal 的 predecessor 或 successor，使得不使用 stack、不使用 recursion 也可以快速走訪樹。一般 binary tree 在走訪時，如果不用遞迴，就常需要額外 stack 記錄回溯路徑；thread 的設計則把原本浪費的空指標轉成可走訪的線索。

二、空指標與 Thread 數量

一般 n 個節點的 binary tree，每個節點有 leftChild 與 rightChild 兩個指標，所以總共有 2n 個 child pointer。因為樹有 n−1 條真正的 parent-child edge，所以非空 child pointer 有 n−1 個，剩下的空指標數量是 2n−(n−1)=n+1。若把所有空指標都改成 thread，則 threaded binary tree 共有 n+1 條 threads。

三、Predecessor 與 Successor

題目說 dashed line 表示 predecessor，solid line 表示 successor，意思是左邊空指標通常會指向該節點在指定 traversal 中的前驅，右邊空指標通常會指向該節點在指定 traversal 中的後繼。節點欄位中的 leftThread 與 rightThread 是布林值，用來判斷 leftChild/rightChild 到底是真正 child 還是 thread。若 leftThread=true，leftChild 不代表左子樹，而代表 predecessor thread；若 leftThread=false，leftChild 才是真正左子樹。右邊同理。

四、Inorder 與 Preorder 的差異

此題要分別寫 inorder 與 preorder threaded tree 的狀態，因此第一步要先列出 traversal sequence。Inorder 是 left subtree、root、right subtree；preorder 是 root、left subtree、right subtree。同一棵樹在不同走訪方式下，predecessor 與 successor 會不同，因此 thread 的指向也會不同。不能把 inorder 的 thread 直接套用到 preorder，這是本題常見錯誤。

五、本題樹形與小題處理方式

圖中的樹結構是 A 為 root，左子 B、右子 F；B 的子樹與 F 的子樹依題目圖形判斷。作答時應先依圖列出 inorder sequence，再逐一檢查每個節點缺少的左或右 child：缺左 child 時填 predecessor thread，缺右 child 時填 successor thread。接著再重新列出 preorder sequence，依相同方式填 preorder threaded tree。題目重點不是只寫 traversal 結果，而是要能在圖上正確標示哪些指標是真 child，哪些指標是 thread。

英文題目原文

完整英翻中

5.（10 分）遞移閉包圖
給定一個有向圖 G=(V,E)，其中 V={A,B,C}，E={(A,B),(B,C)}。

b) 計算 transitive closure matrix T。
c) 計算 reflexive transitive closure matrix R。
d) 簡短說明 T 與 R 的差異。

必備背景知識

一、Adjacency Matrix 與直接邊

Transitive closure 是圖論中用矩陣表示可達性的基本方法。題目給 directed graph G=(V,E)，其中 V={A,B,C}，E={(A,B),(B,C)}。Adjacency matrix A 只表示直接邊：若 i 到 j 有一條直接 directed edge，A[i][j]=1，否則為 0。因此 A 到 B 有直接邊，B 到 C 有直接邊，A 到 C 沒有直接邊，C 也沒有出邊。

二、Transitive Closure 的意義

若頂點順序為 A,B,C，則 A matrix 第一列表示 A 到 A、B、C，應為 0,1,0；第二列為 0,0,1；第三列為 0,0,0。Transitive closure matrix T 表示是否存在長度至少 1 的 path。因為 A 可以經由 B 到 C，所以 T[A][C]=1；A 可到 B，B 可到 C 仍為 1；其他沒有路徑者為 0。

三、Reflexive Transitive Closure

Reflexive transitive closure matrix R 則在 T 的基礎上加入「每個點到自己有長度 0 的路徑」，所以 R 的對角線必須全部為 1。這也是本題小題中很容易混淆的地方：transitive closure 是否包含自己到自己，要看題目是否要求 reflexive。若只問 transitive closure，一般不一定自動把 diagonal 設為 1；若問 reflexive transitive closure，diagonal 一定要補 1。

四、Warshall Algorithm 的背景

題目附的程式是 Warshall style 的 transitive closure：先把 length[i][j] 的直接邊資訊複製到 a[i][j]，接著三層迴圈依序讓每個 k 當作中繼點。如果 i 可以到 k 且 k 可以到 j，那 i 就可以到 j，因此 a[i][j] = a[i][j] || (a[i][k] && a[k][j])。這是一個 dynamic programming 的想法：第 k 輪後，矩陣記錄的是只允許使用 0..k 這些點作為中繼點時的可達性。完成所有 k 後，就得到所有可達關係。

五、小題判斷的重點

此題的 T 和 R 差別在於是否包含自己到自己的空路徑。若題目要求依程式 trace，必須照 k、i、j 的迴圈順序更新矩陣；若只要求寫出 closure，可直接從圖形路徑判斷。作答時要清楚區分「直接可達」與「經過中繼點可達」，並在最終答案中把 A→C 這種由 A→B→C 推出的間接路徑補進矩陣。

解題思維

圖中直接邊只有 A→B 與 B→C，因此 A matrix 只有 A row B column 與 B row C column 為 1。
因為 A 可以經由 B 到 C，所以 T[A,C]=1。
R 是 T 加上 reflexive 關係，所以 A,A；B,B；C,C 都設為 1。

OnlineGDBProgramizcpp.sh
#include <bits/stdc++.h>
using namespace std;

// 印出 0/1 矩陣，方便檢查 adjacency matrix、transitive closure matrix
void printMatrix(const vector<vector<int>>& M) {
    for (const auto& row : M) {
        for (int x : row) cout << x << ' ';
        cout << '\n';
    }
}

int main() {
    int n = 3; // 頂點數量：A, B, C

    // A 矩陣：A->B、B->C 有直接邊，其餘沒有直接邊
    vector<vector<int>> A = {
        {0, 1, 0},
        {0, 0, 1},
        {0, 0, 0}
    };

    // T 一開始等於 adjacency matrix，之後用 Warshall 演算法擴充可達性
    vector<vector<int>> T = A;

    // Warshall 核心：若 i 可到 k 且 k 可到 j，則 i 可到 j
    for (int k = 0; k < n; k++) {
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                T[i][j] = T[i][j] || (T[i][k] && T[k][j]);
            }
        }
    }

    // R 是 reflexive transitive closure，所以在 T 的基礎上把對角線改成 1
    vector<vector<int>> R = T;
    for (int i = 0; i < n; i++) R[i][i] = 1;

    cout << "Adjacency matrix A:\n";
    printMatrix(A);
    cout << "Transitive closure T:\n";
    printMatrix(T);
    cout << "Reflexive transitive closure R:\n";
    printMatrix(R);
    return 0;
}

b. Transitive closure T 推導。 T 表示「是否存在長度至少 1 的 path」。A 可以直接到 B，也可以 A→B→C 到 C，所以 A row 是 [0,1,1]。B 可以直接到 C，所以 B row 是 [0,0,1]。C 沒有 outgoing edge，所以 C row 是 [0,0,0]。

c. Reflexive transitive closure R 推導。 R 是在 T 的基礎上加入每個頂點到自己的 reflexive relation，也就是 A→A、B→B、C→C 都視為可達。因此把 T 的對角線設為 1，得到 R：A row [1,1,1]，B row [0,1,1]，C row [0,0,1]。

必備背景知識

一、Hash Table 與 Hash Function

Hash table 的 quadratic probing 是 open addressing 的一種 collision resolution 方法。題目給 table size 11，hash function h(k)=k mod 11，並指定 probing formula h(k,i)=(h(k)+i²) mod 11，其中 i 從 0 開始。這代表每插入一個 key 時，先算原始位置 h(k)。如果位置空，就直接放入；如果被占用，就依序嘗試 i=1、i=2、i=3... 的位置，也就是 h(k)+1²、h(k)+2²、h(k)+3² 再取 mod 11。

二、本題插入序列的計算背景

三、Quadratic Probing 的特性

最後 table 的 5,6,7,8,10 分別是 27,28,18,29,39，其餘空。Quadratic probing 相對 linear probing 的優點，是探測位置不是連續往下一格，因此比較不容易形成 primary clustering。Linear probing 若某一段連續格子被填滿，後續 hash 到附近的 key 會不斷延長這段 cluster，使搜尋與插入越來越慢。Quadratic probing 用平方位移打散探測路徑，可以減少這種連續堆積。

四、Collision Resolution 的限制

Quadratic probing 仍然不是萬能。它可能產生 secondary clustering，也就是 hash 到同一個初始位置的 key 仍會走完全相同的 probing sequence。除此之外，table size 與 probing 公式也會影響是否能探測到足夠多的位置。本題 table size 為 11，是質數，且資料量很小，因此依題目公式可以順利插入所有 key。

Q7. AOV Network 關鍵路徑

英文題目原文

7. (20 pts) AOV network
Given:
Activities and durations: A(3), B(2), C(4), D(2), E(1)
Precedence edges: A→B, A→C, B→D, C→D, D→E

a. (5 pts) Draw the AOV network and explain the meaning of each vertex and edge.
b. (8 pts) Determine the earliest start time (EST) and latest start time (LST) for each activity.
c. (7 pts) Identify the critical path and the project completion time.

完整英翻中

7.（20 分）AOV 網路
給定活動與持續時間：A(3), B(2), C(4), D(2), E(1)。優先關係邊：A→B, A→C, B→D, C→D, D→E。

a. 畫出 AOV network，並解釋每個 vertex 與 edge 的意義。
b. 求每個活動的 earliest start time（EST）與 latest start time（LST）。
c. 找出 critical path 與 project completion time。

必備背景知識

一、AOV Network 的定義

AOV network 是 Activity On Vertex network，也就是活動放在頂點上的有向無環圖。題目給 activities and durations：A(3), B(2), C(4), D(2), E(1)，表示活動 A 需要 3 單位時間，B 需要 2，依此類推。Precedence edges A→B, A→C, B→D, C→D, D→E 表示先後限制：A 必須完成後 B、C 才能開始；B 與 C 都完成後 D 才能開始；D 完成後 E 才能開始。

二、Earliest Start Time 的背景

計算 EST earliest start time 時，要從 source 活動往後推。沒有前置活動的 A，EST(A)=0，完成時間 EF(A)=0+3=3。B 的前置為 A，所以 EST(B)=3，EF(B)=5。C 的前置為 A，所以 EST(C)=3，EF(C)=7。D 的前置有 B 與 C，因此 D 不能只等 B 完成，而必須等所有前置都完成，所以 EST(D)=max(EF(B),EF(C))=max(5,7)=7，EF(D)=9。E 的前置是 D，所以 EST(E)=9，EF(E)=10。因此 project completion time 是 10。

三、Latest Start Time 的背景

計算 LST latest start time 時，要從專案完成時間往回推。最後活動 E 若不延誤專案，最晚開始 LST(E)=10−1=9。D 必須在 E 開始前完成，所以 LST(D)=LST(E)−duration(D)=9−2=7。B 是 D 的前置，所以 B 最晚完成時間不能超過 LST(D)=7，因此 LST(B)=7−2=5。C 也同理，LST(C)=7−4=3。A 有兩個後繼 B 與 C，所以 A 最晚完成時間要取後繼 LST 的最小值 min(LST(B),LST(C))=min(5,3)=3，因此 LST(A)=3−3=0。

四、Slack 與 Critical Path

Slack 是 LST−EST。若 slack=0，代表此活動沒有延誤空間，任何延誤都會影響整個 project completion time，因此屬於 critical path。由上述計算可知 A、C、D、E 的 EST 與 LST 相同，所以 critical path 是 A→C→D→E，總時間 3+4+2+1=10。B 的 EST=3，LST=5，slack=2，因此 B 可以延後最多 2 單位時間而不影響總工期。

五、小題作答重點

本題要特別注意 D 的開始時間必須等 B 與 C 都完成，所以 EST(D) 要取 max，而不是取 min 或只看其中一條路。回推 LST 時，若一個活動有多個後繼，必須取後繼 LST 減 duration 後的最小值，因為它必須滿足所有後繼的開始限制。這也是 critical path 題型中最常見的錯誤來源。

解題思維

先由左到右畫出 precedence：A 分別指向 B、C；B 與 C 都指向 D；D 指向 E。
Forward pass 算 EST：A 從 0 開始；B、C 都要等 A 結束；D 要等 B 與 C 都結束，所以取最大完成時間。
Backward pass 算 LST：專案完成時間先由 forward pass 得到，再從 E 倒推回 A。
EST 等於 LST 的活動就是不能延遲的活動，串起來就是 critical path。

完整英翻中

b. 使用 Kruskal Algorithm 找 MST，並使用 Disjoint-Set / Union-Find。判斷 Kruskal 與 union-find 相關敘述。
c. 使用 dictionary/hash map 與 heap/priority queue。判斷 dictionary 與 heap 相關敘述。

必備背景知識

一、Prim 與 Dijkstra 的共同表面特徵

第八題是觀念判斷題，核心在於區分 Prim、Dijkstra、Kruskal、Union-Find、dictionary 與 heap 的用途與時間複雜度。Prim 與 Dijkstra 表面上很像，因為兩者都可使用 priority queue，且都從某個起點逐步擴張一棵 tree；但它們的最佳化目標完全不同。

二、Prim 與 Dijkstra 的根本差異

Prim 的目標是找 Minimum Spanning Tree，它每次選的是「連接目前 tree 與外部頂點的最小邊」，最後讓整棵 spanning tree 的總權重最小。Dijkstra 的目標是 single-source shortest paths，它每次選的是「目前已知從 source 到某點距離最小的頂點」，最後建立 shortest path tree。因此 Statement 1 說兩者都使用 greedy approach build a tree，可視為 True；Statement 2 說 Prim always gives shortest path tree from a source 是 False，Prim 不保證 source 到各點最短；Statement 3 說 Dijkstra finds MST 是 False，Dijkstra 不保證總邊權最小；Statement 4 說兩者都可維護 priority queue 並根據目前最佳值更新，概念上 True。

三、Kruskal 與 Union-Find

Kruskal 的部分，Kruskal 一定先把 edges 依權重遞增排序，再依序考慮每條邊。若該邊連接的是兩個不同 connected components，就加入 MST；若兩端已在同一 set，加入會形成 cycle，所以跳過。Union-Find 的作用正是快速判斷兩個頂點是否在同一 set，並在選取邊後合併 connected components。因此若題目說 Kruskal 不需要排序邊，或說它是從某一個 source 開始擴張，通常都是錯的。

四、Heap 的背景知識

Heap 是一種 complete binary tree，通常用 array 表示。Max heap 要求 parent key 大於或等於 children key，因此最大值在 root；min heap 則相反，最小值在 root。Heap 常用於 priority queue，支援插入與刪除最高或最低 priority 元素。對 heap 的敘述題要分清楚：heap 不是 binary search tree，heap 只保證 parent-child 的局部順序，不保證左子樹所有值小於右子樹。

五、Dictionary ADT 與判斷題作答方式

Dictionary ADT 主要支援依 key 查詢、插入、刪除資料。它可以用 sorted array、unsorted array、hash table、binary search tree 等方式實作。不同實作的時間複雜度不同，因此判斷題若說 dictionary 一定只能用某一種資料結構，通常過度絕對。作答第八題時，必須抓住每個演算法的目標：Prim 是 MST，Dijkstra 是 shortest path，Kruskal 是 MST 且需要排序邊與 cycle checking，heap 是 priority queue 的常用實作，dictionary 是依 key 操作的 ADT。
