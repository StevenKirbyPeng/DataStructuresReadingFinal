二、每頁 1000 字以上條列式詳解

  • 觀念定位：Chapter 7 Sorting 是資料結構中非常常考的一章，核心是把一串 records 依照 key 排成 nondecreasing order。投影片特別強調 sorting problem 的輸入是 n 個 records，每個 record 有 key，輸出是一個 permutation，使得 key 的順序滿足 Kσ(1) ≤ Kσ(2) ≤ … ≤ Kσ(n)。因此排序不是只會寫程式，還要懂穩定性、原地性、內部排序、外部排序、比較排序與時間下界。
  • 基本定義：In-place sorting 表示除了工作陣列外，只使用常數數量額外變數。Stable sorting 表示 key 相同的 records 在排序後仍保持原本相對順序。Internal sorting 表示資料可全部放在 main memory；External sorting 表示資料太大，需要 disk 或 tape。Comparison sort 表示演算法只能透過比較兩個 key 的大小決定順序。
  • 圖片說明：右側圖片保留原始投影片。若看到散點圖，代表視覺化排序過程，橫軸通常是 index，縱軸是 value；若看到陣列表格，代表每一輪演算法狀態；若看到程式碼框，重點是迴圈範圍、比較條件、交換或搬移；若看到遞迴樹，重點是 divide-and-conquer 的分解與合併。
  • Bubble Sort 重點：Bubble sort 每一輪相鄰比較，若順序錯就交換。第一輪後最大值會被推到最右邊，像泡泡往上浮，所以稱為 bubble up。它的時間複雜度通常是 O(n²)，因為雙層迴圈需要大量相鄰比較。若加入 swapped flag，最佳情況可提早結束，但投影片版本是固定雙層迴圈。
  • Selection Sort 重點：Selection sort 每一輪從未排序區找最小值，再與未排序區第一個位置交換。它的比較次數幾乎固定，因此 best、average、worst 都是 O(n²)。它交換次數少，最多 n-1 次，但一般不穩定，因為交換可能破壞相同 key 的相對順序。考試常問它與 insertion sort 差在哪裡。
  • Insertion Sort 重點：Insertion sort 把前方視為已排序區，每次取 a[j] 插入正確位置。它在資料幾乎已排序時非常快，最佳情況每一輪只比較一次，總時間 O(n)；最壞情況是反向排序，每個元素都要往前移，時間 O(n²)。Insertion sort 是 stable 且 in-place，這兩點非常常考。
  • Quick Sort 重點：Quick sort 是 divide-and-conquer。先選 pivot，做 partition，使 pivot 左邊都小於等於它，右邊都大於等於它；接著遞迴排序左右子陣列。Combine 階段不需要做事。若 pivot 每次都接近 median，時間是 O(n log n)；若每次都選到最大或最小，會退化成 O(n²)。因此 median-of-3 或 randomized pivot 可降低最壞情況發生機率。
  • Decision Tree 下界：投影片用 decision tree 說明比較排序不可能比 Ω(n log n) 更快。n 個相異元素有 n! 種可能排序結果，因此決策樹至少要有 n! 個 leaves。二元樹高度若為 h，最多有 2^(h-1) 個 leaves，所以 h 至少是 log₂(n!)+1，推得任何 comparison sort 的 worst-case lower bound 是 Ω(n log n)。
  • Merge Sort 重點：Merge sort 也是 divide-and-conquer。Divide：將資料分成兩半。Conquer：遞迴排序左右兩半。Combine：把兩個已排序子串列 merge 成一個排序串列。時間複雜度由 T(n)=2T(n/2)+O(n) 得 O(n log n)。Merge sort 通常 stable，但需要額外陣列，因此不是嚴格 in-place。
  • 考試常見錯誤：很多同學會把 selection sort 誤判為 stable；把 quick sort 誤以為永遠 O(n log n)；把 merge sort 誤以為 in-place；把 insertion sort 最佳情況忘記是 O(n)。另外，投影片程式採用 a[1:n] 的 indexing，有些程式使用 C++ vector 0-based indexing，作答時要注意 index 差異。

三、演算法步驟分析

  1. 先判斷題目要求哪一種排序：Bubble、Selection、Insertion、Quick、Merge 或理論下界。
  2. 若是 trace 題，先寫出每一輪陣列狀態，不要跳步。
  3. 若是 Bubble，注意每一 pass 固定一個最大值到右側。
  4. 若是 Selection，注意每一 pass 找未排序區最小值並交換。
  5. 若是 Insertion，注意前方已排序區如何右移並插入 temp。
  6. 若是 Quick，先標 pivot，再做 partition，最後畫左右遞迴區間。
  7. 若是 Merge，先一路切到單元素，再逐層合併已排序串列。
  8. 最後補上 stable、in-place、best/worst/average time complexity。

二、每頁 1000 字以上條列式詳解

  • 觀念定位：Chapter 7 Sorting 是資料結構中非常常考的一章，核心是把一串 records 依照 key 排成 nondecreasing order。投影片特別強調 sorting problem 的輸入是 n 個 records，每個 record 有 key，輸出是一個 permutation，使得 key 的順序滿足 Kσ(1) ≤ Kσ(2) ≤ … ≤ Kσ(n)。因此排序不是只會寫程式，還要懂穩定性、原地性、內部排序、外部排序、比較排序與時間下界。
  • 基本定義：In-place sorting 表示除了工作陣列外，只使用常數數量額外變數。Stable sorting 表示 key 相同的 records 在排序後仍保持原本相對順序。Internal sorting 表示資料可全部放在 main memory；External sorting 表示資料太大，需要 disk 或 tape。Comparison sort 表示演算法只能透過比較兩個 key 的大小決定順序。
  • 圖片說明：右側圖片保留原始投影片。若看到散點圖，代表視覺化排序過程，橫軸通常是 index，縱軸是 value；若看到陣列表格，代表每一輪演算法狀態；若看到程式碼框，重點是迴圈範圍、比較條件、交換或搬移；若看到遞迴樹，重點是 divide-and-conquer 的分解與合併。
  • Bubble Sort 重點：Bubble sort 每一輪相鄰比較，若順序錯就交換。第一輪後最大值會被推到最右邊，像泡泡往上浮，所以稱為 bubble up。它的時間複雜度通常是 O(n²)，因為雙層迴圈需要大量相鄰比較。若加入 swapped flag，最佳情況可提早結束，但投影片版本是固定雙層迴圈。
  • Selection Sort 重點：Selection sort 每一輪從未排序區找最小值，再與未排序區第一個位置交換。它的比較次數幾乎固定，因此 best、average、worst 都是 O(n²)。它交換次數少，最多 n-1 次，但一般不穩定，因為交換可能破壞相同 key 的相對順序。考試常問它與 insertion sort 差在哪裡。
  • Insertion Sort 重點：Insertion sort 把前方視為已排序區，每次取 a[j] 插入正確位置。它在資料幾乎已排序時非常快，最佳情況每一輪只比較一次，總時間 O(n)；最壞情況是反向排序，每個元素都要往前移，時間 O(n²)。Insertion sort 是 stable 且 in-place，這兩點非常常考。
  • Quick Sort 重點：Quick sort 是 divide-and-conquer。先選 pivot，做 partition，使 pivot 左邊都小於等於它，右邊都大於等於它；接著遞迴排序左右子陣列。Combine 階段不需要做事。若 pivot 每次都接近 median，時間是 O(n log n)；若每次都選到最大或最小，會退化成 O(n²)。因此 median-of-3 或 randomized pivot 可降低最壞情況發生機率。
  • Decision Tree 下界：投影片用 decision tree 說明比較排序不可能比 Ω(n log n) 更快。n 個相異元素有 n! 種可能排序結果，因此決策樹至少要有 n! 個 leaves。二元樹高度若為 h，最多有 2^(h-1) 個 leaves，所以 h 至少是 log₂(n!)+1，推得任何 comparison sort 的 worst-case lower bound 是 Ω(n log n)。
  • Merge Sort 重點：Merge sort 也是 divide-and-conquer。Divide：將資料分成兩半。Conquer：遞迴排序左右兩半。Combine：把兩個已排序子串列 merge 成一個排序串列。時間複雜度由 T(n)=2T(n/2)+O(n) 得 O(n log n)。Merge sort 通常 stable，但需要額外陣列，因此不是嚴格 in-place。
  • 考試常見錯誤：很多同學會把 selection sort 誤判為 stable；把 quick sort 誤以為永遠 O(n log n)；把 merge sort 誤以為 in-place；把 insertion sort 最佳情況忘記是 O(n)。另外，投影片程式採用 a[1:n] 的 indexing，有些程式使用 C++ vector 0-based indexing，作答時要注意 index 差異。

二、每頁 1000 字以上條列式詳解

  • 觀念定位：Chapter 7 Sorting 是資料結構中非常常考的一章，核心是把一串 records 依照 key 排成 nondecreasing order。投影片特別強調 sorting problem 的輸入是 n 個 records，每個 record 有 key，輸出是一個 permutation，使得 key 的順序滿足 Kσ(1) ≤ Kσ(2) ≤ … ≤ Kσ(n)。因此排序不是只會寫程式，還要懂穩定性、原地性、內部排序、外部排序、比較排序與時間下界。
  • 基本定義：In-place sorting 表示除了工作陣列外，只使用常數數量額外變數。Stable sorting 表示 key 相同的 records 在排序後仍保持原本相對順序。Internal sorting 表示資料可全部放在 main memory；External sorting 表示資料太大，需要 disk 或 tape。Comparison sort 表示演算法只能透過比較兩個 key 的大小決定順序。
  • 圖片說明：右側圖片保留原始投影片。若看到散點圖，代表視覺化排序過程，橫軸通常是 index，縱軸是 value；若看到陣列表格，代表每一輪演算法狀態；若看到程式碼框，重點是迴圈範圍、比較條件、交換或搬移；若看到遞迴樹，重點是 divide-and-conquer 的分解與合併。
  • Bubble Sort 重點：Bubble sort 每一輪相鄰比較，若順序錯就交換。第一輪後最大值會被推到最右邊，像泡泡往上浮，所以稱為 bubble up。它的時間複雜度通常是 O(n²)，因為雙層迴圈需要大量相鄰比較。若加入 swapped flag，最佳情況可提早結束，但投影片版本是固定雙層迴圈。
  • Selection Sort 重點：Selection sort 每一輪從未排序區找最小值，再與未排序區第一個位置交換。它的比較次數幾乎固定，因此 best、average、worst 都是 O(n²)。它交換次數少，最多 n-1 次，但一般不穩定，因為交換可能破壞相同 key 的相對順序。考試常問它與 insertion sort 差在哪裡。
  • Insertion Sort 重點：Insertion sort 把前方視為已排序區，每次取 a[j] 插入正確位置。它在資料幾乎已排序時非常快，最佳情況每一輪只比較一次，總時間 O(n)；最壞情況是反向排序，每個元素都要往前移，時間 O(n²)。Insertion sort 是 stable 且 in-place，這兩點非常常考。
  • Quick Sort 重點：Quick sort 是 divide-and-conquer。先選 pivot，做 partition，使 pivot 左邊都小於等於它，右邊都大於等於它；接著遞迴排序左右子陣列。Combine 階段不需要做事。若 pivot 每次都接近 median，時間是 O(n log n)；若每次都選到最大或最小，會退化成 O(n²)。因此 median-of-3 或 randomized pivot 可降低最壞情況發生機率。
  • Decision Tree 下界：投影片用 decision tree 說明比較排序不可能比 Ω(n log n) 更快。n 個相異元素有 n! 種可能排序結果，因此決策樹至少要有 n! 個 leaves。二元樹高度若為 h，最多有 2^(h-1) 個 leaves，所以 h 至少是 log₂(n!)+1，推得任何 comparison sort 的 worst-case lower bound 是 Ω(n log n)。
  • Merge Sort 重點：Merge sort 也是 divide-and-conquer。Divide：將資料分成兩半。Conquer：遞迴排序左右兩半。Combine：把兩個已排序子串列 merge 成一個排序串列。時間複雜度由 T(n)=2T(n/2)+O(n) 得 O(n log n)。Merge sort 通常 stable，但需要額外陣列，因此不是嚴格 in-place。
  • 考試常見錯誤：很多同學會把 selection sort 誤判為 stable；把 quick sort 誤以為永遠 O(n log n)；把 merge sort 誤以為 in-place；把 insertion sort 最佳情況忘記是 O(n)。另外，投影片程式採用 a[1:n] 的 indexing，有些程式使用 C++ vector 0-based indexing，作答時要注意 index 差異。

  1. 先判斷題目要求哪一種排序：Bubble、Selection、Insertion、Quick、Merge 或理論下界。

  3. 若是 Bubble，注意每一 pass 固定一個最大值到右側。
  4. 若是 Selection，注意每一 pass 找未排序區最小值並交換。

  7. 若是 Merge，先一路切到單元素，再逐層合併已排序串列。

二、每頁 1000 字以上條列式詳解

  • 觀念定位：Chapter 7 Sorting 是資料結構中非常常考的一章，核心是把一串 records 依照 key 排成 nondecreasing order。投影片特別強調 sorting problem 的輸入是 n 個 records，每個 record 有 key，輸出是一個 permutation，使得 key 的順序滿足 Kσ(1) ≤ Kσ(2) ≤ … ≤ Kσ(n)。因此排序不是只會寫程式，還要懂穩定性、原地性、內部排序、外部排序、比較排序與時間下界。
  • 基本定義：In-place sorting 表示除了工作陣列外，只使用常數數量額外變數。Stable sorting 表示 key 相同的 records 在排序後仍保持原本相對順序。Internal sorting 表示資料可全部放在 main memory；External sorting 表示資料太大，需要 disk 或 tape。Comparison sort 表示演算法只能透過比較兩個 key 的大小決定順序。
  • 圖片說明：右側圖片保留原始投影片。若看到散點圖，代表視覺化排序過程，橫軸通常是 index，縱軸是 value；若看到陣列表格，代表每一輪演算法狀態；若看到程式碼框，重點是迴圈範圍、比較條件、交換或搬移；若看到遞迴樹，重點是 divide-and-conquer 的分解與合併。
  • Bubble Sort 重點：Bubble sort 每一輪相鄰比較，若順序錯就交換。第一輪後最大值會被推到最右邊，像泡泡往上浮，所以稱為 bubble up。它的時間複雜度通常是 O(n²)，因為雙層迴圈需要大量相鄰比較。若加入 swapped flag，最佳情況可提早結束，但投影片版本是固定雙層迴圈。
  • Selection Sort 重點：Selection sort 每一輪從未排序區找最小值，再與未排序區第一個位置交換。它的比較次數幾乎固定，因此 best、average、worst 都是 O(n²)。它交換次數少，最多 n-1 次，但一般不穩定，因為交換可能破壞相同 key 的相對順序。考試常問它與 insertion sort 差在哪裡。
  • Insertion Sort 重點：Insertion sort 把前方視為已排序區，每次取 a[j] 插入正確位置。它在資料幾乎已排序時非常快，最佳情況每一輪只比較一次，總時間 O(n)；最壞情況是反向排序，每個元素都要往前移，時間 O(n²)。Insertion sort 是 stable 且 in-place，這兩點非常常考。
  • Quick Sort 重點：Quick sort 是 divide-and-conquer。先選 pivot，做 partition，使 pivot 左邊都小於等於它，右邊都大於等於它；接著遞迴排序左右子陣列。Combine 階段不需要做事。若 pivot 每次都接近 median，時間是 O(n log n)；若每次都選到最大或最小，會退化成 O(n²)。因此 median-of-3 或 randomized pivot 可降低最壞情況發生機率。
  • Decision Tree 下界：投影片用 decision tree 說明比較排序不可能比 Ω(n log n) 更快。n 個相異元素有 n! 種可能排序結果，因此決策樹至少要有 n! 個 leaves。二元樹高度若為 h，最多有 2^(h-1) 個 leaves，所以 h 至少是 log₂(n!)+1，推得任何 comparison sort 的 worst-case lower bound 是 Ω(n log n)。
  • Merge Sort 重點：Merge sort 也是 divide-and-conquer。Divide：將資料分成兩半。Conquer：遞迴排序左右兩半。Combine：把兩個已排序子串列 merge 成一個排序串列。時間複雜度由 T(n)=2T(n/2)+O(n) 得 O(n log n)。Merge sort 通常 stable，但需要額外陣列，因此不是嚴格 in-place。
  • 考試常見錯誤：很多同學會把 selection sort 誤判為 stable；把 quick sort 誤以為永遠 O(n log n)；把 merge sort 誤以為 in-place；把 insertion sort 最佳情況忘記是 O(n)。另外，投影片程式採用 a[1:n] 的 indexing，有些程式使用 C++ vector 0-based indexing，作答時要注意 index 差異。

二、每頁 1000 字以上條列式詳解

  • 觀念定位：Chapter 7 Sorting 是資料結構中非常常考的一章，核心是把一串 records 依照 key 排成 nondecreasing order。投影片特別強調 sorting problem 的輸入是 n 個 records，每個 record 有 key，輸出是一個 permutation，使得 key 的順序滿足 Kσ(1) ≤ Kσ(2) ≤ … ≤ Kσ(n)。因此排序不是只會寫程式，還要懂穩定性、原地性、內部排序、外部排序、比較排序與時間下界。
  • 基本定義：In-place sorting 表示除了工作陣列外，只使用常數數量額外變數。Stable sorting 表示 key 相同的 records 在排序後仍保持原本相對順序。Internal sorting 表示資料可全部放在 main memory；External sorting 表示資料太大，需要 disk 或 tape。Comparison sort 表示演算法只能透過比較兩個 key 的大小決定順序。

  • Bubble Sort 重點：Bubble sort 每一輪相鄰比較，若順序錯就交換。第一輪後最大值會被推到最右邊，像泡泡往上浮，所以稱為 bubble up。它的時間複雜度通常是 O(n²)，因為雙層迴圈需要大量相鄰比較。若加入 swapped flag，最佳情況可提早結束，但投影片版本是固定雙層迴圈。
  • Selection Sort 重點：Selection sort 每一輪從未排序區找最小值，再與未排序區第一個位置交換。它的比較次數幾乎固定，因此 best、average、worst 都是 O(n²)。它交換次數少，最多 n-1 次，但一般不穩定，因為交換可能破壞相同 key 的相對順序。考試常問它與 insertion sort 差在哪裡。
  • Insertion Sort 重點：Insertion sort 把前方視為已排序區，每次取 a[j] 插入正確位置。它在資料幾乎已排序時非常快，最佳情況每一輪只比較一次，總時間 O(n)；最壞情況是反向排序，每個元素都要往前移，時間 O(n²)。Insertion sort 是 stable 且 in-place，這兩點非常常考。
  • Quick Sort 重點：Quick sort 是 divide-and-conquer。先選 pivot，做 partition，使 pivot 左邊都小於等於它，右邊都大於等於它；接著遞迴排序左右子陣列。Combine 階段不需要做事。若 pivot 每次都接近 median，時間是 O(n log n)；若每次都選到最大或最小，會退化成 O(n²)。因此 median-of-3 或 randomized pivot 可降低最壞情況發生機率。
  • Decision Tree 下界：投影片用 decision tree 說明比較排序不可能比 Ω(n log n) 更快。n 個相異元素有 n! 種可能排序結果，因此決策樹至少要有 n! 個 leaves。二元樹高度若為 h，最多有 2^(h-1) 個 leaves，所以 h 至少是 log₂(n!)+1，推得任何 comparison sort 的 worst-case lower bound 是 Ω(n log n)。
  • Merge Sort 重點：Merge sort 也是 divide-and-conquer。Divide：將資料分成兩半。Conquer：遞迴排序左右兩半。Combine：把兩個已排序子串列 merge 成一個排序串列。時間複雜度由 T(n)=2T(n/2)+O(n) 得 O(n log n)。Merge sort 通常 stable，但需要額外陣列，因此不是嚴格 in-place。
  • 考試常見錯誤：很多同學會把 selection sort 誤判為 stable；把 quick sort 誤以為永遠 O(n log n)；把 merge sort 誤以為 in-place；把 insertion sort 最佳情況忘記是 O(n)。另外，投影片程式採用 a[1:n] 的 indexing，有些程式使用 C++ vector 0-based indexing，作答時要注意 index 差異。

  • Quick Sort 重點：Quick sort 是 divide-and-conquer。先選 pivot，做 partition，使 pivot 左邊都小於等於它，右邊都大於等於它；接著遞迴排序左右子陣列。Combine 階段不需要做事。若 pivot 每次都接近 median，時間是 O(n log n)；若每次都選到最大或最小，會退化成 O(n²)。因此 median-of-3 或 randomized pivot 可降低最壞情況發生機率。
  • Decision Tree 下界：投影片用 decision tree 說明比較排序不可能比 Ω(n log n) 更快。n 個相異元素有 n! 種可能排序結果，因此決策樹至少要有 n! 個 leaves。二元樹高度若為 h，最多有 2^(h-1) 個 leaves，所以 h 至少是 log₂(n!)+1，推得任何 comparison sort 的 worst-case lower bound 是 Ω(n log n)。
  • Merge Sort 重點：Merge sort 也是 divide-and-conquer。Divide：將資料分成兩半。Conquer：遞迴排序左右兩半。Combine：把兩個已排序子串列 merge 成一個排序串列。時間複雜度由 T(n)=2T(n/2)+O(n) 得 O(n log n)。Merge sort 通常 stable，但需要額外陣列，因此不是嚴格 in-place。
  • 考試常見錯誤：很多同學會把 selection sort 誤判為 stable；把 quick sort 誤以為永遠 O(n log n)；把 merge sort 誤以為 in-place；把 insertion sort 最佳情況忘記是 O(n)。另外，投影片程式採用 a[1:n] 的 indexing，有些程式使用 C++ vector 0-based indexing，作答時要注意 index 差異。
