題目 01：Array vs Linked List 差異動畫動畫教學Chapter 4 Linked Lists

（一）主題來源

Chapter 4 Linked Lists

（二）學習目標

理解連續儲存與指標串接的差異。

（三）核心觀念說明

Array 適合快速索引，但中間插入刪除需要搬移；Linked List 不需要連續記憶體，靠節點 link 串起來。

這個主題的教學重點是讓學生不只是背名詞，而是能看見資料如何被連接、搬移、比較或拜訪。請先觀察圖形狀態，再配合下方 Step-by-step 說明與動畫按鈕逐步理解。

（四）步驟式教學

  1. Step 1：畫出目前節點與 head/first 指標。
  2. Step 2：確認要操作的位置，先保存可能會失去的下一個節點。
  3. Step 3：依正確順序更新 link 指標。
  4. Step 4：檢查串列是否仍能從 head 一路走到 NULL。

C++ 參考程式
#include <iostream>
using namespace std;

struct Node {
    int data;      // 節點資料
    Node* next;    // 指向下一個節點
    Node(int v, Node* n=nullptr): data(v), next(n) {}
};

void printList(Node* head) {
    // 從 head 開始一路走到 nullptr
    for(Node* cur=head; cur!=nullptr; cur=cur->next) cout << cur->data << " -> ";
    cout << "NULL\n";
}

Node* insertFront(Node* head, int value) {
    // 新節點先接到原本 head，再回傳新 head
    return new Node(value, head);
}

void insertAfter(Node* x, int value) {
    if(!x) return;                       // 防止 nullptr
    Node* p = new Node(value, x->next);  // 先接住後半段
    x->next = p;                         // 再讓 x 接到新節點
}

int main(){
    Node* head = nullptr;
    head = insertFront(head, 30);
    head = insertFront(head, 20);
    head = insertFront(head, 10);
    insertAfter(head->next, 50);
    printList(head); // 10 -> 20 -> 50 -> 30 -> NULL
}

（八）常見錯誤

  • 指標更新順序寫反，造成後半段斷鏈。
  • 忘記處理空串列或 NULL。
  • delete 後還繼續使用 dangling pointer。

void pushMaxHeap(vector<int>& h, int x){

void pushMaxHeap(vector<int>& h, int x){

（四）步驟式教學

  1. Step 1：計算所有頂點 indegree。
  2. Step 2：把 indegree=0 的頂點放入 queue。
  3. Step 3：輸出頂點並移除它的 outgoing edges。
  4. Step 4：若所有頂點輸出成功，得到拓樸序。

#include <iostream>
#include <vector>
#include <queue>
using namespace std;
int main(){
    vector<vector<int>> adj={{1,2},{3},{3},{}}; int n=4;
    vector<int> indeg(n,0); for(int u=0;u<n;u++) for(int v:adj[u]) indeg[v]++;
    queue<int> q; for(int i=0;i<n;i++) if(indeg[i]==0) q.push(i);
    while(!q.empty()){
        int u=q.front(); q.pop(); cout << u << ' ';
        for(int v:adj[u]) if(--indeg[v]==0) q.push(v);
    }
}
