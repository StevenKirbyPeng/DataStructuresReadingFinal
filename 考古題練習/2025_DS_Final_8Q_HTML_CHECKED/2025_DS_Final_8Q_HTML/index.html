題目圖片

題目中文翻譯

給定固定長度 3 的字串列表 [cat, bat, ant, car, bar, arc, cab]，使用 LSD Radix Sort 從最後一個字元開始排序。每一輪必須使用 stable sort，並列出每一輪排序後的結果；另外回答時間複雜度與穩定性為什麼重要。

動畫一步一步解題

Step 1

一、題目在問什麼

這題不是只問最後排序結果，而是要你完整模擬 LSD Radix Sort 三個 pass：先看第 3 個字元，再看第 2 個字元，最後看第 1 個字元。還要知道 counting sort 當子程序時的複雜度，以及為什麼每一輪都一定要穩定。

二、必備背景知識

Radix Sort 的核心背景是把一筆資料看成由多個「位元、數字或字元」組成的 key，而不是只用一次比較就決定大小。一般比較排序會直接比較兩個完整字串，例如比較 cat 和 car 時會從第一個字元一路比到不同的位置；但 Radix Sort 的思考方式是把 key 拆成固定長度的欄位，依照某一個欄位分輪處理。這題給的是固定長度 3 的英文小寫字串，所以每個字串都可以視為三個欄位：第一字元、第二字元、第三字元。LSD Radix Sort 的 LSD 是 Least Significant Digit，意思是從最不重要的欄位開始，也就是先排第三字元，再排第二字元，最後排第一字元。若字串長度固定，最後完成時就會得到字典序的結果。

穩定排序是本題最重要的觀念。穩定排序 stable sort 的意思是：當兩筆資料在本輪比較的 key 相同時，它們在輸出中的相對先後順序必須和輸入時一樣。例如兩個字串的最後字元都同樣是 t，若前一輪或原始順序中 cat 在 bat 前面，本輪只看最後字元時就不能任意把它們反過來。LSD Radix Sort 之所以可以從右到左處理，是因為後面欄位的排序結果會被穩定地保留下來；如果某一輪不是 stable，前面已經整理好的較低位資訊就會被破壞，最後結果可能不是正確字典序。因此解這類題目不能只寫「排序三次」，而要特別標示每一輪使用 stable sort。

時間複雜度常寫成 O(k(n+r))。其中 n 是資料筆數，k 是每筆資料的欄位數或 key 長度，r 是每一欄可能出現的種類數。以本題來說，n=7，k=3，若只考慮小寫英文字母，r 可以視為 26。每一輪若用 counting sort 當穩定子程序，會先統計字元出現次數，再計算累積位置，最後由右往左或用穩定方式放回輸出陣列，所以一輪成本是 O(n+r)，總共 k 輪就是 O(k(n+r))。這個分析和比較排序的 O(n log n) 不同，因為 Radix Sort 不是靠任意兩元素比較，而是利用 key 的有限範圍分配到桶或計數位置。

考試作答時的背景流程可以固定成四件事：第一，確認資料長度是否固定；第二，確認從 LSD 還是 MSD 開始；第三，每一輪只看指定欄位並保持穩定；第四，列出每一 pass 後的序列。若題目問穩定性，就回答「相同目前 key 的元素必須保持上一輪順序，否則低位排序成果會消失」。若題目問空間，counting sort 通常需要輸出陣列與 count 陣列，因此不是純 in-place。若題目問適用情況，Radix Sort 適合 key 長度不大、範圍可控、資料量較大的情況；若 key 很長或 r 很大，額外成本可能變高。這些背景能幫你判斷每一步排序結果是否合理。

三、解題思維

看到固定長度字串與 LSD，就先標出每個字串三個位置：第 2 位是最後字元、第 1 位是中間字元、第 0 位是第一字元。每一輪只看一個字元，但同字元組內保持上一輪順序。

七、常見錯誤

常見錯誤：只寫最後答案、Pass 順序從第一個字元開始、同字元時沒有維持 stable order、把 r 忽略成 O(kn) 但沒有說明 r 為常數。

八、考試得分關鍵

滿分關鍵：三個 pass 的中間結果都要寫；時間複雜度要寫 O(k(n+r))；穩定性要能舉例說明。
背景程式資料補充： LSD Radix Sort 的程式觀念可以想成「外層迴圈控制位數、內層穩定排序控制同一位字元」。如果用 counting sort 實作，會先統計每個字元出現次數，再轉成 prefix sum，最後由右往左把資料放到輸出陣列，這樣可以維持 stable。題目裡的 r 表示字元範圍，例如 ASCII 可視為 128 或 256；k 是字串長度；n 是字串數量。
依照流程圖逐步講解：
1. 先確認每個字串長度都是 3，所以只需要做三輪。
2. 第一輪看最後一個字元，目的不是得到最終字典序，而是先把最低位整理好。
3. 第二輪看中間字元，同字元時必須維持上一輪順序。
4. 第三輪看第一個字元，最後才得到真正 lexicographic order。
5. 回答複雜度時，不要只寫 O(n)，因為還有 k 輪與 r 個 bucket。

#include <bits/stdc++.h>
using namespace std;

// 講義結構優先版：使用 link 陣列做 LSD Radix Sort。
// link[i] 表示排序 linked list 中第 i 筆資料的下一筆位置，0 表示結束。
int digit(const vector<string>& a, int current, int pos) {
    return static_cast<unsigned char>(a[current][pos]); // ASCII order
}

void PrintList(const vector<string>& a, const vector<int>& link, int first) {
    for (int p = first; p != 0; p = link[p]) cout << a[p] << " ";
    cout << "\n";
}

int RadixSort(vector<string>& a, vector<int>& link, const int d, const int r, const int n) {
    vector<int> e(r), f(r);          // e: bucket end, f: bucket front
    int first = 1;                   // 講義採 1-based linked list
    for (int i = 1; i < n; ++i) link[i] = i + 1;
    link[n] = 0;

    cout << "Initial: ";
    PrintList(a, link, first);

    for (int i = d - 1; i >= 0; --i) { // 從最低位元/最後一個字元開始
        fill(f.begin(), f.end(), 0);
        fill(e.begin(), e.end(), 0);

        for (int current = first; current; current = link[current]) {
            int k = digit(a, current, i);
            if (f[k] == 0) f[k] = current;
            else link[e[k]] = current;
            e[k] = current;
        }

        int j = 0;
        while (j < r && !f[j]) ++j;
        first = f[j];
        int last = e[j];
        for (int k = j + 1; k < r; ++k) {
            if (f[k]) {
                link[last] = f[k];
                last = e[k];
            }
        }
        link[last] = 0;

        cout << "After pass at position " << i << ": ";
        PrintList(a, link, first);
    }
    return first;
}

int main() {
    vector<string> a = {"", "cat", "bat", "ant", "car", "bar", "arc", "cab"};
    vector<int> link(a.size(), 0);
    int first = RadixSort(a, link, 3, 128, 7);

    cout << "Final sorted order: ";
    PrintList(a, link, first);
    return 0;
}
Q2：Graph：Adjacency Multilist 與 Sequential Adjacency List

題目中文翻譯

給定無向圖 V={A,B,C,D}，E={(A,B),(A,C),(B,C),(C,D)}。說明 Adjacency Multilist 對無向圖的優點，畫出或描述此圖的 Multilist 結構，以及畫出 Sequential Representation of Adjacency List。

動畫一步一步解題

Step 1

一、題目在問什麼

這題在比較無向圖的兩種儲存方式。重點是 Adjacency Multilist 讓一條無向邊只存一次，但仍能從兩端點各自串到這條邊。

二、必備背景知識

圖形 Graph 的表示法是本題的基礎。圖 G=(V,E) 由頂點集合 V 和邊集合 E 組成，若是無向圖，一條邊 (u,v) 同時連接 u 和 v；若是有向圖，則邊有方向。本題談 Adjacency Multilist 與 Sequential Adjacency List，重點不是單純列出鄰居，而是比較不同儲存結構如何讓「找鄰接邊、走訪某頂點 incident edges、刪除邊」更方便。圖的表示法常見有 adjacency matrix、adjacency list、orthogonal list、adjacency multilist 等。矩陣適合稠密圖，查詢兩點是否相連很快；串列適合稀疏圖，空間大約和頂點數加邊數相關。

Sequential Adjacency List通常可以理解成用陣列保存每個頂點的第一個鄰接節點位置，然後每個鄰接節點再以 link 串起來。若是無向圖，一條邊 {u,v} 往往會在 u 的 list 裡出現一次，也在 v 的 list 裡出現一次，因此同一條邊被記錄成兩個 adjacency nodes。這樣做插入與列出鄰居很直覺，但若要刪除一條無向邊，就要在兩個串列中都找到並刪除對應節點，兩份紀錄也要保持一致。若題目要求畫 sequential adjacency list，通常要畫出 vertex array、first pointer、每個節點的 vertex 欄位與 link 欄位。

Adjacency Multilist則是無向圖常見的邊導向結構。一條無向邊只建立一個 edge node，edge node 內保存兩個端點 vertex1、vertex2，以及兩個 link 欄位，分別把這條邊接到 vertex1 的 incident edge list 與 vertex2 的 incident edge list。也就是說，一個 edge node 同時屬於兩個頂點的邊串列，因此叫 multilist。這種結構的好處是邊只存一次，刪除或標記某條邊時比較不容易出現兩份資料不同步的問題；缺點是走訪時要判斷目前是從哪個端點進入，再沿著對應的 link 欄位往下走。

解題時要分清楚「鄰接點」與「邊節點」。Sequential adjacency list 常畫的是鄰接點節點，例如 u 的 list 裡有 v，v 的 list 裡有 u；Adjacency multilist 畫的是 edge node，例如一個節點代表 (u,v)，裡面同時連到 u-list 和 v-list。考試若問空間，sequential list 對無向圖常會有 2E 個 adjacency nodes；multilist 則是 E 個 edge nodes 加 V 個 vertex headers。若問操作，列出一點的 incident edges 兩者都可以，但 multilist 刪除邊更自然。畫圖時記得頂點表放在左或上方，每個頂點 header 指到第一條相關邊；每條邊節點要標出兩端點與兩條鏈結，否則容易被扣結構分。

三、解題思維

先列出所有頂點與邊，再決定每個頂點的第一條 incident edge。對 multilist 而言，要確認每條邊只建立一個 edge node；對 adjacency list 而言，要把無向邊展開成雙向鄰接。

#include <bits/stdc++.h>
using namespace std;

// 講義結構優先版：Adjacency Multilist for undirected graph。
struct EdgeNode {
    bool mark;        // 是否已拜訪/標記
    int ivex, jvex;   // 此 edge 連接的兩個 vertex index
    EdgeNode *ilink;  // 下一條 incident to ivex 的邊
    EdgeNode *jlink;  // 下一條 incident to jvex 的邊
    EdgeNode(int i, int j) : mark(false), ivex(i), jvex(j), ilink(nullptr), jlink(nullptr) {}
};

struct VertexNode {
    char data;
    EdgeNode* firstEdge;
    VertexNode(char c='?') : data(c), firstEdge(nullptr) {}
};

class AMLGraph {
private:
    vector<VertexNode> adjmulist;
    vector<unique_ptr<EdgeNode>> edges;
public:
    AMLGraph(const vector<char>& V) {
        for (char c : V) adjmulist.push_back(VertexNode(c));
    }
    void AddEdge(int u, int v) {
        edges.push_back(make_unique<EdgeNode>(u, v));
        EdgeNode* e = edges.back().get();
        e->ilink = adjmulist[u].firstEdge;
        adjmulist[u].firstEdge = e;
        e->jlink = adjmulist[v].firstEdge;
        adjmulist[v].firstEdge = e;
    }
    void PrintSequentialAdjacencyList() const {
        for (int i = 0; i < (int)adjmulist.size(); ++i) {
            cout << adjmulist[i].data << ": ";
            for (EdgeNode* p = adjmulist[i].firstEdge; p != nullptr; ) {
                int other;
                if (p->ivex == i) { other = p->jvex; p = p->ilink; }
                else { other = p->ivex; p = p->jlink; }
                cout << adjmulist[other].data << " ";
            }
            cout << "\n";
        }
    }
};

int main() {
    vector<char> V = {'A','B','C','D'};
    AMLGraph g(V);
    g.AddEdge(0,1); // (A,B)
    g.AddEdge(0,2); // (A,C)
    g.AddEdge(1,2); // (B,C)
    g.AddEdge(2,3); // (C,D)
    g.PrintSequentialAdjacencyList();
    return 0;
}
Q3：Biconnected Components：DFS Tree、dfn、low、BFS Tree
對應投影片 lec6a Graphs Part I：DFS、BFS、Connected/Biconnected Components、Articulation Point、dfn/low 約後半段頁面。

題目圖片

題目中文翻譯

從 node 1 開始，每次選擇未拜訪且 index 最小的節點，畫 DFS tree；找 backward edges；列出每個節點的 dfs number 與 low；找出所有 biconnected components；再用同樣選點規則畫 BFS tree 與 cross edge。

動畫一步一步解題

Step 1

一、題目在問什麼

這題在考 DFS low value 的完整手算流程。不是只要 DFS 順序，而是要從 DFS tree 推出 back edge、dfn、low，再用 low 判斷 biconnected components。

二、必備背景知識

Biconnected Components 的背景來自 DFS 深度優先搜尋。DFS 會從起點沿著尚未拜訪的鄰接點一路深入，形成 DFS tree。每個頂點被第一次拜訪時會得到一個 dfn，也就是 discovery number，表示 DFS 發現該頂點的順序。除了 tree edge 之外，在無向圖中還可能遇到 back edge，代表目前節點或其子孫可以連回祖先。這些 back edge 會影響 low 值，而 low 值正是判斷 articulation point 與 biconnected component 的關鍵。

low[u] 的意義是從 u 出發，允許走零條或多條 DFS tree edge 到 u 的子孫，再最多走一條 back edge，可以到達的最小 dfn。計算時可用觀念式公式理解：low[u] 會取三種來源的最小值：自己的 dfn[u]、由 back edge 直接連到祖先 v 的 dfn[v]、以及每個 DFS child w 的 low[w]。當某個 child w 滿足 low[w] >= dfn[u] 時，代表 w 的子樹沒有辦法繞過 u 回到 u 的祖先，因此 u 可能是 articulation point，而從 stack 中彈出的邊會形成一個 biconnected component。

雙連通分量可以理解成在同一個 component 內，任兩條邊或任兩點之間具有較強的連通關係；更直觀地說，若移除某個 articulation point 會讓圖分裂，那 articulation point 就是不同 biconnected components 的交界。考試常要求你畫 DFS tree、標 dfn/low，再圈出 biconnected components。最容易錯的是把 BFS tree 和 DFS tree 混在一起。BFS 是一層一層擴張，適合最短邊數距離；DFS 是一路深入，適合 dfn、low、articulation、biconnected component 這種遞迴回溯分析。

作答流程應該先固定鄰接點拜訪順序，因為 dfn/low 會受鄰接順序影響；若題目有指定編號順序，就不能自己任意換。接著畫 DFS tree，把非樹邊用虛線或不同標記表示 back edge。第三步，自底向上計算 low，葉節點先看是否有 back edge，父節點再把 child 的 low 傳回來。第四步，用 low[child] >= dfn[parent] 判斷分割點與 component 邊界。若題目另外要求 BFS tree，就重新用 queue 做層序搜尋，不要沿用 DFS tree。這些背景知識可以防止你把 dfn 當成 level，或把 low 當成最短距離，兩者完全不同。

動畫一步一步解題

Step 1

一、題目在問什麼

這題要你理解 thread 是拿原本 NULL child pointer 來指向 traversal predecessor 或 successor。不同 traversal order，thread 指向會不同。

二、必備背景知識

Threaded Binary Tree 的背景是二元樹走訪時，許多 NULL 指標其實沒有被利用。一般 linked binary tree 的節點有 leftChild 與 rightChild 兩個指標，若某節點沒有左子或右子，該欄位會是 NULL。Threaded tree 會把這些空指標改成線索 thread，指向走訪順序中的 predecessor 或 successor，讓 inorder 或 preorder traversal 可以不用 stack 或 recursion 就找到下一個節點。因此節點除了 leftChild、rightChild 外，還需要 leftThread、rightThread 之類的布林標記，用來區分「這個指標是真的 child」還是「這個指標是 thread」。

Inorder Thread 的順序是 Left、Node、Right。若某節點沒有左孩子，leftChild 可以指向 inorder predecessor；若沒有右孩子，rightChild 可以指向 inorder successor。舉例來說，某節點在 inorder 序列中前一個拜訪的節點就是 predecessor，下一個拜訪的節點就是 successor。當要找 inorder successor 時，若 rightThread 為 true，就直接沿右線索到 successor；若 rightThread 為 false，代表右指標是右子樹，就要進入右子樹後一路往最左下走。這是 threaded tree 程式最常考的邏輯。

Preorder Thread 的順序是 Node、Left、Right。它和 inorder thread 的 successor 判斷不同。preorder 的下一個節點通常優先是左孩子，若沒有左孩子才看右孩子；若兩者都沒有，就要透過 thread 往上或往下一個可走的右分支。因為 preorder 的 predecessor/successor 關係不像 inorder 那麼容易由左右子樹界線判斷，所以畫圖時一定要先列出完整 preorder 序列，再依照序列去補 thread。很多錯誤都是直接把 inorder 的線索套到 preorder，導致方向正確但目標節點錯誤。

考試畫 threaded tree 時要做三件事：第一，先列出指定 traversal 的完整序列；第二，針對每個節點檢查 leftChild/rightChild 是否為空；第三，只有空指標才可以改成 thread，非空 child 不能被 thread 覆蓋。若題目有 header node，header 的 left 通常指向 root，某些最前或最後節點的 thread 會回到 header，方便判斷結束。程式結構上，講義常用 class ThreadedNode 或 ThreadedTree，節點內含 data、leftChild、rightChild、leftThread、rightThread。解題時若能分清楚 child edge 與 thread edge，並用虛線表示 thread，就能避免把樹的原始結構改壞。

三、解題思維

先寫 traversal sequence，再對每個沒有 left child 或 right child 的節點補 predecessor/successor thread。Inorder 與 Preorder 必須分開做，不能混用。

using namespace std;

// 講義結構優先版：Threaded Binary Tree Node。
struct ThreadNode {
    char data;
    ThreadNode *leftChild, *rightChild;
    bool leftThread, rightThread; // true 表示 child pointer 是 thread
    ThreadNode(char x) : data(x), leftChild(nullptr), rightChild(nullptr),
                         leftThread(false), rightThread(false) {}
};

void Inorder(ThreadNode* root) {
    if (!root) return;
    if (!root->leftThread) Inorder(root->leftChild);
    cout << root->data << " ";
    if (!root->rightThread) Inorder(root->rightChild);
}

void Preorder(ThreadNode* root) {
    if (!root) return;
    cout << root->data << " ";
    if (!root->leftThread) Preorder(root->leftChild);
    if (!root->rightThread) Preorder(root->rightChild);
}

void BuildInorderThreads(ThreadNode* root, ThreadNode*& prev, int& threadCount) {
    if (!root) return;
    BuildInorderThreads(root->leftChild, prev, threadCount);
    if (root->leftChild == nullptr) {
        root->leftThread = true;
        root->leftChild = prev;       // predecessor thread
        ++threadCount;
    }
    if (prev && prev->rightChild == nullptr) {
        prev->rightThread = true;
        prev->rightChild = root;      // successor thread
        ++threadCount;
    }
    prev = root;
    BuildInorderThreads(root->rightChild, prev, threadCount);
}

int main() {
    // 建立題目二元樹：Inorder = C D B G E A F；Preorder = A B C D E G F
    ThreadNode *A=new ThreadNode('A'), *B=new ThreadNode('B'), *C=new ThreadNode('C');
    ThreadNode *D=new ThreadNode('D'), *E=new ThreadNode('E'), *F=new ThreadNode('F');
    ThreadNode *G=new ThreadNode('G');
    A->leftChild=B; A->rightChild=F;
    B->leftChild=C; B->rightChild=E;
    C->rightChild=D;
    E->leftChild=G;

    cout << "Inorder sequence:\n";
    Inorder(A);
    cout << "\n\nPreorder sequence:\n";
    Preorder(A);
    cout << "\n\n";

    int threadCount = 0;
    ThreadNode* prev = nullptr;
    BuildInorderThreads(A, prev, threadCount);
    if (prev && prev->rightChild == nullptr) { prev->rightThread = true; ++threadCount; }

    cout << "Number of threads in n-node threaded binary tree:\n";
    cout << "n + 1\n";
    return 0;
}

題目圖片

題目中文翻譯

給定 directed graph G=(V,E)，V={A,B,C}，E={(A,B),(B,C)}。寫出 adjacency matrix A，計算 transitive closure matrix T，計算 reflexive transitive closure matrix R，並解釋 T 與 R 的差別。

動畫一步一步解題

Step 1

一、題目在問什麼

這題在問可達性矩陣。A 是一步可達；T 是走一條以上路徑可達；R 是 T 再加上每個節點到自己的可達性。

二、必備背景知識

Transitive Closure 是有向圖可達性的完整表格。若從頂點 i 可以走一條或多條有向邊到頂點 j，就說 i reaches j，閉包矩陣 T[i][j] 應為 1。若不能到達則為 0。這和 shortest path 不同：transitive closure 只關心有沒有路徑，不關心路徑長度或成本。若題目要求 reflexive closure，則每個頂點到自己也視為可達，所以對角線 T[i][i] 必須設為 1。這點很常被忽略，尤其原圖 adjacency matrix 的對角線若是 0，做 reflexive transitive closure 前要先把對角線補成 1。

Warshall 演算法 是計算 transitive closure 的標準動態規劃方法。它的核心想法是逐步允許中繼點。假設目前處理到 k，代表我們允許路徑中使用 0 到 k 這些頂點作為 intermediate vertex。更新規則是：若 i 已經可以到 k，且 k 可以到 j，那 i 就可以經由 k 到 j，因此 T[i][j] = T[i][j] OR (T[i][k] AND T[k][j])。這個規則和 Floyd all-pairs shortest path 很像，但 Floyd 用加法與 min，Warshall 用 AND 與 OR。

矩陣操作背景也很重要。Warshall 通常用三層迴圈：for k、for i、for j。k 必須放在最外層，因為每一輪的語意是「新增允許的中繼點 k」。若把 k 放到內層，雖然某些情況可能仍得到結果，但會失去標準動態規劃的階段意義，考試上通常不建議。初始矩陣可由 adjacency matrix 複製而來，再視題目是否需要 reflexive closure 補上對角線。每一輪 k 後可以列出矩陣，表示加入某個中繼點後新增了哪些可達關係。

作答時的固定流程是：第一，先寫出原始 adjacency matrix；第二，若題目要求 reflexive，把對角線全部設為 1；第三，從 k=0 到 n-1 逐輪更新；第四，圈出因 k 新增的 1；第五，最後矩陣就是 transitive closure。若要解釋為什麼正確，可說每一輪 k 都在回答「是否存在一條路徑從 i 到 j，且中繼頂點只使用已處理集合」。當所有頂點都當過 k 後，所有可能路徑都被考慮。這題常見錯誤包括忘記自反閉包、把無向圖規則套到有向圖、或把權重矩陣當成可達矩陣。

Hash Table 的背景是用 hash function 把 key 映射到表格位置，理想情況下搜尋、插入、刪除接近 O(1)。本題是 Quadratic Probing，屬於 open addressing，也就是所有元素都直接存放在同一個 hash table 陣列中，不另外建立 linked list。當 h(key) 算出的 home address 已經被占用，就依照探測序列繼續找下一個候選位置。不同探測法的差異在於下一個位置怎麼算：linear probing 用 +1、+2、+3；quadratic probing 則用平方偏移，例如 h(key)+i^2 或 h(key)+c1*i+c2*i^2，再對 table size 取 modulo。

Quadratic Probing 想改善 linear probing 的 primary clustering 問題。Linear probing 若連續位置被占用，很容易形成一大段 cluster，使後續碰撞越來越多；Quadratic probing 因為跳躍距離是平方，探測位置比較分散，可以降低 primary clustering。不過它仍可能有 secondary clustering：如果兩個 key 的 home address 一樣，它們後續探測序列也會一樣。因此題目要求手算時，必須嚴格按照給定公式與 table size，不可以自己換成別的 probing 方式。

手算插入流程是每插入一個 key 就先算 h(key)。若位置空，就直接放入；若不空，就令 i=1、2、3……依序試 h(key)+i^2 mod m，直到找到空格。每一次碰撞與每一次探測都要記錄，因為考題可能要求列出 probing sequence。若 table size m 選得不好，quadratic probing 不一定能探測到所有位置，所以教材常強調表格大小最好是質數，且負載因子 load factor 不要太高。若負載因子太高，即使平均仍比線性探測好，實際探測次數也會快速增加。

刪除背景也要知道：open addressing 裡不能把刪除位置直接清成 empty，否則搜尋可能在中途誤以為 key 不存在。通常要用 deleted marker 或 tombstone，表示該位置曾經有元素但已刪除，搜尋時要繼續探測，插入時可視規則重用該位置。雖然本題可能只考插入，但理解 deleted marker 可以幫你解釋 hash table 的操作限制。作答時最常見錯誤是 modulo 算錯、i 從 0 或 1 搞混、碰撞後忘記回到同一個原始 h(key) 加平方偏移、或把 quadratic probing 寫成 linear probing。只要每一個 key 都列出 home address、碰撞、探測序列、最後位置，就能得到完整分數。

Step 1

一、題目在問什麼

這題要做專案排程。正向推算最早開始時間，反向推算最晚開始時間，再用 slack 判斷 critical path。

二、必備背景知識

AOV/AOE Network 與 critical path 的背景來自有向無環圖 DAG。AOV network 通常把頂點視為活動，邊表示先後關係；AOE network 則常把邊視為活動、頂點視為事件。考試題若問 EST、LST、Critical Path，通常核心是專案排程：哪些活動必須等前置活動完成，哪些活動延誤會導致整體工期延誤。無論題目採用頂點活動或邊活動，第一步都要確認圖不能有 cycle，因為有 cycle 就無法建立合法的拓樸順序，也無法定義從開始到結束的有限排程。

EST 是 earliest start time 或 earliest event time 的概念，表示某活動或事件最早可以開始的時間。計算 EST 通常要依照拓樸順序由前往後做 forward pass。對每一個節點 v，EST[v] 等於所有前驅 u 的 EST[u] 加上邊或活動時間後的最大值，因為 v 必須等所有前置條件都完成才能開始。這裡用最大值不是最小值，因為最晚完成的前置活動才是真正限制 v 開始的瓶頸。

LST 是 latest start time 或 latest event time，表示不延誤總工期的最晚開始時間。計算 LST 要由終點往回做 backward pass，通常先把終點的 LST 設為整體最短完工時間，也就是 forward pass 得到的最大 EST。接著反向依拓樸順序更新，對每個節點 u，LST[u] 等於所有後繼 v 的 LST[v] 減去活動時間的最小值，因為 u 必須早到能讓所有後續工作不延誤。這裡用最小值，因為最緊的後繼限制會決定 u 最晚能拖到什麼時候。

Critical Path 是 slack 為 0 的路徑。Slack 或 float 可理解成 LST-EST，若某活動或事件的 slack 為 0，代表沒有可延誤時間；只要它延誤，整個專案完成時間就會延誤。作答時建議先畫拓樸順序，再列表計算 EST/LST，最後圈出 EST=LST 的節點或活動，串成 critical path。常見錯誤是 forward pass 用最小值、backward pass 用最大值、沒有先做拓樸排序、或把 critical path 當成邊數最多的路徑。其實 critical path 是總時間最長的限制路徑，不一定是節點數最多。若題目給權重，必須用權重加總，而不是只數幾條邊。

        for (char p: pred[u]) EST[u] = max(EST[u], EFT[p]);

            LFT[u] = INT_MAX;
            for (char v: succ[u]) LFT[u] = min(LFT[u], LST[v]);

二、必備背景知識

True / False 題 的背景通常橫跨多個資料結構觀念，不能只背單一句子。Prim、Kruskal、Dijkstra、Union-Find、Dictionary、Heap 都是資料結構期末常見核心。判斷題的技巧是先找出命題中的限制條件，例如「一定」、「所有」、「只能」、「負權重也可以」、「時間一定是」這類字眼。很多敘述不是完全錯在演算法名稱，而是錯在適用條件、資料結構選擇、或時間複雜度前提。

Prim 與 Kruskal 都是求 minimum spanning tree 的 greedy algorithm，但操作方式不同。Prim 從一個起點開始，每次選擇連接已選集合與未選集合的最小邊，常搭配 priority queue。Kruskal 則把所有邊依權重排序，從小到大選邊，只要不形成 cycle 就加入 MST，因此需要 Union-Find 判斷兩端點是否已在同一集合。兩者都要求處理的是無向連通加權圖；若圖不連通，得到的是 minimum spanning forest。判斷題若把 MST 套到有向圖或最短路徑，就要小心。

Dijkstra 是單源最短路徑演算法，前提是邊權重不可為負。它每次選出目前距離估計最小且尚未確定的頂點，將其距離固定，再 relax 鄰接邊。若有負權重邊，即使沒有負環，Dijkstra 的 greedy 決策也可能過早固定錯誤距離。若題目問所有點對最短路徑，就不是單純 Dijkstra 一次，而可能要對每個 source 跑一次，或使用 Floyd-Warshall。若圖無權重，BFS 就能求最短邊數路徑。

Union-Find 又稱 disjoint set，支援 Find 與 Union。若使用 union by rank/size 加 path compression，攤還時間非常接近常數，常寫成 O(alpha(n))。它常用在 Kruskal、connected components、cycle detection。Dictionary 是抽象資料型態，表示 key-value 或 key-item 搜尋結構，可以用 sorted array、BST、AVL tree、hash table 等實作，不等於某一種固定資料結構。Heap 則是 complete binary tree 且符合 heap order，max heap 的 root 最大，min heap 的 root 最小；heap 不保證 inorder 有序，也不適合直接搜尋任意 key。判斷題若說 heap 是 binary search tree，通常是錯的。作答時要把「ADT 規格」和「實作資料結構」分開，並檢查演算法是否符合輸入限制。

class Sets {
private:
    vector<int> parent;
public:
    Sets(int n = 0) : parent(n, -1) {}
    int Find(int x) {
        if (parent[x] < 0) return x;
        return parent[x] = Find(parent[x]);
    }
    void Union(int a, int b) {
        int rootA = Find(a), rootB = Find(b);
        if (rootA != rootB) parent[rootB] = rootA;
    }
};

struct Edge {
    int u, v, w;
    bool operator<(const Edge& other) const { return w < other.w; }
};
