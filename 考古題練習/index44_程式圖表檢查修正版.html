核心概念

  • Sequential representation：記憶體連續，random access O(1)，但中間 insert/delete 需要搬移 O(n)。
  • Linked representation：節點不必連續，靠 pointer 串接；若已知插入位置，insert/delete 可以 O(1)。
  • Singly list：每個 node 只有 link 指向下一個。
  • Doubly list：每個 node 有 left/right，可以雙向走訪。
  • Circular list：最後一個 node 指回 header 或 first，常用 header node 簡化空串列特例。

Q1. 為什麼 linked list 在已知位置時 insert/delete 是 O(1)，但搜尋第 k 個元素是 O(n)？
因為 insert/delete 只改 pointer，不需要搬移元素；但 linked list 沒有 index，只能從 first 一個一個走到第 k 個。
Q2. BST 的 inorder traversal 會得到什麼結果？
會得到 key 的遞增排序，因為 BST 規則是 left < root < right。
Q3. BFS 使用哪一種資料結構？DFS 呢？
BFS 使用 queue；DFS 使用 recursion 或 stack。
Q4. 依序插入 20, 15, 30, 5, 40 到 Max Heap，畫出最後 heap。
最後 root 是 40。可能的 array 表示：[40,30,20,5,15]。插入 40 後一路 bubble up 到 root。
Q5. 對圖做 Topological Sort，若 indegree=0 有多個點，怎麼辦？
可以任選一個，因此 topological order 可能不唯一。考試通常接受符合所有先後限制的答案。
Q6. Hash table size=11，h(k)=k mod 11，插入 22, 33, 44，用 quadratic probing。
22→0；33 原本 0 衝突，0+1²=1；44 原本 0 衝突，試 1 也衝突，再試 0+2²=4，所以放 4。
Q7. 給一個 AOE network，要求 EST、LST、Critical Path。解題流程？
先 topological order；正向取 max 算 EST；反向取 min 算 LST；Slack = LST - EST；Slack=0 的邊組成 critical path。
Q8. Sorting 題問：「哪幾個 stable？哪幾個 in-place？」怎麼答？
Stable：Bubble、Insertion、Merge、Radix（若每一位使用 stable sort）。通常不 stable：Selection、Quick、Heap。In-place：Bubble、Insertion、Selection、Quick、Heap；Merge 通常不是。
Q9. Kruskal 和 Prim 最大差異？
Kruskal 從邊開始，排序所有 edges，避免 cycle；Prim 從一個 vertex 開始，逐步擴張 tree，每次選跨出去的最小 edge。

DFS 深度優先搜尋：詳細中文解說

DFS（Depth First Search，深度優先搜尋）的核心精神是「先一路走到底，再回頭找別條路」。在資料結構的圖論題目中，DFS 通常用來判斷連通性、尋找 connected components、建立 DFS tree、計算 articulation point、檢查 cycle，或在樹狀結構中做先序、中序、後序等遞迴走訪。DFS 可以用遞迴實作，也可以用 stack 實作。遞迴版本比較容易寫，因為每一次呼叫函式時，系統呼叫堆疊會自動記住「目前走到哪個節點、接下來還有哪些鄰居尚未檢查」。考試手算時，最重要的是固定鄰接點順序，否則 DFS 結果可能不同。若題目給 adjacency list，就依照 list 順序走；若題目給 adjacency matrix，通常依照編號由小到大檢查。DFS 的時間複雜度在 adjacency list 中是 O(V+E)，因為每個 vertex 會被拜訪一次，每條 edge 最多被檢查一次；若使用 adjacency matrix，則可能是 O(V²)，因為每個節點都要掃描整列矩陣。DFS 常見錯誤是忘記 visited 標記，導致無向圖中 A 走到 B、B 又走回 A，形成無限遞迴。另一個常見錯誤是把「發現節點」和「完成節點」混在一起，像 articulation point 或 topological sort 題目，常需要分清楚 discovery time 與 finish time。簡單記憶：DFS 像走迷宮，看到路就往前走，走不通才退回上一個岔路。

詳細範例一：用 DFS 找圖的走訪順序

假設圖的鄰接表為 A:{B,C}、B:{D,E}、C:{F}、D:{}、E:{}、F:{}，從 A 開始 DFS，並依照字母順序拜訪。第一步拜訪 A，標記 visited[A]=true。A 的第一個鄰居是 B，所以進入 B。B 的第一個鄰居是 D，進入 D；D 沒有鄰居，所以回到 B。接著 B 的下一個鄰居是 E，進入 E；E 沒有鄰居，回到 B，再回到 A。A 的 B 已完成，下一個鄰居是 C，進入 C。C 的鄰居是 F，進入 F，最後完成。走訪順序為 A, B, D, E, C, F。這個範例說明 DFS 不是先把同一層全部看完，而是只要有下一條路，就會繼續往深處走。

BFS 廣度優先搜尋：詳細中文解說

BFS（Breadth First Search，廣度優先搜尋）的核心精神是「先看離起點最近的一圈，再看下一圈」。它和 DFS 最大差異是資料結構不同：DFS 使用 stack 或遞迴，BFS 使用 queue。因為 queue 是先進先出，所以 BFS 會按照距離起點的層次逐層展開。這個特性讓 BFS 特別適合用來求「無權重圖」或「每條邊權重相同」的最短路徑。從起點出發，第一次到達某個節點時，所走的邊數一定最少。BFS 的標準流程是：先把起點標記 visited 並放入 queue；每次從 queue 前端取出一個節點 u，檢查它的所有鄰居 v；如果 v 尚未拜訪，就標記 visited[v]=true，設定 parent[v]=u，也可以設定 distance[v]=distance[u]+1，然後把 v 放入 queue。考試手算 BFS 時，常需要畫出 BFS tree，也就是每個節點第一次被誰發現。BFS 的時間複雜度在 adjacency list 中是 O(V+E)，在 adjacency matrix 中可能是 O(V²)。常見錯誤是等到節點從 queue 取出才標記 visited，這可能導致同一個節點被重複放入 queue；較安全的做法是「一放入 queue 就標記」。簡單記憶：BFS 像水波擴散，從起點一圈一圈往外推。

詳細範例一：用 BFS 求無權重最短距離

假設圖為 A-B、A-C、B-D、C-D、D-E，從 A 出發。初始化 distance[A]=0，queue=[A]。取出 A 後，發現 B 和 C，設定 distance[B]=1、distance[C]=1，queue=[B,C]。取出 B，發現 D，設定 distance[D]=2，queue=[C,D]。取出 C 時也看到 D，但 D 已經被拜訪，所以不更新。取出 D，發現 E，設定 distance[E]=3。最後距離為 A=0、B=1、C=1、D=2、E=3。這說明 BFS 第一次到達 D 時，就是邊數最少的路徑 A-B-D 或 A-C-D，距離都是 2。

BFS 圖解：先拜訪同距離節點，再往下一層。

詳細範例二：用 BFS 建立 BFS Tree

假設圖的鄰接表為 1:{2,3,4}、2:{5}、3:{5,6}、4:{}、5:{}、6:{}，從 1 開始 BFS。先放入 1，取出 1 後依序發現 2、3、4，所以 parent[2]=1、parent[3]=1、parent[4]=1。接著取出 2，發現 5，parent[5]=2。再取出 3，看到 5 但它已被拜訪，所以不改 parent；接著發現 6，parent[6]=3。最後 BFS tree 的邊是 (1,2)、(1,3)、(1,4)、(2,5)、(3,6)。這棵樹代表從起點 1 到各節點的最短邊數路徑。

BFS 圖解：先拜訪同距離節點，再往下一層。

Radix Sort 基數排序：詳細中文解說

Radix Sort（基數排序）是一種不靠兩兩比較大小的排序法，而是把數字拆成一位一位來處理。最常見版本是 LSD Radix Sort，也就是從 Least Significant Digit（最低位、個位數）開始，依序處理十位、百位、千位。每一輪都依照目前位數把元素放入 0~9 的桶子中，再按照桶子順序收回。Radix Sort 的關鍵要求是「每一輪使用穩定排序」，因為低位數排好的相對順序必須被保留，才能讓高位數排序後得到正確結果。如果每一輪桶子收回時破壞原本順序，最後可能錯誤。Radix Sort 適合整數、固定長度字串或可拆成多個 digit/key 的資料。時間複雜度通常寫成 O(d(n+r))，其中 d 是最大位數，n 是資料數量，r 是基底，例如十進位 r=10。若 d 和 r 相對固定，Radix Sort 可以接近 O(n)。它的缺點是需要額外桶子空間，不像 quick sort 或 heap sort 那樣容易做到原地排序。考試最常問的是每一輪後的陣列順序，尤其要注意短數字的缺位視為 0，例如 2 在十位、百位都視為 0。簡單記憶：Radix Sort 像依照身分證號碼最後一碼、倒數第二碼、倒數第三碼逐輪分類，分類時要保持原本排隊順序。

詳細範例一：數字 170,45,75,90,802,24,2,66 排序

第一輪看個位數：170 和 90 進桶 0，802 和 2 進桶 2，24 進桶 4，45 和 75 進桶 5，66 進桶 6。依桶號收回得到 170,90,802,2,24,45,75,66。第二輪看十位數：802 和 2 的十位都是 0，24 的十位是 2，45 是 4，66 是 6，170 和 75 是 7，90 是 9，收回得到 802,2,24,45,66,170,75,90。第三輪看百位數：2,24,45,66,75,90 的百位都是 0，170 是 1，802 是 8，最後得到 2,24,45,66,75,90,170,802。

Radix 圖解：依照位數放入桶子，再依桶號收回。

詳細範例二：固定長度字串排序

假設要排序三個字母代碼：CAB、ABC、BAC、AAA。可以把每個字母視為一個 digit，從最右邊字母開始。第一輪看第 3 個字母：AAA(A)、CAB(B)、ABC(C)、BAC(C)，收回 AAA,CAB,ABC,BAC。第二輪看第 2 個字母：CAB(A)、BAC(A)、AAA(A)、ABC(B)，但同桶中要保持上一輪相對順序，所以 A 桶是 CAB,BAC,AAA，B 桶是 ABC，收回 CAB,BAC,AAA,ABC。第三輪看第 1 個字母：AAA(A)、ABC(A)、BAC(B)、CAB(C)，最後排序為 AAA,ABC,BAC,CAB。這說明 Radix Sort 不只可用在數字，也可用在固定長度文字鍵值。

Radix 圖解：依照位數放入桶子，再依桶號收回。

Dijkstra 最短路徑：詳細中文解說

Dijkstra 演算法用來求單一起點到所有其他節點的最短距離，條件是所有邊權重都不能是負數。它的核心是 greedy：每次從尚未確定的節點中選出目前 dist 最小者，因為沒有負邊，所以這個距離已經不可能再被其他路徑變得更小，可以安全確定。確定後再用它去 relax 鄰居，也就是檢查 dist[u]+w(u,v) 是否小於 dist[v]。若更小，就更新 dist[v] 和 parent[v]。實作上常搭配 priority_queue，讓選最小距離更快。考試手算時要畫表格，欄位包含節點、目前距離、前一個節點、是否確定。常見錯誤是看到一條邊就立即把節點定案；正確做法是只有「目前全體未確定者中距離最小」才能定案。

詳細範例一：基本手算範例

A到B=2,A到C=5,B到C=1,B到D=4,C到D=1。從A出發，先定A，更新B=2,C=5；再定B，C變3,D變6；再定C，D變4；最後D=4，路徑A-B-C-D。

範例圖解：把文字步驟轉成圖形流程。

詳細範例二：考試應用範例

若圖中有負邊 A到B=2,A到C=5,C到B=-10，Dijkstra 可能先把B=2定案，但其實A-C-B=-5更短，所以不能用 Dijkstra，應改 Bellman-Ford。

Bellman-Ford 最短路徑：詳細中文解說

Bellman-Ford 也是單一起點最短路徑演算法，但它可以處理負權重邊，並且可以偵測 negative cycle。它的做法不是每次貪心選最小點，而是重複對所有邊做 relax。若圖有 V 個節點，沒有負環時，最短路徑最多包含 V-1 條邊，因此對所有邊重複 relax V-1 輪即可。第 V 輪如果還能更新，代表存在負環，因為正常最短路徑不該需要超過 V-1 條邊。它比 Dijkstra 慢，時間複雜度 O(VE)，但適用範圍更廣。

詳細範例一：基本手算範例

邊 A-B=4,A-C=5,C-B=-3。初始化A=0,B=∞,C=∞。第一輪 relax 後 C=5,B=4，再由 C-B 更新 B=2。第二輪不再改變，所以最短距離B=2，路徑A-C-B。

範例圖解：把文字步驟轉成圖形流程。

詳細範例二：考試應用範例

若 A-B=1,B-C=-2,C-B=-2，B與C形成負環，每繞一次距離更小。第V輪仍可更新，Bellman-Ford 判定存在 negative cycle，無法給正常最短路徑。

範例圖解：把文字步驟轉成圖形流程。

Floyd-Warshall 全點對最短路徑：詳細中文解說

Floyd-Warshall 用來求所有點到所有點的最短距離。它使用動態規劃概念，逐步允許中繼點 k。核心公式是 D[i][j]=min(D[i][j],D[i][k]+D[k][j])。意思是：從 i 到 j 的路徑，要不要經過 k？如果經過 k 比原本更短，就更新。它使用三層迴圈，時間複雜度 O(V^3)，適合節點數不太大但需要所有 pair 距離的題目。

詳細範例一：基本手算範例

A到B=3,B到C=4,A到C=10。初始 A到C=10。當k=B時，檢查 A到B+B到C=7，比10短，所以更新 A到C=7。

若有四個城市，每一格矩陣代表直接距離，Floyd 會逐一把每個城市當中繼站，最後得到任兩城市間最短距離表，適合交通網路題。

Topological Sort 拓樸排序：詳細中文解說

Topological Sort 適用於 DAG，也就是有向無環圖。它要找一個排列，使每條邊 u->v 都滿足 u 出現在 v 前面。常用方法是 indegree 法：先把 indegree=0 的節點放入 queue，取出後刪除它的 outgoing edges，讓鄰居 indegree 減一，若變0就入 queue。若最後取出的節點數小於 V，代表有 cycle。拓樸排序常用於課程先修、工作排程、編譯相依性。

Critical Path 關鍵路徑：詳細中文解說

Quick Sort 快速排序：詳細中文解說

Quick Sort 也用 divide and conquer，選 pivot 後做 partition，把小於 pivot 放一邊，大於 pivot 放另一邊，再遞迴排序左右區間。平均 O(n log n)，但若 pivot 選得很差，例如已排序資料每次選第一個，最壞 O(n²)。Quick sort 通常不穩定，但可原地排序。考試常要求追蹤 partition 後結果。

詳細範例一：基本手算範例

陣列 [5,2,8,1,3] 選5為pivot，partition後可能得到 [2,1,3,5,8]，5左邊都小於5，右邊都大於5，再分別排左右。

Insertion Sort 插入排序：詳細中文解說

Insertion Sort 像整理撲克牌，左邊維持已排序區，右邊逐一拿 key 插入正確位置。若資料幾乎已排序，它非常快，best O(n)，average/worst O(n²)。它是 stable 且 in-place。考試常問每一 pass 後狀態。

詳細範例一：基本手算範例

[5,2,4]：先看2，5往右移，插入2，得[2,5,4]；再看4，5往右移，插入4，得[2,4,5]。

Bubble Sort 泡沫排序：詳細中文解說

Bubble Sort 反覆比較相鄰元素，若順序錯就交換。每一輪會把最大值泡到右端。若加上 swapped flag，在已排序情況可提早停止，best O(n)；否則通常 O(n²)。它 stable 且 in-place，但效率較差。

詳細範例一：基本手算範例

[4,1,3,2] 第一輪比較4和1交換得[1,4,3,2]，4和3交換得[1,3,4,2]，4和2交換得[1,3,2,4]，最大4到最後。

Selection Sort 選擇排序：詳細中文解說

Selection Sort 每一輪從未排序區找最小值，放到目前位置。它的交換次數少，但比較次數固定 O(n²)，即使資料已排序也一樣。Selection sort 通常不 stable，但 in-place。考試常問為何不穩定：因為最小值和前面元素交換可能跨過相同 key。

#include <iostream>
#include <vector>
using namespace std;

vector<vector<int>> adj;
vector<bool> visited;

void DFS(int u){
    visited[u] = true;
    cout << u << " ";
    for(int v : adj[u]){
        if(!visited[v]){
            DFS(v);
        }
    }
}

    DFS(0);

}

BFS

#include <iostream>
#include <vector>
#include <queue>
using namespace std;

int main(){
    vector<vector<int>> adj = {
        {1,2}, {0,3}, {0,3}, {1,2,4}, {3}
    };

    vector<bool> visited(adj.size(), false);
    vector<int> dist(adj.size(), -1);
    queue<int> q;

    int start = 0;
    visited[start] = true;
    dist[start] = 0;
    q.push(start);

    cout << "BFS order: ";
    while(!q.empty()){
        int u = q.front();
        q.pop();
        cout << u << " ";

        for(int v : adj[u]){
            if(!visited[v]){
                visited[v] = true;       // 放入 queue 時就標記
                dist[v] = dist[u] + 1;   // 無權重最短距離
                q.push(v);
            }
        }
    }

    cout << "\nDistance from 0:\n";
    for(int i=0;i<(int)dist.size();i++){
        cout << i << " : " << dist[i] << endl;
    }
    return 0;
}

Max Heap Push

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

// 1-based Max Heap：heap[0] 不使用，方便 parent=i/2
vector<int> heap = {-1};

void pushHeap(int x){
    heap.push_back(x);
    int i = heap.size() - 1;

    while(i > 1 && heap[i/2] < heap[i]){
        swap(heap[i/2], heap[i]);
        i /= 2;
    }
}

int main(){
    pushHeap(20);
    pushHeap(15);
    pushHeap(30);
    pushHeap(5);
    pushHeap(40);

    cout << "Max Heap array: ";
    for(int i=1;i<(int)heap.size();i++){
        cout << heap[i] << " ";
    }
    cout << endl;
    return 0;
}

Dijkstra

#include <iostream>
#include <vector>
#include <queue>
#include <utility>
#include <functional>
using namespace std;

const int INF = 1e9;

vector<int> dijkstra(int start, const vector<vector<pair<int,int>>>& adj){
    int n = adj.size();
    vector<int> dist(n, INF);

    priority_queue<
        pair<int,int>,
        vector<pair<int,int>>,
        greater<pair<int,int>>
    > pq;

    dist[start] = 0;
    pq.push({0, start});

    while(!pq.empty()){
        int du = pq.top().first;
        int u  = pq.top().second;
        pq.pop();

        if(du != dist[u]) continue;

        for(const auto& edge : adj[u]){
            int v = edge.first;
            int w = edge.second;

            if(dist[u] + w < dist[v]){
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
        }
    }
    return dist;
}

int main(){
    vector<vector<pair<int,int>>> adj(4);
    adj[0].push_back({1,2}); // A->B
    adj[0].push_back({2,5}); // A->C
    adj[1].push_back({2,1}); // B->C
    adj[1].push_back({3,4}); // B->D
    adj[2].push_back({3,1}); // C->D

    vector<int> dist = dijkstra(0, adj);

    for(int i=0;i<(int)dist.size();i++){
        cout << "0 到 " << i << " 最短距離 = " << dist[i] << endl;
    }
    return 0;
}

Topological Sort

#include <iostream>
#include <vector>
#include <queue>
using namespace std;

vector<int> topoSort(vector<vector<int>> adj, vector<int> indeg){
    int n = adj.size();
    queue<int> q;

    for(int i=0;i<n;i++){
        if(indeg[i] == 0) q.push(i);
    }

    vector<int> order;
    while(!q.empty()){
        int u = q.front();
        q.pop();
        order.push_back(u);

        for(int v : adj[u]){
            indeg[v]--;
            if(indeg[v] == 0){
                q.push(v);
            }
        }
    }
    return order;
}

int main(){
    // 0->1, 0->2, 1->3, 2->3
    vector<vector<int>> adj = {{1,2},{3},{3},{}};
    vector<int> indeg = {0,1,1,2};

    vector<int> order = topoSort(adj, indeg);

    cout << "Topological order: ";
    for(int x : order) cout << x << " ";
    cout << endl;
    return 0;
}
