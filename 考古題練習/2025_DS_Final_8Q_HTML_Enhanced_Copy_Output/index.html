Q1：Radix Sort：LSD 基數排序、時間複雜度與穩定性
對應投影片 lec7 Sorting：基數排序屬排序章節補充；也需搭配穩定排序觀念。若使用 lec7_without_radix_sort，Radix Sort 可能是老師額外補充題型。

題目圖片

題目中文翻譯

給定固定長度 3 的字串列表 [cat, bat, ant, car, bar, arc, cab]，使用 LSD Radix Sort 從最後一個字元開始排序。每一輪必須使用 stable sort，並列出每一輪排序後的結果；另外回答時間複雜度與穩定性為什麼重要。

一、題目在問什麼

這題不是只問最後排序結果，而是要你完整模擬 LSD Radix Sort 三個 pass：先看第 3 個字元，再看第 2 個字元，最後看第 1 個字元。還要知道 counting sort 當子程序時的複雜度，以及為什麼每一輪都一定要穩定。

二、必備背景知識

Radix Sort 的核心是「多關鍵字排序」。LSD 代表 Least Significant Digit，也就是從最不重要的位元或字元開始排。因為前一輪排序結果要被後一輪保留，所以每一輪都必須 stable。穩定排序的意思是：如果兩個元素在目前比較鍵相同，它們原本的相對順序不能被改變。

三、解題思維

看到固定長度字串與 LSD，就先標出每個字串三個位置：第 2 位是最後字元、第 1 位是中間字元、第 0 位是第一字元。每一輪只看一個字元，但同字元組內保持上一輪順序。

五、逐步解答

階段排序依據結果
初始-cat, bat, ant, car, bar, arc, cab

時間複雜度：若有 n 個字串、每個長度 k、字元範圍大小 r，每一輪 counting sort 為 O(n+r)，共 k 輪，所以總時間為 O(k(n+r))。若 r 視為常數，則為 O(kn)。

穩定性：LSD Radix Sort 必須穩定，因為低位排序後形成的相對順序，要在高位相同時被保留下來。若不穩定，像 car 與 cab 在第一字元同為 c 時，前面已排好的後面字元順序可能被破壞。

六、最終答案

最終排序結果：ant, arc, bar, bat, cab, car, cat。時間複雜度 O(k(n+r))，穩定排序是 LSD Radix Sort 正確性的必要條件。

七、常見錯誤

常見錯誤：只寫最後答案、Pass 順序從第一個字元開始、同字元時沒有維持 stable order、把 r 忽略成 O(kn) 但沒有說明 r 為常數。

八、考試得分關鍵

滿分關鍵：三個 pass 的中間結果都要寫；時間複雜度要寫 O(k(n+r))；穩定性要能舉例說明。
背景程式資料補充： LSD Radix Sort 的程式觀念可以想成「外層迴圈控制位數、內層穩定排序控制同一位字元」。如果用 counting sort 實作，會先統計每個字元出現次數，再轉成 prefix sum，最後由右往左把資料放到輸出陣列，這樣可以維持 stable。題目裡的 r 表示字元範圍，例如 ASCII 可視為 128 或 256；k 是字串長度；n 是字串數量。
依照流程圖逐步講解：
1. 先確認每個字串長度都是 3，所以只需要做三輪。
2. 第一輪看最後一個字元，目的不是得到最終字典序，而是先把最低位整理好。
3. 第二輪看中間字元，同字元時必須維持上一輪順序。
4. 第三輪看第一個字元，最後才得到真正 lexicographic order。
5. 回答複雜度時，不要只寫 O(n)，因為還有 k 輪與 r 個 bucket。

程式輸出結果

必要 C++ 程式碼 / 驗證程式

#include <bits/stdc++.h>
using namespace std;

// LSD Radix Sort：固定長度字串，由最後一個字元排到第一個字元。
// stable_sort 用來保證每一輪排序穩定。
int main() {
    vector<string> a = {"cat","bat","ant","car","bar","arc","cab"};
    int k = 3;
    cout << "Initial: ";
    for (auto &s:a) cout << s << " ";
    cout << "\n";

    for (int pos = k - 1; pos >= 0; --pos) {
        stable_sort(a.begin(), a.end(), [pos](const string& x, const string& y) {
            return x[pos] < y[pos];
        });
        cout << "After pass at position " << pos << ": ";
        for (auto &s:a) cout << s << " ";
        cout << "\n";
    }
}

題目中文翻譯

給定無向圖 V={A,B,C,D}，E={(A,B),(A,C),(B,C),(C,D)}。說明 Adjacency Multilist 對無向圖的優點，畫出或描述此圖的 Multilist 結構，以及畫出 Sequential Representation of Adjacency List。

一、題目在問什麼

這題在比較無向圖的兩種儲存方式。重點是 Adjacency Multilist 讓一條無向邊只存一次，但仍能從兩端點各自串到這條邊。

二、必備背景知識

一般 adjacency list 對無向圖會把每條邊存兩次，例如 (A,B) 會出現在 A 的串列，也會出現在 B 的串列。Adjacency Multilist 則用 edge node 記錄 ivex、jvex、ilink、jlink，因此同一個 edge node 可以同時屬於兩個 vertex 的 incident edge list。

三、解題思維

先列出所有頂點與邊，再決定每個頂點的第一條 incident edge。對 multilist 而言，要確認每條邊只建立一個 edge node；對 adjacency list 而言，要把無向邊展開成雙向鄰接。

六、最終答案

Adjacency Multilist 的優點是無向邊只需存一次，節省空間並方便刪除邊；一般 adjacency list 則會將每條無向邊存兩份。

七、常見錯誤

常見錯誤：把 multilist 畫成一般 adjacency list；一條無向邊建立兩個 edge node；ilink 與 jlink 沒有依端點分別連接。

八、考試得分關鍵

滿分關鍵：要明確說出 edge node 欄位與「一條邊只存一次」；鄰接串列要列出 A、B、C、D 四個頂點。
背景程式資料補充： 圖形的程式表示通常會先把頂點轉成 index，例如 A=0、B=1、C=2、D=3。一般 adjacency list 對無向圖會雙向加入，因此 (A,B) 會加入 A 的 list，也會加入 B 的 list。Adjacency Multilist 則比較像「邊節點串列」，每條邊只建立一次，透過 ilink/jlink 分別接到兩個端點的 incident edge list。
依照流程圖逐步講解：
1. 先列出所有頂點與所有邊，避免漏邊。
2. 若是普通 adjacency list，無向邊要雙向存。
3. 若是 multilist，每條邊只建立一個 edge node。
4. 每個 vertex header 指向第一條 incident edge。
5. 回答優點時要明確講「節省重複邊儲存、刪除邊較方便」。

Q3：Biconnected Components：DFS Tree、dfn、low、BFS Tree
對應投影片 lec6a Graphs Part I：DFS、BFS、Connected/Biconnected Components、Articulation Point、dfn/low 約後半段頁面。

題目圖片

題目中文翻譯

從 node 1 開始，每次選擇未拜訪且 index 最小的節點，畫 DFS tree；找 backward edges；列出每個節點的 dfs number 與 low；找出所有 biconnected components；再用同樣選點規則畫 BFS tree 與 cross edge。

一、題目在問什麼

這題在考 DFS low value 的完整手算流程。不是只要 DFS 順序，而是要從 DFS tree 推出 back edge、dfn、low，再用 low 判斷 biconnected components。

二、必備背景知識

dfn 是 DFS 拜訪順序。low(w) 是 w 或 w 的子孫透過 tree edge 加最多一條 back edge 能回到的最小 dfn。若某子樹 low(child) >= dfn(parent)，parent 與該子樹之間可能形成 articulation/biconnected component 的切割點。

三、解題思維

先依最小 index 規則排出每個節點鄰居順序，再做 DFS。DFS tree 畫好後，非 tree edge 若連到祖先就是 back edge。接著由葉節點往回算 low。

七、常見錯誤

常見錯誤：DFS 沒有遵守最小 index；把 undirected parent edge 當 back edge；low 值由上往下算；BFS cross edge 漏掉 (6,7)。

八、考試得分關鍵

滿分關鍵：DFS tree、back edge、dfn/low 表、BCC、BFS tree 都要一致，否則後面答案會全部連錯。
背景程式資料補充： DFS 的 dfn/low 題通常可用遞迴函式完成。dfn[u] 是 u 第一次被拜訪的時間戳；low[u] 是 u 的子樹可以透過 back edge 回到的最小 dfn。程式裡 parent[v]=u 是為了避免把無向圖中回到父節點的邊誤判成 back edge。
依照流程圖逐步講解：
1. 把每個點的鄰居排序，才能符合「選最小 index」規則。
2. DFS 一進入節點就設定 dfn 和 low。
3. 遇到未拜訪節點，該邊是 tree edge。
4. 遇到已拜訪且不是 parent 的祖先節點，該邊是 back edge。
5. 遞迴回來後用 child 的 low 更新 parent 的 low。
6. 最後用 low 條件切出 biconnected components。

必要 C++ 程式碼 / 驗證程式

#include <bits/stdc++.h>
using namespace std;

vector<vector<int>> adj;
vector<int> dfn, low, parent;
int timer = 0;

void dfs(int u) {
    dfn[u] = low[u] = ++timer;
    for (int v : adj[u]) {
        if (!dfn[v]) {
            parent[v] = u;
            dfs(v);
            low[u] = min(low[u], low[v]);
        } else if (v != parent[u]) {
            low[u] = min(low[u], dfn[v]); // back edge
        }
    }
}

int main() {
    int n = 8;
    adj.assign(n+1, {});
    vector<pair<int,int>> edges = {{1,2},{1,3},{2,3},{2,5},{2,8},{5,4},{5,7},{4,6},{6,7}};
    for (auto [u,v] : edges) { adj[u].push_back(v); adj[v].push_back(u); }
    for (int i=1;i<=n;i++) sort(adj[i].begin(), adj[i].end());

    dfn.assign(n+1,0); low.assign(n+1,0); parent.assign(n+1,-1);
    dfs(1);

    for (int i=1;i<=n;i++)
        cout << "v=" << i << " dfn=" << dfn[i] << " low=" << low[i] << "\n";
}
Q4：Threaded Binary Tree：Inorder 與 Preorder Thread
對應投影片 lec5_a Trees Part I：Threaded Binary Trees、Inorder Traversal、Preorder Traversal、Thread 欄位說明約後段頁面。

題目圖片

題目中文翻譯

給定一棵二元樹與 threaded binary tree 的資料結構。回答 n-node threaded binary tree 有多少 threads，並分別畫出 In-order 與 Pre-order 的 threaded binary tree 狀態。

一、題目在問什麼

這題要你理解 thread 是拿原本 NULL child pointer 來指向 traversal predecessor 或 successor。不同 traversal order，thread 指向會不同。

二、必備背景知識

一棵有 n 個節點的 binary tree 有 2n 個 child pointer，但真正的邊只有 n-1 條，所以 NULL pointer 數為 2n-(n-1)=n+1。Threaded binary tree 就是把這些 NULL pointer 改成 threads。

三、解題思維

先寫 traversal sequence，再對每個沒有 left child 或 right child 的節點補 predecessor/successor thread。Inorder 與 Preorder 必須分開做，不能混用。

六、最終答案

七、常見錯誤

常見錯誤：把 inorder thread 與 preorder thread 混在一起；忘記 dummy head；把 real child 也畫成 thread。

八、考試得分關鍵

滿分關鍵：先寫 traversal sequence；再對每個 NULL left/right pointer 補 predecessor/successor；寫出 n+1 的推導。
背景程式資料補充： Threaded binary tree 的程式結構通常包含 leftChild、rightChild、leftThread、rightThread。當 leftThread=true 時，leftChild 不是真正左子，而是 predecessor thread；當 rightThread=true 時，rightChild 不是真正右子，而是 successor thread。不同 traversal order 會造成 predecessor/successor 不同。
依照流程圖逐步講解：
1. 先不要急著畫 thread，先寫出 traversal sequence。
2. Inorder 依 left-root-right；Preorder 依 root-left-right。
3. 找每個節點缺少的左或右 child pointer。
4. 缺左 child 就連到 predecessor；缺右 child 就連到 successor。
5. dummy head 通常用來處理最前與最後節點的 thread。

Q5：Transitive Closure Graph：Warshall 演算法與 Reflexive Closure
對應投影片 lec6b Graphs Part II：Shortest Paths and Transitive Closure、Warshall/Transitive Closure 約前中段頁面。

題目圖片

題目中文翻譯

給定 directed graph G=(V,E)，V={A,B,C}，E={(A,B),(B,C)}。寫出 adjacency matrix A，計算 transitive closure matrix T，計算 reflexive transitive closure matrix R，並解釋 T 與 R 的差別。

一、題目在問什麼

這題在問可達性矩陣。A 是一步可達；T 是走一條以上路徑可達；R 是 T 再加上每個節點到自己的可達性。

二、必備背景知識

Transitive closure 記錄任兩點之間是否存在 path。Warshall algorithm 使用 a[i][j] = a[i][j] OR (a[i][k] AND a[k][j])，表示若 i 可到 k 且 k 可到 j，則 i 可到 j。

三、解題思維

先按照 A,B,C 的順序建立 3x3 adjacency matrix，再利用 A→B→C 推出 A→C。最後將 diagonal 設為 1 得到 reflexive transitive closure。

七、常見錯誤

常見錯誤：忘記 A→C；把 T 的 diagonal 也全部設成 1；沒有說清楚 T 與 R 的差別。

八、考試得分關鍵

滿分關鍵：頂點順序要固定；T 要包含間接可達；R 要比 T 多 reflexive diagonal。
背景程式資料補充： Transitive closure 可以用 Warshall 演算法。程式中的三層迴圈 k、i、j 含義是：是否允許節點 k 當中繼點，若 i 可以到 k 且 k 可以到 j，則 i 可以到 j。Reflexive transitive closure 只是再加入每個節點到自己的長度 0 路徑。
依照流程圖逐步講解：
1. 固定頂點順序 A,B,C，矩陣才不會混亂。
2. A 矩陣只放直接邊：A→B、B→C。
3. 因為 A→B 且 B→C，所以 transitive closure 補上 A→C。
4. T 不一定包含自己到自己。
5. R 是 reflexive，所以 diagonal 全部變成 1。

程式輸出結果

Transitive closure T: 0 1 1 0 0 1 0 0 0 Reflexive transitive closure R: 1 1 1 0 1 1 0 0 1

必要 C++ 程式碼 / 驗證程式

#include <bits/stdc++.h>
using namespace std;

int main() {
    // A=0, B=1, C=2
    int n = 3;
    vector<vector<int>> T(n, vector<int>(n,0));
    T[0][1] = 1; // A -> B
    T[1][2] = 1; // B -> C

    for (int k=0;k<n;k++)
        for (int i=0;i<n;i++)
            for (int j=0;j<n;j++)
                T[i][j] = T[i][j] || (T[i][k] && T[k][j]);

    cout << "Transitive closure T:\n";
    for (auto &row:T) {
        for (int x:row) cout << x << " ";
        cout << "\n";
    }

    cout << "Reflexive transitive closure R:\n";
    for (int i=0;i<n;i++) T[i][i] = 1;
    for (auto &row:T) {
        for (int x:row) cout << x << " ";
        cout << "\n";
    }
}

題目圖片

題目中文翻譯

雜湊表大小為 11，h(k)=k mod 11。使用 quadratic probing：h(k,i)=(h(k)+i^2) mod 11，i 從 0 開始。插入 27,18,29,28,39，寫出最後 hash table，並說明 quadratic probing 相對 linear probing 的一個優點與缺點。

一、題目在問什麼

這題要你逐一插入 key，碰撞時依 i=0,1,2... 帶入 h(k,i)。最後要列出 index 0 到 10 的表格。

二、必備背景知識

Open addressing 的碰撞處理會在同一張表中找下一個位置。Linear probing 用 h(k)+i，容易 primary clustering。Quadratic probing 用 h(k)+i^2，可減少 primary clustering，但可能產生 secondary clustering，且不一定探查所有位置。

三、解題思維

每個 key 先算 h(k)=k mod 11。如果該格已被佔用，才開始嘗試 i=1,2,...，直到找到空格。

七、常見錯誤

常見錯誤：i 從 1 開始而漏掉原 hash 位置；39 的 i=2 位置應為 (6+4)%11=10；把 quadratic probing 當 linear probing。

滿分關鍵：每個 key 的 probing 過程都要寫，最後表格要完整列出 0～10。
背景程式資料補充： Quadratic probing 是 open addressing 的一種。插入時不另外建立 linked list，而是在同一張 hash table 中探查下一個位置。公式 h(k,i)=(h(k)+i²) mod m，其中 i=0 表示先看原始 hash 位置，碰撞才試 i=1、i=2。
依照流程圖逐步講解：
1. 建立 11 格的空 hash table。
2. 每個 key 先算 k mod 11。
3. 空格就直接放入。
4. 發生碰撞就依序試 i=1、i=2。
5. 最後表格要列 index 0 到 10，不能只寫有放值的位置。

必要 C++ 程式碼 / 驗證程式

#include <bits/stdc++.h>
using namespace std;

int main() {
    const int m = 11;
    vector<int> table(m, -1);
    vector<int> keys = {27,18,29,28,39};

    for (int k: keys) {
        int h = k % m;
        for (int i=0; i<m; i++) {
            int pos = (h + i*i) % m;
            if (table[pos] == -1) {
                table[pos] = k;
                break;
            }
        }
    }

    for (int i=0;i<m;i++)
        cout << i << ": " << (table[i] == -1 ? string("-") : to_string(table[i])) << "\n";
}

題目圖片

題目中文翻譯

給定活動與時間 A(3), B(2), C(4), D(2), E(1)，先後關係 A→B, A→C, B→D, C→D, D→E。畫 AOV network，求每個活動的 EST 與 LST，找 critical path 與 project completion time。

一、題目在問什麼

這題要做專案排程。正向推算最早開始時間，反向推算最晚開始時間，再用 slack 判斷 critical path。

二、必備背景知識

AOV network 中 vertex 表示 activity，edge 表示先後關係。EST 是所有前置活動完成後最早可開始時間。LST 是不延誤整個專案的最晚開始時間。slack = LST - EST，slack=0 的活動在 critical path 上。

三、解題思維

先畫 DAG，再做 topological order。由起點往終點算 EST/EFT；再從專案完成時間往回算 LFT/LST。

背景程式資料補充： AOV network 的程式實作通常會用 successor list 與 predecessor list。Forward pass 要看所有 predecessor 的完成時間最大值；Backward pass 要看所有 successor 的最晚開始時間最小值。Critical path 是 slack=0 的活動串接而成。
依照流程圖逐步講解：
1. 先畫出活動先後關係 DAG。
2. Forward pass：EST 等於所有前置活動 EFT 的最大值。
3. 專案完成時間是終點活動的最大 EFT。
4. Backward pass：LFT 等於所有後繼活動 LST 的最小值。
5. LST=LFT-duration。
6. slack=0 的節點就是 critical path 上的活動。

#include <bits/stdc++.h>
using namespace std;

int main() {
    map<char,int> dur = {{'A',3},{'B',2},{'C',4},{'D',2},{'E',1}};
    vector<pair<char,char>> edges = {{'A','B'},{'A','C'},{'B','D'},{'C','D'},{'D','E'}};

    map<char, vector<char>> succ, pred;
    for (auto [u,v]: edges) { succ[u].push_back(v); pred[v].push_back(u); }

    vector<char> topo = {'A','B','C','D','E'};
    map<char,int> EST, EFT;
    for (char u: topo) {
        EST[u] = 0;
        for (char p: pred[u]) EST[u] = max(EST[u], EFT[p]);
        EFT[u] = EST[u] + dur[u];
    }
    int project = EFT['E'];

    map<char,int> LST, LFT;
    for (auto it = topo.rbegin(); it != topo.rend(); ++it) {
        char u = *it;
        if (succ[u].empty()) LFT[u] = project;
        else {
            LFT[u] = INT_MAX;
            for (char v: succ[u]) LFT[u] = min(LFT[u], LST[v]);
        }
        LST[u] = LFT[u] - dur[u];
    }

    for (char u: topo)
        cout << u << ": EST=" << EST[u] << " LST=" << LST[u]
             << " slack=" << LST[u]-EST[u] << "\n";
}
