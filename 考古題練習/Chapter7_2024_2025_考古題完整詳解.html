2025 考古題第 1 題：Radix Sort 完整詳解

原題截圖：2025 Final Examination Q1

英文題目重點與中文翻譯

題目給定固定長度 3 的字串：cat, bat, ant, car, bar, arc, cab。要求使用 LSD Radix Sort，從最後一個字元排到第一個字元，每一輪必須使用 ASCII order 且 stable sort。
小題題意配分重點
a模擬每一輪 LSD Radix Sort列出依第 3、2、1 個字元排序後的結果
b時間複雜度n 個長度 k 字串，counting sort 當 stable subroutine
c穩定性說明 Radix Sort 為何需要 stable，並給反例

演算法原理

LSD Radix Sort 的核心不是一次比較整個字串，而是「從低位到高位」逐位排序。因為每一輪都使用穩定排序，所以後面已經排好的低位順序，在排高位時不會被破壞。

手算過程

階段排序依據結果說明
Initial-cat, bat, ant, car, bar, arc, cab原題順序
Pass 1第 3 個字元cab, arc, car, bar, cat, bat, ant依 b, c, r, r, t, t, t；相同字元保持原順序
Pass 2第 2 個字元cab, car, bar, cat, bat, ant, arca 組保持上一輪相對順序，接著 n、r
Pass 3第 1 個字元ant, arc, bar, bat, cab, car, cat最終字典序
最後答案：ant, arc, bar, bat, cab, car, cat。

動畫展示：LSD 每一輪排序

Initial：cat, bat, ant, car, bar, arc, cab

時間複雜度與穩定性

時間複雜度：若每個字串長度為 k，字元範圍為 r，counting sort 每一輪是 O(n+r)，總共 k 輪，所以是 O(k(n+r))。若 r 視為常數 ASCII 256，則可寫為 O(kn)。
穩定性：LSD Radix Sort 必須 stable。若排序第 1 個字元時把同為 c 的 cab、car、cat 相對順序打亂，前面已由第 2、3 字元建立好的順序會被破壞，結果就不一定是字典序。

C++ 程式碼：LSD Radix Sort

#include <bits/stdc++.h>
using namespace std;

// 取得字串 s 在 pos 位置的字元。pos = 0 表示最左邊，pos = 2 表示最右邊。
char keyAt(const string& s, int pos) {
    return s[pos];
}

// 對固定長度字串做 LSD Radix Sort。
// 重點：每一輪必須使用「stable sort」，從最右邊字元排到最左邊字元。
void lsdRadixSort(vector<string>& a, int length) {
    const int R = 256;                 // ASCII 字元種類數
    vector<string> aux(a.size());      // 暫存陣列，用於穩定計數排序

    // d 從 length-1 到 0：由 Least Significant Digit 往左排
    for (int d = length - 1; d >= 0; --d) {
        vector<int> count(R + 1, 0);   // count[c+1] 統計字元 c 的出現次數

        // 1. 統計每個字元出現次數
        for (const string& s : a) {
            count[(unsigned char)keyAt(s, d) + 1]++;
        }

        // 2. 轉成起始位置：count[c] = 字元 c 應該放置的起始 index
        for (int r = 0; r < R; ++r) {
            count[r + 1] += count[r];
        }

        // 3. 依照原本順序放入 aux，因此同字元會保持原相對順序，達成 stable
        for (const string& s : a) {
            int c = (unsigned char)keyAt(s, d);
            aux[count[c]++] = s;
        }

        // 4. 複製回原陣列
        a = aux;
    }
}

int main() {
    vector<string> a = {"cat", "bat", "ant", "car", "bar", "arc", "cab"};
    lsdRadixSort(a, 3);
    for (auto& s : a) cout << s << ' ';
    cout << '\n';
}

題目重點與中文翻譯

題目給一個 1-based array：a[1..7] = 10, 30, 7, 13, 45, 23, 62，a[0] 不使用。右邊給的是 HeapSort 與 Adjust 程式，要求模擬 build max heap、執行排序、增加某個 key，最後修改程式讓輸出變成 nonincreasing order。

演算法原理：Adjust / Heapify

Adjust(root,n) 的假設是：root 的左右子樹都已經是 max heap，只有 root 這個值 e 可能太小。做法是每次找較大的 child 往上補，直到 e 可以放入某個位置。

修改成 nonincreasing order 的方法

原本 Max HeapSort 把最大值換到尾端，因此最後是由小到大。若要得到由大到小，可以把 Adjust 改成 Min Heapify，每次把最小值換到尾端，最後陣列會由大到小。
// 原本 max heap 選較大 child：
if (j < n && a[j] < a[j+1]) j++;
if (e >= a[j]) break;

// 改成 min heap 選較小 child：
if (j < n && a[j] > a[j+1]) j++;
if (e <= a[j]) break;

C++ 程式碼：HeapSort 與 Adjust

#include <bits/stdc++.h>
using namespace std;

// Max Heap 的向下調整：假設 root 的左右子樹已經是 heap，只有 root 可能不合法。
// a 使用 1-based index：a[1] 是 heap root，a[0] 不用。
void Adjust(vector<int>& a, int root, int n) {
    int e = a[root];                 // 暫存 root 的值，最後放到正確位置
    int child = 2 * root;            // root 的左小孩 index

    while (child <= n) {
        // 選較大的 child，因為 Max Heap 父節點要大於等於左右小孩
        if (child < n && a[child] < a[child + 1]) child++;

        // 若 e 已經比最大的小孩大，代表找到位置
        if (e >= a[child]) break;

        // 否則把較大的小孩往上移
        a[child / 2] = a[child];
        child *= 2;                 // 往下一層繼續找
    }
    a[child / 2] = e;               // e 放入最後停下的位置
}

// Heap Sort：用 Max Heap 排成非遞減順序。
void HeapSort(vector<int>& a, int n) {
    // 1. Build heap：從最後一個非葉節點 n/2 開始往上 Adjust
    for (int i = n / 2; i >= 1; --i) Adjust(a, i, n);

    // 2. 每次把最大值 a[1] 交換到未排序區最後，再縮小 heap
    for (int i = n - 1; i >= 1; --i) {
        swap(a[1], a[i + 1]);       // 最大值歸位
        Adjust(a, 1, i);            // 對剩下 a[1..i] 重建 heap 性質
    }
}

int main() {
    vector<int> a = {0, 10, 30, 7, 13, 45, 23, 62}; // a[0] 不用
    HeapSort(a, 7);
    for (int i = 1; i <= 7; ++i) cout << a[i] << ' ';
    cout << '\n';
}

練習 1：Quick Sort Trace

題目：Given array [26, 5, 37, 1, 61, 11, 59, 15, 48, 19]. Use QuickSort with the leftmost pivot. Show the array status after each partition.
答案重點：第一次 pivot=26，partition 後變成 [11,5,19,1,15] 26 [59,61,48,37]。接著遞迴左半與右半，最後為 1,5,11,15,19,26,37,48,59,61。
#include <bits/stdc++.h>
using namespace std;

// 使用左端點作 pivot 的 Quick Sort，對應老師投影片常見版本。
void QuickSort(vector<int>& a, int left, int right) {
    if (left >= right) return;        // 區間長度 0 或 1，不用排序

    int i = left;
    int j = right + 1;
    int pivot = a[left];              // pivot 選左端點

    while (true) {
        do { i++; } while (i <= right && a[i] < pivot);  // 找左邊第一個 >= pivot
        do { j--; } while (a[j] > pivot);                // 找右邊第一個 <= pivot
        if (i >= j) break;
        swap(a[i], a[j]);             // 把錯邊的兩個元素交換
    }

    swap(a[left], a[j]);              // pivot 放到正確位置 j
    QuickSort(a, left, j - 1);        // 遞迴排序左半
    QuickSort(a, j + 1, right);       // 遞迴排序右半
}

int main() {
    vector<int> a = {26, 5, 37, 1, 61, 11, 59, 15, 48, 19};
    QuickSort(a, 0, (int)a.size() - 1);
    for (int x : a) cout << x << ' ';
}

練習 2：Merge Sort Trace

題目：Given array [38,27,43,3,9,82,10]. Use recursive merge sort. Draw the divide-and-conquer tree and show final output.
答案：先拆成 [38,27,43,3] 與 [9,82,10]，繼續拆到單一元素，再合併為 [3,27,38,43] 與 [9,10,82]，最後合併為 [3,9,10,27,38,43,82]。
#include <bits/stdc++.h>
using namespace std;

// 合併兩段已排序區間 a[l..m] 與 a[m+1..r]
void mergeRange(vector<int>& a, vector<int>& tmp, int l, int m, int r) {
    int i = l;       // 左半目前位置
    int j = m + 1;   // 右半目前位置
    int k = l;       // tmp 寫入位置

    while (i <= m && j <= r) {
        // <= 可以保留相同 key 的原相對順序，所以 merge sort 是 stable
        if (a[i] <= a[j]) tmp[k++] = a[i++];
        else tmp[k++] = a[j++];
    }
    while (i <= m) tmp[k++] = a[i++];
    while (j <= r) tmp[k++] = a[j++];

    for (int t = l; t <= r; ++t) a[t] = tmp[t];
}

void mergeSort(vector<int>& a, vector<int>& tmp, int l, int r) {
    if (l >= r) return;
    int m = (l + r) / 2;
    mergeSort(a, tmp, l, m);      // 排左半
    mergeSort(a, tmp, m + 1, r);  // 排右半
    mergeRange(a, tmp, l, m, r);  // 合併
}

int main() {
    vector<int> a = {38, 27, 43, 3, 9, 82, 10};
    vector<int> tmp(a.size());
    mergeSort(a, tmp, 0, (int)a.size() - 1);
    for (int x : a) cout << x << ' ';
}

練習 4：Decision Tree 下界

題目：Why does every comparison sort require Ω(n log n) comparisons in the worst case?
排序 n 個相異元素有 n! 種可能排列。比較排序的每次比較只有 yes/no 兩種結果，因此可視為二元決策樹。若樹高為 h，最多有 2^(h-1) 個葉節點；要容納 n! 個結果，必須 2^(h-1) ≥ n!，所以 h ≥ log₂(n!)+1 = Ω(n log n)。
