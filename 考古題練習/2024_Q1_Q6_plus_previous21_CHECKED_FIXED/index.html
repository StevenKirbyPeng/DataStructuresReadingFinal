第 1 題：Representations of a tree or a forest（樹與森林表示法）

本題必備背景知識：樹、森林、Traversal 與 Degree-two Tree

這類題目核心是「一般樹」和「二元樹表示法」的轉換。一般樹的每個節點可以有任意多個 children；森林則是多棵互不相交的樹。考試常要求用 generalized list 表示，例如 1(2,3,4(5))，括號內代表該節點的孩子，逗號代表兄弟節點，而且孩子順序必須依照圖中由左到右。

Traversal 要先確認題目問的是原本的一般樹，還是轉成二元樹後的走訪。一般樹的 post-order 是「每一棵子樹先走完，最後才走 root」；level-order 是用 queue 由上到下、同層由左到右。森林轉 degree-two tree 使用 left-child right-sibling 規則：第一個孩子接到 leftChild，同層下一個兄弟接到 rightChild，多棵樹的 root 也用 rightChild 串起來。

作答時建議同時給「圖形」與「leftChild/rightChild 對照表」，因為只畫圖容易因線條方向不清楚而被扣分。

解題思維：樹與森林表示法：先看結構，再決定走訪與轉換

先用流程圖決定作答順序，再回到題目圖片逐步填答案。

流程說明 1：先把題目中的三棵一般樹分開，不要一開始就轉成二元樹。T1 的 root 是 1，children 是 2、3、4，而 4 還有 child 5；T2 的 root 是 9，children 是 10、6；T3 的 root 是 7，child 是 8。
流程說明 2：List 表示法只針對 T1，因此答案是 1(2,3,4(5))。括號代表 children，逗號代表同一層兄弟；這一步重點是不能把 5 寫成 1 的孩子。
流程說明 3：Postorder 對一般樹是「先走完每個 child subtree，再輸出 root」，所以 T1 是 2,3,5,4,1。Level-order 是 BFS，先 root，再同層 children，所以 T1 是 1,2,3,4,5。
流程說明 4：森林轉 degree-two tree 時，用 left-child right-sibling：1 的 left 是 2，2 的 right 是 3，3 的 right 是 4，4 的 left 是 5；森林中 T1 root 的 right 接 T2 root 9，9 的 right 接 T3 root 7。

題目中文說明

題目給三棵有序樹 T1、T2、T3。要求：

  1. 用 list / generalized list 表示 T1。
  2. 列出 T1 的 post-order traversal。
  3. 列出 T1 的 level-order traversal。
  4. 把森林 T1、T2、T3 轉成 degree-two tree，也就是 left-child right-sibling binary tree。

完整答案

(a) T1 的 list 表示：

1(2, 3, 4(5))

意思是：1 是 root，1 的 children 依序是 2、3、4，而 4 有 child 5。

(b) T1 的 post-order：

2 → 3 → 5 → 4 → 1

(c) T1 的 level-order：

1 → 2 → 3 → 4 → 5

(d) 森林轉 degree-two tree：

1.left = 2, 1.right = 9
2.right = 3, 3.right = 4, 4.left = 5
9.left = 10, 10.right = 6, 9.right = 7
7.left = 8

為什麼是這樣？

一般樹轉二元樹的規則是：

  1. 每個節點的「第一個 child」放到 leftChild。
  2. 同一層的「下一個 sibling」放到 rightChild。
  3. 森林的多棵樹根節點也視為 siblings，所以 T1 的 root 1 的 rightChild 會接 T2 的 root 9，9 的 rightChild 再接 T3 的 root 7。

這是考樹與森林最常出的核心觀念。

這題不要只畫圖，最好同時寫出 leftChild / rightChild 對應關係。閱卷時這比只有圖更清楚。

第 2 題：MAX Heap（最大堆積、Heapify、Heap Sort）

本題必備背景知識：Max Heap、Adjust 與 Heap Sort

Heap 是一棵「完全二元樹」，通常用 1-indexed array 表示：節點 i 的 left child 是 2i，right child 是 2i+1，parent 是 ⌊i/2⌋。Max Heap 的性質是每個 parent 的 key 都大於或等於 children，因此最大值一定在 root，也就是 a[1]。

Build Max Heap 時不是從 root 開始，而是從最後一個非葉節點 ⌊n/2⌋ 往前做到 1。Adjust 的概念是把 root 暫存為 e，每次從左右孩子中選較大的 child 往上拉，直到 e 可以放入正確位置。Heap Sort 則反覆把 root 最大值與 heap 尾端交換，再縮小 heap 範圍並重新 Adjust。

考試最容易錯的是 index 0 是否使用、heap 範圍是否縮小、以及每一輪 sorted area 不可以再參與 Adjust。

解題思維：MAX Heap：把程式行號轉成 heap 操作

先用流程圖決定作答順序，再回到題目圖片逐步填答案。

流程說明 1：先看程式碼第 12–13 行，這是建立 Max Heap，不是排序。n=7，所以 i 從 3、2、1 依序 Adjust。每次 Adjust 都只修復以 i 為 root 的子樹。
流程說明 2：第 14–17 行才是 Heap Sort：把目前最大值 a[1] 換到最後一格，heap size 減 1，再 Adjust(a,1,i) 讓剩下的元素恢復 Max Heap。這段會讓陣列由小到大排列。
流程說明 3：遇到「把 7 增加成 31」這種題目時，因為 key 變大，可能破壞與 parent 的 Max Heap 關係，所以不是向下 Adjust，而是往上比較 parent，必要時上濾。
流程說明 4：若要 nonincreasing order，使用 MaxHeap 原程式直接排序會得到 increasing order。最簡單的修改思維是把 Adjust 改成 MinHeap 比較規則，讓每次把最小值放到尾端，最後陣列就會由大到小。

(d) 修改成 nonincreasing order

原本 Max Heap + HeapSort 會得到遞增。要得到遞減，可改成 Min Heap，讓最小值每次被換到後面，最後陣列會由大到小。

只需修改 Adjust 裡選 child 與 break 條件：

原本 Max Heap：
if (j < n && a[j] < a[j+1]) j++;
if (e >= a[j]) break;

改成 Min Heap：
if (j < n && a[j] > a[j+1]) j++;
if (e <= a[j]) break;

其他 HeapSort 主流程可保持不變。

第 3 題：證明 Binary Tree Lemma：n0 = n2 + 1

本題必備背景知識：Binary Tree 的節點度數與邊數

Binary tree 中每個節點的 degree 只能是 0、1、2。degree 0 的節點稱為 leaf；degree 2 的節點表示同時有左右兩個 child。這題要證明的是所有非空 binary tree 都滿足 n0 = n2 + 1，也就是葉節點數永遠比 degree-2 節點數多 1。

標準證法通常從「邊數」切入：一棵有 N 個節點的樹一定有 N−1 條邊；另一方面，若用節點的 child 數來數邊，degree 1 節點貢獻 1 條邊，degree 2 節點貢獻 2 條邊，所以邊數也等於 n1 + 2n2。把兩種邊數表示式相等後整理，就能得到結論。

證明題重點不是代數很長，而是要清楚定義 n0、n1、n2，並說明為什麼兩個邊數公式都正確。

解題思維：證明題：從「節點數」與「邊數」兩種計數切入

先用流程圖決定作答順序，再回到題目圖片逐步填答案。

流程說明 1：證明題不要只舉例，必須對任意非空 binary tree 都成立。最穩定的方法是用 edge 數量，因為 tree 的 edge 數一定是節點數減 1。
流程說明 2：把 binary tree 的節點依 degree 分成三類：degree 0、degree 1、degree 2。因此總節點數 N=n0+n1+n2。
流程說明 3：每個 degree 1 節點提供 1 條連到 child 的邊，每個 degree 2 節點提供 2 條連到 children 的邊，leaf 不提供邊，所以 edge=n1+2n2。
流程說明 4：另一方面 tree 的 edge=N−1=n0+n1+n2−1。兩式相等後消去 n1，得到 2n2=n0+n2−1，因此 n0=n2+1。

題目中文說明

要證明：對任何非空 binary tree T，如果 n0 是 leaf nodes 數量，n2 是 degree 2 nodes 數量，則：

n0 = n2 + 1

完整證明：用邊數計算

設：

  • n0：degree 0 的節點數，也就是 leaf。
  • n1：degree 1 的節點數。
  • n2：degree 2 的節點數。

整棵樹的總節點數：

N = n0 + n1 + n2

因為一棵有 N 個節點的樹，一定有 N−1 條邊：

edges = N - 1 = n0 + n1 + n2 - 1

另一方面，在 binary tree 中，每個 degree 1 節點貢獻 1 條往 child 的邊，每個 degree 2 節點貢獻 2 條往 child 的邊，所以：

edges = n1 + 2n2

兩個邊數公式相等：

n0 + n1 + n2 - 1 = n1 + 2n2

左右同減 n1，再同減 n2：

n0 - 1 = n2

因此：

n0 = n2 + 1

證畢。

考試可直接寫的版本

Let n0, n1, n2 be the numbers of nodes with degree 0, 1, and 2.
Total nodes N = n0 + n1 + n2.
A tree with N nodes has N − 1 edges.
Counting edges by node degree gives edges = n1 + 2n2.
Therefore n0 + n1 + n2 − 1 = n1 + 2n2.
So n0 − 1 = n2, hence n0 = n2 + 1.

第 4 題：Binary Search Tree（BST 建立、Rank、刪除）

本題必備背景知識：Binary Search Tree、Rank 與刪除

Binary Search Tree（BST）的規則是：對任一節點 x，左子樹所有 key 小於 x，右子樹所有 key 大於 x。插入時從 root 開始比較，小於往左，大於往右，直到遇到空位置放入新節點。因此同一組數字若插入順序不同，形成的 BST 也可能不同。

Rank 題目常搭配 leftsize。leftsize 通常表示「左子樹節點數加 1」，也就是該節點在自己子樹中依 inorder 排序的名次。找第 k 小時，若 k 等於 leftsize 就找到；若 k 小於 leftsize 往左；若 k 大於 leftsize，往右並把 k 減掉 leftsize。

刪除 BST 節點時分三種情況：leaf 直接刪、只有一個 child 則用 child 接上、兩個 children 時通常用 inorder successor（右子樹最小值）或 predecessor（左子樹最大值）取代。

解題思維：BST：依序插入、用 leftSize 找 rank、刪除二子節點

先用流程圖決定作答順序，再回到題目圖片逐步填答案。

流程說明 1：BST 題目最常錯在「把數字排序後再畫樹」。正確方法是按照題目給的順序逐一插入：10, 7, 30, 57, 23, 99, 47, 20。
流程說明 2：建立完成後，rank 查詢不是看節點值大小直接猜，而是使用 leftSize。leftSize 代表左子樹節點數加上自己，因此找第 k 小時可以用 k 和 leftSize 比較決定往哪邊走。
流程說明 3：本題 inorder 排序是 7,10,20,23,30,47,57,99，因此第 6 小是 47。若用 leftSize 查詢，走訪節點的比較過程要寫出來，才容易拿到完整分數。
流程說明 4：刪除 30 時，30 有左子樹 23 和右子樹 57，因此使用 inorder successor，也就是右子樹最小值 47 取代 30，再刪除原本的 47。

第 5 題：Set / Disjoint Set（Weighted Union 與 Collapsing Find）

本題必備背景知識：Disjoint Set、Weighted Union 與 Path Compression

Disjoint set 用來維護「不相交集合」，常見操作有 Find 與 Union。課本常用 parent array 表示森林：若 parent[i] < 0，代表 i 是 root，負值可用來記錄樹高或集合大小；若 parent[i] ≥ 0，代表 parent[i] 是 i 的父節點。

Weighted Union 的目標是避免樹長太高。若負值代表高度，合併時把高度較小的樹接到高度較大的 root；若兩棵高度相同，接完後新 root 的高度要加 1，因此 parent 的負值會再減 1。CollapsingFind 是路徑壓縮，Find 完 root 後，把沿途節點直接接到 root，讓後續查詢更快。

作答時一定要清楚列出 parent array。圖形只能輔助，最後表格中的每個 parent[i] 才是主要答案。

解題思維：Disjoint Set：先寫 parent array，再照 union/find 程式更新

先用流程圖決定作答順序，再回到題目圖片逐步填答案。

流程說明 1：這題要先把圖轉成 parent array。非 root 節點 parent[i] 是父節點編號；root 節點 parent[i] 是負高度，所以要先判斷哪幾個是 root。
流程說明 2：WeightedUnion(1,6) 不一定把 6 接到 1，也不一定照參數順序接。要照程式比較 parent[1] 與 parent[6]。因為負值越小代表高度越大，所以高度較小的樹要接到高度較大的 root。
流程說明 3：CollapsingFind(0) 只會壓縮從 0 走到 root 的那條路徑。例如 0→5→6，就只會把 0、5 這條路徑上的節點接近 root；其他沒有走過的 2、3、4、7 不會被改。
流程說明 4：作答時建議每小題都列 parent array，因為 disjoint set 的答案最容易因為只畫圖而不清楚。

第 6 題：Selection Tree（Winner Tree 與 Loser Tree）

本題必備背景知識：Selection Tree、Winner Tree 與 Loser Tree

Selection tree 常用於 k-way merge。每個 leaf 對應一個已排序序列的目前元素，internal node 存放比較結果。若要做遞增合併，每次要選出最小值；若要做遞減合併，則選最大值。題目通常會給多個序列，先取每個序列第一個元素放在 leaf，再由下往上比較。

Winner tree 的 internal node 存「勝者」，所以 root 就是目前全體 winner。輸出 winner 後，只需要把該 leaf 換成同一序列的下一個元素，然後沿著 leaf 到 root 的路徑重新比賽。Loser tree 則相反，internal node 存「輸者」，真正 winner 通常另外保存；它的優點是更新時比較次數固定且實作效率好。

考試最常錯的是把 winner tree 與 loser tree 的內部節點意義混淆。請記住：winner tree 存 winner；loser tree 存 loser，champion 另外記。

解題思維：Selection Tree：先決定 winner/loser 存什麼，再更新一條路徑

先用流程圖決定作答順序，再回到題目圖片逐步填答案。

流程說明 1：Selection tree 題目一定要先確認是 winner tree 還是 loser tree。Winner tree 的 internal node 存勝者；Loser tree 的 internal node 存輸者，真正 champion 通常另外記錄。
流程說明 2：本題若做 ascending merge，winner 是較小值。初始 leaf 是 10、20、8、21，因此 winner tree 的 root 會是 8。輸出 8 後，leaf 6 換成同序列下一個元素 15，只需重新比較 leaf 6 到 root 的路徑。
流程說明 3：Loser tree 的寫法要特別注意：內部節點不是存目前最小值，而是每場比較的 loser。因此如果把 loser tree 的 node 當成 winner tree 填，答案會整題相反。
流程說明 4：作答時可以畫 tree，也可以列 node1、node2、node3 的值；若題目要 status，建議兩者至少列出節點值，避免圖中數字不清楚。

題目中文說明

有四個已排序序列放在 leaf 4、5、6、7：

Leaf序列第一個元素
410, 19, 2610
520, 30, 4020
68, 15, 508
721, 35, 3921

假設要做 ascending merge，所以 winner 是較小者。

(a) Winner tree 初始狀態

Leaf values：4=10, 5=20, 6=8, 7=21。

node2 = min(10,20) = 10
node3 = min(8,21)  = 8
node1 = min(10,8)  = 8

Winner tree： node1=8，node2=10，node3=8。

(b) Winner tree 處理下一個元素後

目前 winner 是 leaf6 的 8。輸出 8 後，leaf6 換成下一個元素 15。

leaf6: 8 → 15
node3 = min(15,21) = 15
node1 = min(10,15) = 10

更新後： node1=10，node2=10，node3=15。

(c) Loser tree 初始狀態

Loser tree 的 internal node 存「比較輸掉的人」，真正總 winner 通常另外放在 champion/root 外部。

leaf4 10 vs leaf5 20：winner=10，loser=20 → node2=20
leaf6 8  vs leaf7 21：winner=8， loser=21 → node3=21
winner 10 vs winner 8：winner=8， loser=10 → node1=10

答案： champion=8，node1=10，node2=20，node3=21。

(d) Loser tree 處理下一個元素後

輸出 champion 8，leaf6 換成 15，只需沿 leaf6 到 root 重賽。

leaf6 15 vs node3 loser 21：winner=15，loser=21，node3仍為21
winner 15 vs node1 loser 10：winner=10，loser=15 → node1=15

答案： champion=10，node1=15，node2=20，node3=21。

Q6 動畫：Winner Tree / Loser Tree

考試重點

Winner tree：節點存 winner。Loser tree：節點存 loser，總 winner 另外保存。兩者不要混在一起。
