Q1(a)(b)(c) 樹 T1 的 list 表示、Post-order、Level-order 第 1 張截圖

本題必備背景知識：樹、森林、Traversal 與 Degree-two Tree

這類題目核心是「一般樹」和「二元樹表示法」的轉換。一般樹的每個節點可以有任意多個 children；森林則是多棵互不相交的樹。考試常要求用 generalized list 表示，例如 1(2,3,4(5))，括號內代表該節點的孩子，逗號代表兄弟節點，而且孩子順序必須依照圖中由左到右。

Traversal 要先確認題目問的是原本的一般樹，還是轉成二元樹後的走訪。一般樹的 post-order 是「每一棵子樹先走完，最後才走 root」；level-order 是用 queue 由上到下、同層由左到右。森林轉 degree-two tree 使用 left-child right-sibling 規則：第一個孩子接到 leftChild，同層下一個兄弟接到 rightChild，多棵樹的 root 也用 rightChild 串起來。

作答時建議同時給「圖形」與「leftChild/rightChild 對照表」，因為只畫圖容易因線條方向不清楚而被扣分。

完整答案

小題答案理由
a. List 表示 T11(2,3,4(5))根 1 有三個孩子 2、3、4；節點 4 有孩子 5。
b. Post-order2 → 3 → 5 → 4 → 1後序是先拜訪每個子樹，再拜訪根。4 的子樹要先輸出 5，再輸出 4。
c. Level-order1 → 2 → 3 → 4 → 5層序是由上到下、同層由左到右。

解題步驟

  • 先找 root：圖上 T1 最上方節點是 1。
  • 依照由左到右順序列出 1 的 children：2、3、4。
  • 看到 4 下方還有 5，所以 5 是 4 的 child，不是 1 的 sibling。
  • 廣義串列寫法：root(children)，兄弟用逗號，故為 1(2,3,4(5))。
  • Post-order 對一般樹：每個孩子子樹先完成，最後才寫根。
  • Level-order：使用 queue，先放 1，依序取出時加入它的孩子。
Post-order 流程： visit(1) → visit(2)輸出2 → visit(3)輸出3 → visit(4) → visit(5)輸出5 → 輸出4 → 輸出1。
常見扣分：把 5 寫成 1 的第四個孩子，會導致 list 寫成 1(2,3,4,5)，Post-order 也會錯。

解題規則

  • 一般樹轉二元樹時，每個節點的「第一個孩子」接到 leftChild。
  • 同一層的下一個兄弟接到 rightSibling。
  • 森林有多棵樹時，各棵樹的 root 也用 rightSibling 串起來：T1 root=1，T2 root=9，T3 root=7，所以 1.right=9，9.right=7。

Q2(a) MAX Heap：執行 Build Heap / MAX_Heapify 後陣列 第 3 張截圖

本題必備背景知識：Max Heap、Adjust 與 Heap Sort

Heap 是一棵「完全二元樹」，通常用 1-indexed array 表示：節點 i 的 left child 是 2i，right child 是 2i+1，parent 是 ⌊i/2⌋。Max Heap 的性質是每個 parent 的 key 都大於或等於 children，因此最大值一定在 root，也就是 a[1]。

Build Max Heap 時不是從 root 開始，而是從最後一個非葉節點 ⌊n/2⌋ 往前做到 1。Adjust 的概念是把 root 暫存為 e，每次從左右孩子中選較大的 child 往上拉，直到 e 可以放入正確位置。Heap Sort 則反覆把 root 最大值與 heap 尾端交換，再縮小 heap 範圍並重新 Adjust。

考試最容易錯的是 index 0 是否使用、heap 範圍是否縮小、以及每一輪 sorted area 不可以再參與 Adjust。

完整答案

Build Max Heap 後： [62,45,23,13,30,10,7]

階段動作陣列 index 1..7
原始尚未 heapify10, 30, 7, 13, 45, 23, 62
i=3節點 7 與較大孩子 62 交換10, 30, 62, 13, 45, 23, 7
i=2節點 30 與較大孩子 45 交換10, 45, 62, 13, 30, 23, 7
i=1節點 10 與 62 交換，再與 23 交換62, 45, 23, 13, 30, 10, 7

為什麼從 n/2 開始？

  • 容量 8、index 0 不用，實際元素 n=7。
  • 完全二元樹中 index floor(n/2)+1 到 n 都是 leaf，不需要 Adjust。
  • 所以從 i=floor(7/2)=3 開始，往 i=2、i=1 做 Adjust。
  • 每次 Adjust 都讓以 i 為 root 的子樹滿足 max-heap：parent ≥ children。
// index 0 不用，a[1..n]
for (int i = n/2; i >= 1; --i) {
    Adjust(a, i, n);   // 將 i 為根的子樹調成 max heap
}
// 結果：62 45 23 13 30 10 7

Q3 Free / Available Space List：題目與操作觀念 第 6 張截圖

本題必備背景知識：Available Space List / Static Linked List

Available space list 是用陣列模擬動態記憶體配置的技術。每個陣列格子通常有 data 與 link 欄位；正在使用的節點由 first 串起來，未使用的節點由 avail 串起來。當需要新節點時，從 avail 的開頭取出；當刪除節點時，把該節點接回 avail 的開頭。

配置節點的標準動作是 p = avail; avail = link[p];；釋放節點的標準動作是 link[p] = avail; avail = p;。這些操作都可以在 Θ(1) 完成，但前提是指標更新順序正確，否則會斷鏈或形成 cycle。

作答時要同時畫出 first chain 與 avail chain，並用表格列出每個 index 的 link，才能完整檢查是否有節點重複使用。

完整解答架構

這題考「靜態 linked list / available space list」。答案一定要同時交代：data 欄、link 欄、first 指標、avail 指標。

操作正確作法會改到的欄位
Allocate / new nodep = avail; avail = link[avail];avail 會往下一個空節點移動
Insert after xdata[p]=value; link[p]=link[x]; link[x]=p;新節點 link 與前驅 x 的 link
Delete after xp=link[x]; link[x]=link[p]; link[p]=avail; avail=p;前驅 x 的 link、被刪節點 p 的 link、avail

考試寫法

  • 不要只畫圖，最好再用表格列出每個 index 的 data/link。
  • 每做一步先更新 avail，再接入主串列，否則容易形成 cycle。
  • 刪除節點時，被刪節點不是消失，而是回收到 available list。
  • 最後答案要畫 first → ... → 0 與 avail → ... → 0 兩條鏈。
int GetNode() {
    if (avail == 0) throw "out of space";
    int p = avail;
    avail = link[avail];
    return p;
}
void RetNode(int p) {
    link[p] = avail;
    avail = p;
}

Q7 Threaded Binary Tree：題目總覽 第 16 張截圖

本題必備背景知識：Threaded Binary Tree 與 Inorder Threads

Threaded binary tree 利用二元樹中的空指標來儲存 traversal 的 predecessor 或 successor。一般二元樹有 2n 個 child pointer，但只有 n−1 條真正的邊，所以會有 n+1 個空指標；這些空指標可以改成 threads，讓 inorder traversal 不必使用 stack 或 recursion。

若某節點沒有 left child，leftChild 可作為 predecessor thread；若沒有 right child，rightChild 可作為 successor thread。leftThread/rightThread 欄位用來判斷該指標是真正 child 還是 thread。題目若有 dummy head，通常用來統一 traversal 的起點與終點。

解題時先寫出 inorder 序列，再依序列判斷每個節點的 predecessor 與 successor，最後再處理插入節點時哪些 thread 需要更新。

樹結構判讀

原始樹：A 為 root；左子樹 A-B-C-D，其中 B.left=C，C.right=D；右子樹 A-F-E-G，其中 F.left=E，E.left=G。

Inorder = C D B A G E F

Preorder = A B C D F E G

Threaded tree 基本觀念

  • 若節點沒有 left child，可用 leftChild 欄位作為 inorder predecessor thread。
  • 若節點沒有 right child，可用 rightChild 欄位作為 inorder successor thread。
  • 虛線通常表示 predecessor，實線通常表示 successor；題目圖上也有說明。
  • Dummy head 用來讓 traversal 開始與結束更方便。

Q8 ChainIterator：postdecrement / predecrement 第 19 張截圖

本題必備背景知識：Linked List、Iterator 與 Circular List

Singly linked list 的每個節點只有 data 與 link，link 指向下一個節點。因為沒有 prev 指標，所以 iterator 若要做 -- 往前移動，通常必須從 first 開始掃描，找到目前 current 的前一個節點，時間複雜度是 Θ(n)。

Postdecrement it-- 的語意是「先回傳舊 iterator，再把 iterator 往前移」；predecrement --it 則是「先移動，再回傳自己」。Circular list 的尾端插入要特別注意 last 的定義：若 last 指向最後一個節點，插入時通常先讓 newNode 接到原本第一個節點，再讓原 last 接到 newNode，最後更新 last。

填空題一定要注意更新順序，先保存舊指標再改 link，避免失去原本的節點鏈。

完整填空答案

空格答案理由
[A]current == first 或 current == 0若 iterator 在開頭或空位置，不能再往前。實際類別可能用 current==nullptr。
[B]p->link != current找 current 的前一個節點。
[C]p = p->link沿 linked list 往後掃。
[D]old後置遞減要回傳遞減前的 iterator。
[E]current = p真正把目前 iterator 移到前一個節點。
ChainIterator operator--(int) {      // postdecrement
    if (current == nullptr) throw "empty iterator";
    ChainIterator old = *this;      // 保留舊值，因為後置--要回傳舊狀態
    ChainNode<T>* p = first;
    while (p != nullptr && p->link != current) {
        p = p->link;
    }
    current = p;                    // 移到前一個節點
    return old;
}

重點

  • Postdecrement：先保留 this，移動 current，最後回傳舊值。
  • Predecrement：移動 current 後回傳 *this，因此回傳型態通常是 reference。
  • Singly linked list 沒有 prev pointer，所以 -- 需要從 first 開始找前驅，時間 Θ(n)。

Q10 Available Space Lists：回收節點與配置節點 第 21 張截圖

本題必備背景知識：Available Space List / Static Linked List

Available space list 是用陣列模擬動態記憶體配置的技術。每個陣列格子通常有 data 與 link 欄位；正在使用的節點由 first 串起來，未使用的節點由 avail 串起來。當需要新節點時，從 avail 的開頭取出；當刪除節點時，把該節點接回 avail 的開頭。

配置節點的標準動作是 p = avail; avail = link[p];；釋放節點的標準動作是 link[p] = avail; avail = p;。這些操作都可以在 Θ(1) 完成，但前提是指標更新順序正確，否則會斷鏈或形成 cycle。

作答時要同時畫出 first chain 與 avail chain，並用表格列出每個 index 的 link，才能完整檢查是否有節點重複使用。

完整答案

動作指標更新意義
配置節點p = avail; avail = link[p];從 available list 拿出第一個空節點。
釋放節點link[p] = avail; avail = p;把 p 接到 available list 的最前面。
檢查主串列與 avail list 不重疊避免同一節點被使用兩次。

  • Available list 是把未使用陣列格子用 link 串起來。
  • 這是用陣列模擬動態記憶體配置的方法。
  • 配置與釋放都可以做到 Θ(1)。
  • 若 avail=0 代表沒有空節點。

int link[MAX];  // link[i] 指向下一個節點

    int p = avail;
    avail = link[p];
    return p;
}

void freeNode(int p){
    link[p] = avail;
    avail = p;
